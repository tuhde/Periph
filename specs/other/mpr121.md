# Chip Spec: MPR121

**Manufacturer:** Freescale Semiconductor (NXP)  
**Datasheet:** `datasheets/other/mpr121.pdf`  
**Category:** `other`  
**Transports:** I²C

## Overview

The MPR121 is a second-generation proximity capacitive touch sensor controller. It measures capacitance on up to 12 independent electrode inputs (ELE0–ELE11) using a constant-DC-current charge scheme, capable of sensing capacitances from 10 pF to over 2000 pF with 0.01 pF resolution. A 13th virtual "ELEPROX" electrode sums all physical inputs to form a single large proximity-detection surface. Each channel has independent baseline tracking, configurable touch/release thresholds with hysteresis, and a debounce count. An active-low open-collector IRQ pin fires on any touch or release event, making interrupt-driven operation straightforward.

The chip has two operating modes: **Stop Mode** (default after reset, no measurement) and **Run Mode** (controlled by ECR register). Configuration registers may only be written in Stop Mode; only the Electrode Configuration Register (ECR at 0x5E) and GPIO registers (0x73–0x7A) may be written at any time. Eight of the twelve electrode pins (ELE4–ELE11) are multifunctional and can also serve as GPIO outputs or LED drivers when not used for touch sensing.

Supply voltage: 1.71–3.6 V (low supply 1.71–2.75 V with VDD+VREG shorted; high supply 2.0–3.6 V with 0.1 µF on VREG). Current: 29 µA typical at 16 ms scan interval; 3 µA in Stop Mode. Package: 3 × 3 × 0.65 mm 20-pin QFN.

## Transport Configuration

### I²C

- **Address:** `0x5A` (ADDR=VSS) / `0x5B` (ADDR=VDD) / `0x5C` (ADDR=SDA) / `0x5D` (ADDR=SCL)
- **Max clock:** 400 kHz

Auto-increment is supported for specific register ranges:
- Electrode Filtered Data (0x04–0x1D): auto-increment wraps within the block.
- Electrode Charge Time (0x6C–0x72): auto-increment wraps within the block.

All other registers must be accessed individually (no auto-increment).

#### `ELE8_PROX_TOUCH` (`0x01`)

| 7 | `OVCF` | Over-current flag. Write 0 to clear; device enters Stop Mode when set |

#### Touch status combined (read both 0x00 and 0x01)

The full 13-bit touch bitmask is assembled as: `(reg01 & 0x1F) << 8 | reg00`. Bit N = ELEn touched; bit 12 = ELEPROX. Reading both registers in sequence (start at 0x00, read 2 bytes) captures a coherent snapshot.

#### Electrode Filtered Data (`0x04`–`0x1D`)

Each electrode occupies two consecutive bytes: LSB (even address) and MSB (odd address). The 10-bit value is: `data = lsb | ((msb & 0x03) << 8)`. Range 0–1023; value is inversely proportional to capacitance. ELEPROX at 0x1C–0x1D.

Per-electrode address: `lsb_addr = 0x04 + electrode * 2` (ELE0–ELE11); ELEPROX = `0x1C`.

#### Electrode Baseline Value (`0x1E`–`0x2A`)

Each register holds the 8 MSBs of the 10-bit baseline. To compare against filtered data: `baseline_10bit = register_value << 2`. Address: `0x1E + electrode` (ELE0–ELE11); ELEPROX = `0x2A`. Writable in Stop Mode only.

#### Touch/Release Thresholds (`0x41`–`0x5A`)

Touch threshold for ELEn: `0x41 + n*2`. Release threshold for ELEn: `0x42 + n*2`.  
ELEPROX: touch=`0x59`, release=`0x5A`. Range 0–255.

Touch condition: `baseline_10bit − filtered_data > touch_threshold`  
Release condition: `baseline_10bit − filtered_data < release_threshold`

Typical values: touch = 12 (0x0C), release = 6 (0x06). Touch must always be greater than release.

#### `DEBOUNCE` (`0x5B`)

| Bits | Name | Description |
|------|------|-------------|
| 6:4 | `DR` | Debounce count for release (0–7 consecutive detections required) |
| 2:0 | `DT` | Debounce count for touch (0–7) |

#### `CDC_CONFIG` (`0x5C`) — default `0x10`

| Bits | Name | Description |
|------|------|-------------|
| 7:6 | `FFI` | First filter iterations: 00=6, 01=10, 10=18, 11=34 samples |
| 5:0 | `CDC` | Global charge/discharge current: 0=disabled, 1–63 µA (16=default) |

#### `CDT_CONFIG` (`0x5D`) — default `0x24`

| Bits | Name | Description |
|------|------|-------------|
| 7:5 | `CDT` | Global charge/discharge time: 0=disabled, 1=0.5 µs, 2=1 µs…7=32 µs (1=default) |
| 3:2 | `SFI` | Second filter iterations: 00=4, 01=6, 10=10, 11=18 samples |
| 2:0 | `ESI` | Electrode sample interval: 0=1 ms, 1=2 ms…4=16 ms (default), 7=128 ms |

Note: CDT bits [7:5], SFI bits [3:2], ESI bits [2:0] — ESI[2:0] occupies bits 2:0.

#### `ECR` — Electrode Configuration Register (`0x5E`)

| Bits | Name | Description |
|------|------|-------------|
| 7:6 | `CL` | Calibration Lock / baseline init: 00=tracking on, init from baseline register; 01=tracking off; 10=tracking on, init from 5 MSBs of first measurement; 11=tracking on, init from all 10 bits of first measurement |
| 5:4 | `ELEPROX_EN` | Proximity enable: 00=off, 01=ELE0–ELE1, 10=ELE0–ELE3, 11=ELE0–ELE11 summed |
| 3:0 | `ELE_EN` | Number of electrodes to enable: 0=Stop, 1=ELE0 only, 2=ELE0–ELE1, …, 12(0xC)=ELE0–ELE11. Any value ≥12 enables all 12 |

Writing a non-zero ECR enters Run Mode immediately. Writing ECR=0x00 enters Stop Mode.

#### `AUTOCONFIG0` (`0x7B`)

| Bits | Name | Description |
|------|------|-------------|
| 7:6 | `FFI` | Same as CDC_CONFIG FFI — must match |
| 5:4 | `RETRY` | Retries if auto-config fails: 00=none, 01=2×, 10=4×, 11=8× |
| 3:2 | `BVA` | Same as ECR CL bits — baseline tracking mode used during auto-config |
| 1 | `ARE` | Auto-reconfiguration enable (retry on OOR each scan) |
| 0 | `ACE` | Auto-configuration enable (run once on Stop→Run transition) |

#### `AUTOCONFIG1` (`0x7C`)

| Bits | Name | Description |
|------|------|-------------|
| 7 | `SCTS` | Skip charge time search (use current CDT/CDTx instead of searching) |
| 2 | `OORIE` | OOR interrupt enable |
| 1 | `ARFIE` | Auto-reconfiguration fail interrupt enable |
| 0 | `ACFIE` | Auto-configuration fail interrupt enable |

## Initialization Sequence

### Minimal (12 electrodes, autoconfiguration enabled)

1. Write `0x63` to `SRST` (`0x80`) to software reset. Wait 1 ms.
2. (Device is now in Stop Mode; ECR=0x00.)
3. Set touch thresholds: write `0x0C` to each of `0x41, 0x43, 0x45, 0x47, 0x49, 0x4B, 0x4D, 0x4F, 0x51, 0x53, 0x55, 0x57`.
4. Set release thresholds: write `0x06` to each of `0x42, 0x44, 0x46, 0x48, 0x4A, 0x4C, 0x4E, 0x50, 0x52, 0x54, 0x56, 0x58`.
5. Set baseline filter defaults: write `0x01` to `MHDR` (0x2B), `NHDR` (0x2C), `MHDF` (0x2F), `NHDF` (0x30).
6. Set auto-configuration USL/TL/LSL based on VDD (e.g., for 3.3 V: USL=0xC9, TL=0xB4, LSL=0x82; see formulas).
7. Write `AUTOCONFIG0` (0x7B) = `0x0B` (FFI=00, RETRY=00, BVA=10, ARE=1, ACE=1).
8. Write `ECR` (0x5E) = `0x8C` (CL=10, ELEPROX_EN=00, ELE_EN=1100 → all 12). Device enters Run Mode.

### Without autoconfiguration (manual CDC/CDT)

Steps 1–5 same. Skip steps 6–7. Set `CDC_CONFIG` (0x5C) and `CDT_CONFIG` (0x5D) as desired or leave defaults. Write ECR = `0x8C`.

## Data Conversion

### Filtered data

Filtered data is a 10-bit unsigned integer. It is inversely proportional to capacitance: higher capacitance → lower data value.

```python
lsb = read(0x04 + electrode * 2)
msb = read(0x04 + electrode * 2 + 1)
filtered_10bit = lsb | ((msb & 0x03) << 8)
```

ELEPROX (electrode=12): `lsb = read(0x1C); msb = read(0x1D)`.

### Baseline

The baseline register contains bits [9:2] of the 10-bit baseline (only the 8 MSBs are stored). To compare against filtered data:

```python
baseline_10bit = read(0x1E + electrode) << 2
```

### Touch detection logic

```
delta = baseline_10bit - filtered_10bit
if delta > touch_threshold:   # electrode touched
if delta < release_threshold: # electrode released
```

### Autoconfiguration limits (USL/TL/LSL)

Given VDD in millivolts:

```
USL = int((VDD_mV - 700) / VDD_mV * 256)   # e.g. 3300 mV → 201 = 0xC9
TL  = int(USL * 0.9)                        # e.g. 181 = 0xB5 (round down)
LSL = int(USL * 0.65)                       # e.g. 130 = 0x82
```

These are 8-bit values but represent the high 8 bits of a 10-bit electrode data target; LSL < TL < USL must hold.

## Timing Constraints

- Soft reset (write 0x63 to 0x80): wait at least 1 ms before proceeding; I²C module is not reset.
- Stop→Run transition (ECR write): auto-configuration (if ACE=1) runs before first measurement; allow up to `ESI × FFI_samples × 13 × 2` ms for autoconfiguration to complete before expecting valid touch data. Practical safe wait: 50 ms.
- Config register writes (0x2B–0x7F except ECR): only valid in Stop Mode. Writing in Run Mode is silently ignored.
- IRQ pin: asserted low on any touch or release event; cleared by reading both touch status registers (0x00 and 0x01).
- Multi-byte read from 0x00: reading two bytes starting at 0x00 returns both status registers as a coherent snapshot (no partial-update race).

## Implementation Notes

- **No chip ID register:** The MPR121 has no WHO_AM_I equivalent. Presence is confirmed by writing a known value to a threshold register in Stop Mode and reading it back.
- **Stop Mode for config:** The driver must guarantee ECR=0x00 before writing any register 0x2B–0x7F (except ECR itself and GPIO registers 0x73–0x7A). The recommended pattern is: `stop()` → configure → `start()`.
- **OVCF handling:** If the over-current flag (OVCF, bit 7 of register 0x01) is set, the device enters Stop Mode and all other status bits are cleared. Clear OVCF by writing 0 to bit 7 of 0x01, then reconfigure ECR to re-enter Run Mode.
- **Baseline ramp-up:** When CL=00 (baseline init from baseline register), a freshly reset device has baseline=0 and will falsely report all electrodes as touched until the baseline ramps up. Use CL=10 or CL=11 in ECR to load the baseline from the first measured electrode data, avoiding the ramp-up delay.
- **Threshold polarity:** Touch and release are triggered by the *difference* (baseline − filtered). Since filtered data decreases when capacitance increases (touch), the delta is positive when touched. Thresholds are positive integers; release threshold must always be lower than touch threshold.
- **ELEPROX_EN in ECR:** Setting ELEPROX_EN=11 (all 12 electrodes summed) consumes extra scan time. Enable proximity only when needed.
- **GPIO pins ELE4–ELE11:** ECR has higher priority; if a pin is configured as GPIO (EN bit in 0x77 set) but also enabled as an electrode (ELE_EN in ECR covers it), the GPIO function is immediately disabled. Always disable the electrode in ECR before enabling GPIO.
- **AUTOCONFIG0 FFI field:** Must match the FFI field in CDC_CONFIG (0x5C) for correct autoconfiguration results.
- **CDT[7:5] in CDT_CONFIG (0x5D):** This is the global CDT occupying bits 7:5. ESI occupies bits 2:0, and SFI occupies bits 3:2.

## Sigrok Decoder

Decoder ID: `mpr121`  
Transport: I²C  
I²C addresses decoded: `0x5A`, `0x5B`, `0x5C`, `0x5D`

Annotates: all register reads and writes with symbolic names. Touch status reads decoded as per-electrode touch state ("ELE0: touch", "ELE3: release", etc.). Filtered data reads decoded to raw 10-bit values. Baseline reads decoded to 10-bit values (shifted left 2). Threshold writes decoded with electrode number.
