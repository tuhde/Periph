# Chip Spec: PCF8576

**Manufacturer:** NXP Semiconductors  
**Datasheet:** `datasheets/display/pcf8576.pdf`  
**Category:** `display`  
**Transports:** I²C

## Overview

The PCF8576 is a 40×4 universal LCD segment driver for low multiplex rates. It generates the drive waveforms for any static or multiplexed LCD with up to 4 backplane signals and up to 40 segment outputs, allowing it to drive up to 160 individual LCD elements. A 40×4-bit internal display RAM stores the segment state; writing display data over I²C updates what is shown without any CPU involvement in the AC drive waveforms. The chip is write-only — it never returns data to the controller. Up to 16 PCF8576s can be cascaded on a single I²C bus using hardware subaddresses and the SA0 pin, enabling very large LCD panels.

## Transport Configuration

### I²C
- **Address:** `0x38` (SA0 pin = VSS) — `0x39` (SA0 pin = VDD)
- **Max clock:** 400 kHz
- **Write-only:** the device never transmits data; do not issue read transactions

## Command Set

The PCF8576 uses a command-stream protocol rather than addressed registers. Each I²C write transaction contains one or more command bytes followed by zero or more display data bytes.

**Continuation bit (bit 7, symbol C):**
- `C = 1`: more command bytes follow immediately after this byte
- `C = 0`: this is the last command byte; all subsequent bytes are display data

| Command | Bit pattern | Description |
|---------|-------------|-------------|
| mode-set | `C 1 0 x E B M1 M0` | Configure drive mode, bias, display on/off |
| load-data-pointer | `C 0 P5 P4 P3 P2 P1 P0` | Set display RAM write address (0–39) |
| device-select | `C 1 1 0 0 A2 A1 A0` | Set subaddress counter (0–7); for cascaded use |
| bank-select | `C 1 1 1 1 0 I O` | Select input/output RAM bank; static and 1:2 only |
| blink-select | `C 1 1 1 0 AB BF1 BF0` | Configure blinking |

### mode-set bit fields

| Bits | Symbol | Values |
|------|--------|--------|
| 7 | C | continuation (see above) |
| 6:5 | — | `10` fixed |
| 4 | — | unused |
| 3 | E | `0` = display disabled (all outputs = V_LCD); `1` = enabled |
| 2 | B | `0` = 1/3 bias (default); `1` = 1/2 bias |
| 1:0 | M[1:0] | `00` = 1:4 multiplex (default); `01` = static; `10` = 1:2 multiplex; `11` = 1:3 multiplex |

### load-data-pointer bit fields

| Bits | Symbol | Values |
|------|--------|--------|
| 7 | C | continuation |
| 6 | — | `0` fixed |
| 5:0 | P[5:0] | RAM column address, 0–39 (default 0) |

### device-select bit fields

| Bits | Symbol | Values |
|------|--------|--------|
| 7 | C | continuation |
| 6:3 | — | `1100` fixed |
| 2:0 | A[2:0] | Hardware subaddress, 0–7; must match A0/A1/A2 pin state |

### bank-select bit fields

| Bits | Symbol | Values |
|------|--------|--------|
| 7 | C | continuation |
| 6:2 | — | `11110` fixed |
| 1 | I | Input bank: `0` = rows 0–1 (default); `1` = rows 2–3 |
| 0 | O | Output bank: `0` = rows 0–1 (default); `1` = rows 2–3 |

### blink-select bit fields

| Bits | Symbol | Values |
|------|--------|--------|
| 7 | C | continuation |
| 6:3 | — | `1110` fixed |
| 2 | AB | `0` = normal blinking (default); `1` = alternate RAM bank blinking (static/1:2 only) |
| 1:0 | BF[1:0] | `00` = off (default); `01` = ~2 Hz; `10` = ~1 Hz; `11` = ~0.5 Hz |

### Power-on reset defaults

| Setting | Default |
|---------|---------|
| Display enable (E) | 0 (off) |
| Drive mode (M) | `00` = 1:4 multiplex |
| Bias (B) | 0 = 1/3 bias |
| Blinking | off |
| Data pointer | 0 |
| Subaddress counter | 0 |
| Display RAM | all 0 (all segments off) |

## Initialization Sequence

1. Wait ≥1 ms after power-on for internal power-on reset to complete
2. Send device-select command (with `C=1`) to set subaddress matching the A0/A1/A2 pin state
3. Send mode-set command (with `C=0` if no data follows, else `C=1`) to set drive mode, bias, and enable the display
4. Send load-data-pointer command (with `C=0`) to reset the write address to 0
5. Send 20 data bytes (1:4 multiplex, one byte per digit) or 5 data bytes (static, one byte per 8 segments) to initialise all RAM to blank

For single-device applications with A0/A1/A2 tied to VSS, steps 2 and 4 may be omitted on reset because the POR already sets subaddress=0 and data pointer=0; however explicitly sending them is recommended for robustness.

## Implementation Stages

### Minimal

Goal: drive a single 7-segment LCD display (static or 1:4 multiplex) out of the box. No configuration beyond the transport.

| Operation | Parameters | Returns | Notes |
|-----------|------------|---------|-------|
| `init` | transport | — | Runs init sequence with defaults below |
| `clear` | — | — | Zero all 40 columns of display RAM; all segments off |
| `write_raw` | `address: int`, `data: bytes` | — | Set data pointer to `address`, write raw data bytes |
| `set_digit_7seg` | `position: int`, `segments: int` | — | Write one 7-segment byte at column `position × 2`; see encoding below |

**Sensible defaults:**
- Drive mode: 1:4 multiplex (4 backplanes)
- Bias: 1/3
- Display enabled (E=1)
- Subaddress: 0 (hardware pins A0/A1/A2 = VSS)
- Display RAM cleared to 0 on init

### Full

| Operation | Parameters | Returns | Notes |
|-----------|------------|---------|-------|
| *(inherits Minimal)* | | | |
| `enable` | — | — | Turn display on (E=1) |
| `disable` | — | — | Blank display output (E=0); RAM contents preserved |
| `set_mode` | `backplanes: int`, `bias: int` | — | Reconfigure drive mode and bias at runtime |
| `set_blink` | `frequency: int`, `alternate_bank: bool` | — | Set blink frequency (0=off, 1=~2Hz, 2=~1Hz, 3=~0.5Hz); alternate_bank for static/1:2 only |

**Additional configuration options:**
- `backplanes`: 1 (static), 2 (1:2 multiplex), 3 (1:3 multiplex), 4 (1:4 multiplex)
- `bias`: 0 = 1/3 (recommended for 1:3 and 1:4 multiplex), 1 = 1/2
- Alternate-bank blinking: prepare the alternate RAM bank with a different image; the chip swaps banks at the blink frequency with no CPU intervention

## Data Encoding

### Display RAM layout

The 40×4-bit RAM has columns 0–39 corresponding to segment outputs S0–S39, and rows 0–3 corresponding to backplane outputs BP0–BP3. A `1` bit lights the segment; `0` extinguishes it.

### 7-segment encoding in 1:4 multiplex mode (default)

In 1:4 multiplex mode each transmitted byte covers 2 adjacent RAM columns and one digit of a 7-segment + decimal point display. The data pointer auto-increments by 2 after each byte.

Digit position `p` maps to data pointer address `p × 2`. Maximum 20 digits (40 segments ÷ 2).

**Byte bit assignment (MSB→LSB):**

| Bit | 7 | 6 | 5 | 4 | 3 | 2 | 1 | 0 |
|-----|---|---|---|---|---|---|---|---|
| Segment | a | c | b | DP | f | e | g | d |

Where standard 7-segment labels are: a=top, b=upper-right, c=lower-right, d=bottom, e=lower-left, f=upper-left, g=middle, DP=decimal point.

**Digit lookup (DP=0):**

| Digit | Byte |
|-------|------|
| 0 | `0xED` |
| 1 | `0x60` |
| 2 | `0xA7` |
| 3 | `0xE3` |
| 4 | `0x6A` |
| 5 | `0xCB` |
| 6 | `0xCF` |
| 7 | `0xE0` |
| 8 | `0xEF` |
| 9 | `0xEB` |
| blank | `0x00` |

Add `0x10` to any byte to set the decimal point.

### Static drive mode

In static mode each byte covers 8 adjacent columns. The data pointer auto-increments by 8 after each byte. All 8 bits map directly to 8 segment outputs (LSB = lowest-numbered segment in the group). Maximum 40 independent segments or 5 bytes to fill the whole RAM.

## Timing Constraints

- Wait ≥1 ms after power-on before first I²C transaction (POR settling time)
- If an I²C transaction is aborted early, the data pointer state is unknown — always resend a load-data-pointer command before the next display write
- Do not remove the clock (CLK / internal oscillator) while the display is active — freezing the clock locks the LCD in a DC state, which degrades the liquid crystal
- V_LCD and V_DD must be applied and removed together to prevent DC build-up across the LCD panel

## Implementation Notes

- The PCF8576 is **write-only**; `transport.read()` and `transport.write_read()` must never be called. The driver only uses `transport.write()`.
- The device-select command sets a **subaddress counter**, not the I²C address. In single-device use (A0/A1/A2 = VSS) it is safe to skip device-select on every write because the subaddress counter is already 0 after POR. In cascaded use, send device-select before every update to target the correct device in the chain.
- In 1:3 multiplex mode the unused bit at position BP2/S2, BP2/S5, BP2/S8 … is shared between adjacent characters. The datasheet recommends against using BP2/S(3n+2) in 1:3 mode, or accounting for it by rewriting the full byte.
- Bank-select and alternate-bank blinking have no effect in 1:3 and 1:4 multiplex modes.
- The SA0 pin selects one of two I²C address clusters (0x38 or 0x39). Combined with A0/A1/A2, up to 16 devices can share a single bus.
- The chip is light-sensitive; in bare-die form it must be protected from ambient light.
