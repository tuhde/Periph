# Chip Spec: WS2814

**Manufacturer:** Super Lighting LED  
**Datasheet:** `datasheets/led/ws2814.pdf`  
**Category:** led  
**Transports:** NeoPixel (WS2812B single-wire NZR via SPI bit-encoding)

## Overview

The WS2814 is an addressable RGBW LED driver IC with four constant-current output channels (R, G, B, W). It uses the standard NeoPixel single-wire NZR protocol at 800 Kbps — identical to the WS2812B and SK6812RGBW — with two distinctions: the 32-bit data word is transmitted in **RGBW** order (no reorder needed) and the reset pulse must be ≥280 µs (longer than both predecessors). The IC supports 12 V or 24 V LED supplies via an internal voltage-regulator stage; the logic side runs from 3.7–5.3 V. A secondary data input (DIN2) provides an auxiliary signal path for redundancy on long strips.

The driver in this library sits on top of the NeoPixel transport (`specs/transport_neopixel.md`). The chip spec for API, implementation notes, and Node-RED conventions is `specs/led/_neopixel_rgbw_base.md`. This document records only WS2814-specific chip values.

## Chip-Specific Values

| Property | Value | Notes |
|----------|-------|-------|
| Wire channel order | RGBW | `channel_order = (0, 1, 2, 3)` — identity mapping, no reorder |
| Reset pulse | ≥280 µs | `reset_bytes = 90` (90 × 3.333 µs ≈ 300 µs at 2.4 MHz SPI) |
| Bits per pixel | 32 | 4 channels × 8 bits |
| Data rate | 800 Kbps | 1.25 µs per bit |
| Max pixels at 30 fps | ~826 | 826 × 40 µs + 280 µs ≈ 33.3 ms |

## Transport Configuration

### NeoPixel

- **Encoding:** SPI bit-encoding at 2.4 MHz (see `specs/transport_neopixel.md`)
- **Bit order:** MSB-first per byte; channel order is RGBW (wire bytes: R7–R0, G7–G0, B7–B0, W7–W0)
- **Frame format:** `n × 4` bytes (32 bits per pixel), followed by ≥280 µs reset (90 SPI zero-bytes)
- **Max pixels at 30 fps:** ~826

## Pixel Data Format

### 32-bit pixel word (MSB-first)

| Bits 31–24     | Bits 23–16     | Bits 15–8      | Bits 7–0       |
|----------------|----------------|----------------|----------------|
| Red (R7–R0)    | Green (G7–G0)  | Blue (B7–B0)   | White (W7–W0)  |

User (R, G, B, W) → wire order is identity: no reorder. The driver writes R, G, B, W bytes directly.

## Timing Constraints

- **Bit period:** T0H+T0L, T1H+T1L ≥ 1.25 µs
- **T0H:** 220–380 ns; **T0L:** 580 ns–1 µs
- **T1H:** 580 ns–1 µs; **T1L:** 580 ns–1 µs
- **Reset pulse (RES):** ≥280 µs low → 90 zero-bytes at 2.4 MHz SPI (≈300 µs)
- **Transmission time for n pixels:** n × 40 µs + 300 µs reset
  - 30 pixels ≈ 1.5 ms; 300 pixels ≈ 12.3 ms; 826 pixels ≈ 33.3 ms (≈30 fps)
- **T0H timing note:** the standard 2.4 MHz SPI encoding produces T0H ≈ 417 ns, slightly above the 380 ns upper bound. In practice the WS2814 accepts this (the data cycle period ≥1.25 µs is satisfied). This is consistent with all other NeoPixel implementations.

## Absolute Maximum Ratings

| Parameter | Value |
|-----------|-------|
| VDD (logic supply) | 3.7–5.3 V |
| R/G/B/W output withstand | 20 V |
| Logic input voltage | 0.7–VDD+0.7 V |
| Operating temperature | −25 to +85 °C |
| Output current per channel | ~16.5 mA (typ) |

## Dual-Input Feature

The WS2814 has two data inputs:

- **DIN1** — primary data input; standard cascade connection from the MCU or the previous pixel's DOUT
- **DIN2** — auxiliary data input; hardware backup path for fault tolerance on long runs

DIN2 requires no software support. The driver always drives DIN1. DIN2 is a hardware design concern (PCB layout, signal integrity on long strips).

## Typical Supply Configurations

| Supply | LEDs per channel | Notes |
|--------|-----------------|-------|
| 12 V | 3 | Built-in voltage regulator drops supply to ~3.3 V for IC; add 100 Ω VDD resistor |
| 24 V | 6 | Same scheme; 2.5 kΩ VDD resistor |

### Demo flow

An Inject node fires every 2 seconds alternating between warm-white (`{ r:255, g:200, b:150, w:255 }`), cool-white (`{ r:200, g:210, b:255, w:255 }`), and off (`{ command:"off" }`). A second Inject node sends an RGB rainbow fill via `{ pixels:[[r,g,b,0], …] }` covering 30 pixels to showcase that the white channel defaults to zero when unused. The flow demonstrates the dedicated white element and 12 V/24 V strip compatibility.

## Examples

### Demo

Demonstrate warm-white + RGB rainbow on a 30-pixel WS2814 strip: begin with 5 seconds of rainbow rotation (RGB channels, w=0 per pixel) at ~30 fps, then flash warm white (r=255, g=200, b=150, w=255) at full brightness for 2 seconds, then dim the warm white to 50% using the brightness property and hold for 2 seconds, then cycle to cool white (r=200, g=210, b=255, w=255). Print the current mode and brightness once per second. This exercises the white channel, brightness property, per-pixel RGBW addressing, and HSV convenience method, and demonstrates that the WS2814's RGBW order requires no reorder compared to the SK6812RGBW.

## Sigrok Decoder

Two-layer stack: `logic → neopixel (reset_us=280) → ws2814`.

The `neopixel` transport decoder handles all NZR timing. For WS2814, set its `reset_us` option to `280` (the chip requires ≥280 µs vs SK6812RGBW's ≥80 µs).

The `ws2814` chip decoder (id `ws2814`, input `['neopixel']`) groups bytes into 32-bit RGBW pixels and annotates each pixel with its index, R/G/B/W values (no reorder — wire order is RGBW), and a `#RRGGBBWW` hex colour string. Emits a warning on incomplete pixels at reset.
