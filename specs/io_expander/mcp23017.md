# Chip Spec: MCP23017 (IO Expander)

**Manufacturer:** Microchip Technology  
**Datasheet:** `datasheets/io_expander/mcp23017.pdf`  
**Category:** `io_expander`  
**Transports:** I²C

## Overview

The MCP23017 is a 16-bit general-purpose bidirectional I/O port expander that communicates with a host via I²C. Its 16 GPIO pins are split into two 8-bit ports (PORTA: GPA0–GPA7, PORTB: GPB0–GPB7), each with dedicated direction (IODIR), pull-up (GPPU), and output-latch (OLAT) registers. Unlike single-byte quasi-bidirectional expanders, direction is controlled explicitly: setting an IODIR bit to 1 makes the pin an input; clearing it makes the pin an output. Each pin can source or sink up to 25 mA. An internal 100 kΩ pull-up can be enabled per-pin on pins configured as inputs. Two independent interrupt outputs (INTA for PORTA, INTB for PORTB) can be asserted on any input change or when a pin differs from a preconfigured default value; the interrupt capture register (INTCAP) freezes the port state at the moment the interrupt occurred. Three hardware address pins (A0, A1, A2) allow up to eight MCP23017 devices on the same I²C bus. Note: on the MCP23017, GPA7 and GPB7 are output-only; their IODIR bits must be cleared (set as output) via software.

## Transport Configuration

### I²C
- **Address:** `0x20` (default, A2=A1=A0=0) — `0x21`–`0x27` (A2:A0 bitmask added to `0x20`)
- **Max clock:** 100 kHz @ 1.8V–5.5V; 400 kHz @ 2.7V–5.5V; 1.7 MHz @ 4.5V–5.5V

## Register Map

The driver uses IOCON.BANK = 0 (power-on reset default) throughout and must never change the BANK bit. In BANK=0 mode, Port A and Port B registers alternate in ascending address order.

| Address | Name      | R/W | Reset  | Description |
|---------|-----------|-----|--------|-------------|
| `0x00`  | IODIRA    | R/W | `0xFF` | Direction A: 1=input, 0=output |
| `0x01`  | IODIRB    | R/W | `0xFF` | Direction B |
| `0x02`  | IPOLA     | R/W | `0x00` | Input polarity A: 1=inverted |
| `0x03`  | IPOLB     | R/W | `0x00` | Input polarity B |
| `0x04`  | GPINTENA  | R/W | `0x00` | Interrupt-on-change enable A |
| `0x05`  | GPINTENB  | R/W | `0x00` | Interrupt-on-change enable B |
| `0x06`  | DEFVALA   | R/W | `0x00` | Default compare value A (for INTCON=1 mode) |
| `0x07`  | DEFVALB   | R/W | `0x00` | Default compare value B |
| `0x08`  | INTCONA   | R/W | `0x00` | Interrupt control A: 1=compare vs DEFVAL, 0=compare vs previous |
| `0x09`  | INTCONB   | R/W | `0x00` | Interrupt control B |
| `0x0A`  | IOCON     | R/W | `0x00` | I/O expander configuration |
| `0x0B`  | IOCON     | R/W | `0x00` | Same register — alternate address |
| `0x0C`  | GPPUA     | R/W | `0x00` | Pull-up enable A: 1=100 kΩ pull-up enabled |
| `0x0D`  | GPPUB     | R/W | `0x00` | Pull-up enable B |
| `0x0E`  | INTFA     | R   | `0x00` | Interrupt flag A: 1=pin caused interrupt |
| `0x0F`  | INTFB     | R   | `0x00` | Interrupt flag B |
| `0x10`  | INTCAPA   | R   | —      | Interrupt captured A: port state at interrupt time |
| `0x11`  | INTCAPB   | R   | —      | Interrupt captured B |
| `0x12`  | GPIOA     | R/W | `0x00` | Port A: read=pin levels; write=output latch (same as OLATA) |
| `0x13`  | GPIOB     | R/W | `0x00` | Port B |
| `0x14`  | OLATA     | R/W | `0x00` | Output latch A: read=latch value (not pin level) |
| `0x15`  | OLATB     | R/W | `0x00` | Output latch B |

### Bit Fields

#### `IOCON` (`0x0A`)

| Bits | Name    | Description |
|------|---------|-------------|
| 7    | BANK    | Register bank mode: 1=Port A and B registers in separate banks; 0=interleaved (default) |
| 6    | MIRROR  | INT mirror: 1=INTA and INTB ORed (either port activates both); 0=INTA→PORTA, INTB→PORTB |
| 5    | SEQOP   | Sequential operation: 1=address pointer does not auto-increment; 0=auto-increment (default) |
| 4    | DISSLW  | SDA slew rate: 1=disabled; 0=enabled |
| 3    | HAEN    | Hardware address enable (MCP23S17 SPI variant only; ignored on MCP23017) |
| 2    | ODR     | INT pin as open-drain: 1=open-drain (overrides INTPOL); 0=active driver |
| 1    | INTPOL  | INT polarity when ODR=0: 1=active-high; 0=active-low |
| 0    | —       | Unimplemented, read as 0 |

#### `IODIRA` / `IODIRB` (`0x00`/`0x01`)

| Bits | Name     | Description |
|------|----------|-------------|
| 7    | IO7      | Pin direction bit 7. **Note:** for MCP23017, GPA7/GPB7 are output-only; this bit must be 0. |
| 6:0  | IO6:IO0  | Pin direction bits 6–0: 1=input, 0=output |

#### `GPINTENA` / `GPINTENB` (`0x04`/`0x05`)

| Bits | Name       | Description |
|------|------------|-------------|
| 7:0  | GPINT7:0   | 1=pin enabled for interrupt-on-change; 0=disabled |

#### `INTCONA` / `INTCONB` (`0x08`/`0x09`)

| Bits | Name     | Description |
|------|----------|-------------|
| 7:0  | IOC7:0   | 1=compare against DEFVAL register; 0=compare against previous pin value |

## Pin Capabilities

| Capability | Support |
|-----------|---------|
| Pin count | 16 (GPA0–GPA7 + GPB0–GPB7) |
| Ports | 2 × 8-bit (PORTA, PORTB) |
| Direction | per-pin via IODIR register (1=input, 0=output) |
| Pull-up | yes — internal 100 kΩ per-pin when configured as input (GPPU register) |
| Pull-down | no |
| Open-drain output | no — push-pull output only |
| Push-pull output | yes — source and sink up to 25 mA per pin |
| Interrupt output | yes — INTA (PORTA) and INTB (PORTB); configurable active-high, active-low, or open-drain |
| Interrupt modes | interrupt-on-change (from previous value) or interrupt-on-default-mismatch (vs DEFVAL) |
| Drive strength | no |

## Initialization Sequence

1. Write OLATA = `0x00`, OLATB = `0x00` — clear output latches before enabling outputs.
2. Write IODIRA = `0x7F`, IODIRB = `0x7F` — GPA0–GPA6 and GPB0–GPB6 as inputs; GPA7 and GPB7 as outputs (required by hardware).
3. All other registers remain at power-on defaults: GPPU = `0x00` (pull-ups off), IPOL = `0x00`, GPINTEN = `0x00` (interrupts disabled), IOCON = `0x00`.

No startup delay is required after power-on reset or after the RESET pin is released.

## Implementation Stages

Each chip is implemented in two stages. The Full class extends Minimal.

The defining characteristic of an IO expander driver is that it exposes individual **Pin objects** that implement each platform's native GPIO interface — see the GPIO Interface section below. Users obtain a pin from the driver and use it exactly like a hardware GPIO pin.

### Minimal

Goal: expose all 16 pins as GPIO objects with direction control and value read/write. No interrupt or pull-up configuration required.

**Driver API**

| Operation | Parameters | Returns | Notes |
|-----------|------------|---------|-------|
| `__init__` | `transport`, `addr=0x20` | — | Clear OLAT; set IODIR = 0x7F for both ports; store shadow OLAT = [0, 0] |
| `pin` | `n: int` | Pin | Return Pin proxy for 0-based pin number; 0–7 = PORTA, 8–15 = PORTB |
| `read_port` | `port: int` | int | Read all 8 pins of port (0=PORTA, 1=PORTB) from GPIOA/GPIOB |
| `write_port` | `port: int`, `mask: int` | — | Write all 8 output pins of port via OLATA/OLATB; update shadow |

**Sensible defaults:** IODIRA = IODIRB = `0x7F` (GPA7/GPB7 as outputs, rest as inputs); OLATA = OLATB = `0x00`; pull-ups disabled; interrupts disabled. Shadow register maintained for OLATA and OLATB to enable bit-level read-modify-write without an I²C read.

**Pin API — Minimal**

| Operation | MicroPython | CircuitPython | C++ | Node.js | Rust |
|-----------|-------------|---------------|-----|---------|------|
| Get pin | `chip.pin(n, Pin.IN)` | `chip.pin(n)` | `chip.pin(n)` | `chip.pin(n, 'in')` | `chip.pin(n)` |
| Set direction | `pin.init(Pin.OUT)` | `pin.direction = Direction.OUTPUT` | `pin.mode(OUTPUT)` | `pin.setDirection('out', cb)` | *(type-level: `OutputPin` vs `InputPin`)* |
| Set high | `pin.on()` | `pin.value = True` | `pin.high()` | `pin.writeSync(1)` | `pin.set_high()?` |
| Set low | `pin.off()` | `pin.value = False` | `pin.low()` | `pin.writeSync(0)` | `pin.set_low()?` |
| Read | `pin.value()` | `pin.value` | `pin.read()` | `pin.readSync()` | `pin.is_high()?` |
| Toggle | `pin.toggle()` | *(manual)* | `pin.toggle()` | *(manual)* | *(manual)* |
| Release / close | — | `pin.deinit()` | — | `pin.unexport()` | *(drop)* |

### Full

Goal: expose complete chip functionality. Extends Minimal.

**Driver API additions**

| Operation | Parameters | Returns | Notes |
|-----------|------------|---------|-------|
| *(inherits Minimal)* | | | |
| `configure_pullup` | `port: int`, `mask: int` | — | Enable/disable per-pin pull-ups via GPPUA/GPPUB; only active on input pins |
| `configure_polarity` | `port: int`, `mask: int` | — | Set polarity inversion via IPOLA/IPOLB; 1=invert GPIO read |
| `configure_interrupt` | `port: int`, `int_pin`, `callback`, `mode='change'`, `mirror=False` | — | Enable interrupt for port; `mode='change'` compares vs previous, `mode='default'` compares vs DEFVAL; `int_pin` is host GPIO for INT line; `mirror=True` sets IOCON.MIRROR so either port activates both INT pins |
| `set_default_value` | `port: int`, `mask: int` | — | Write DEFVALA/DEFVALB for default-comparison interrupt mode |
| `clear_interrupt` | `port: int` | int | Read and return INTCAPA/INTCAPB (clears INT for that port); also returns INTFA/INTFB in the upper byte if the implementation chooses to pack both |
| `read_interrupt_flags` | `port: int` | int | Read INTFA/INTFB without clearing interrupt |
| `stop_interrupt` | `port: int` | — | Disable interrupt for port; clear GPINTENA/GPINTENB |

**Pin API additions — Full**

| Operation | MicroPython | CircuitPython | C++ | Node.js | Rust |
|-----------|-------------|---------------|-----|---------|------|
| Set pull-up | `pin.init(pull=Pin.PULL_UP)` | `pin.pull = Pull.UP` | `pin.mode(INPUT_PULLUP)` | *(via driver `configure_pullup`)* | *(via driver)* |
| Attach interrupt | `pin.irq(handler, Pin.IRQ_RISING)` | *(via driver)* | `pin.attachInterrupt(handler, RISING)` | `pin.watch(handler)` | *(via driver callback)* |
| Detach interrupt | `pin.irq(None)` | *(via driver)* | `pin.detachInterrupt()` | `pin.unwatch(handler)` | *(via driver)* |

**Additional configuration options:** per-pin pull-up (GPPU), per-port polarity inversion (IPOL), IOCON.MIRROR to route both INT pins together, INTCON to select pin-change vs default-compare interrupt mode.

## GPIO Interface

This section defines the per-platform contracts for Pin objects. See `AGENTS.md` for implementation patterns.

### Python — MicroPython

Pin objects implement a compatible subset of `machine.Pin`. Constants (`IN`, `OUT`, `IRQ_FALLING`, `IRQ_RISING`, `PULL_UP`) are defined as class attributes on `Mcp23017Minimal`.

Minimal interface: `init(mode)`, `value([x])`, `on()`, `off()`, `toggle()`  
Full adds: `init(mode, pull)`, `irq(handler, trigger)` — trigger is `IRQ_FALLING`, `IRQ_RISING`, or `IRQ_FALLING | IRQ_RISING`.

### Python — CircuitPython

Pin objects implement `digitalio.DigitalInOut`.

Minimal interface: `direction` (read/write), `value` (read/write), `switch_to_input()`, `switch_to_output(value=False)`  
Full adds: `pull` (read/write, `Pull.UP` or `None`; `Pull.DOWN` raises `AttributeError`), `drive_mode` (push-pull only; setting `DriveMode.OPEN_DRAIN` raises `AttributeError`).

### Python — Linux

Same interface as MicroPython (`init`, `value`, `on`, `off`, `toggle`). Interrupt delivery in Full uses a background polling thread that calls `clear_interrupt(port)` every 5 ms; when a change is detected the callback is invoked. If a `gpiod` line for the INT pin is passed to `configure_interrupt`, use edge detection instead of polling.

### C++

`IOExpanderPin` is a nested class in `Mcp23017Minimal`. Arduino GPIO constants are reused (`INPUT`, `OUTPUT`, `INPUT_PULLUP`, `HIGH`, `LOW`).

Minimal interface:
```cpp
class IOExpanderPin {
public:
    void mode(uint8_t m);     // INPUT, OUTPUT
    void write(uint8_t v);    // HIGH, LOW
    uint8_t read();
    void high()   { write(HIGH); }
    void low()    { write(LOW);  }
    void toggle() { write(read() ^ 1); }
};
```

Full adds: `INPUT_PULLUP` support in `mode()`; `attachInterrupt(void (*handler)(uint8_t changed_mask), uint8_t mode)` and `detachInterrupt()` in `Mcp23017Full`. `mode` accepts `RISING`, `FALLING`, or `CHANGE`. On Linux, deliver via a `poll()` thread on the INT GPIO sysfs fd. On Zephyr, deliver via `gpio_add_callback()`.

### Node.js

Pin objects implement the [`onoff`](https://www.npmjs.com/package/onoff) `Gpio` interface subset so they are drop-in replacements.

Minimal interface: `readSync()`, `writeSync(value)`, `read(callback)`, `write(value, callback)`, `direction` property, `unexport()`  
Full adds: `watch(callback)`, `unwatch(callback)`, `setActiveLow(invert)` — interrupt delivery via polling the INT GPIO using `epoll` (Linux sysfs) or a 5 ms poll interval when no INT line is available.

### Rust

Output pins implement `embedded_hal::digital::OutputPin`; input pins implement `embedded_hal::digital::InputPin`. The driver stores its I²C bus in a `core::cell::RefCell<I2C>`. Pin objects hold a shared reference `&'_ RefCell<...>` and borrow it only during each operation. Multiple pin objects may coexist, but simultaneous access from different execution contexts is not safe — document this in the driver's module-level doc comment.

```rust
// Minimal — output pin
impl<I2C: I2c> embedded_hal::digital::OutputPin for OutputPin<'_, I2C> {
    type Error = I2C::Error;
    fn set_high(&mut self) -> Result<(), Self::Error> { ... }
    fn set_low(&mut self) -> Result<(), Self::Error> { ... }
}

// Minimal — input pin
impl<I2C: I2c> embedded_hal::digital::InputPin for InputPin<'_, I2C> {
    type Error = I2C::Error;
    fn is_high(&self) -> Result<bool, Self::Error> { ... }
    fn is_low(&self) -> Result<bool, Self::Error> { ... }
}
```

Full adds `embedded_hal::digital::StatefulOutputPin`: the OLAT register is read-capable so output state can be reported without a shadow register, but maintaining a shadow avoids an extra I²C transaction.

Re-export from category `mod.rs`:
```rust
pub use mcp23017::{Mcp23017Minimal, Mcp23017Full, ExPin};
```

### JVM

The `Pin` class is a `public static class` nested inside `Mcp23017Minimal`. Interrupt delivery is always via a background polling thread using a host GPIO line or 5 ms polling when no INT line is available.

Minimal interface:
```java
chip.pin(n)         // → Mcp23017Minimal.Pin
pin.setInput()      // set IODIR bit = 1
pin.setOutput()     // set IODIR bit = 0
pin.setHigh()       // set OLAT bit = 1
pin.setLow()        // set OLAT bit = 0
pin.read()          // → boolean (GPIO register, actual pin level)
pin.toggle()        // invert OLAT bit via shadow
```

Full adds:
```java
pin.setPullUp(true/false)              // set GPPU bit
chip.configureInterrupt(int port, IntConsumer callback)   // enable INT for port
chip.clearInterrupt(int port)          // → int captured bitmask; clears INT
chip.stopInterrupt(int port)           // disable INT, stop polling thread
```

In Kotlin, `configureInterrupt` accepts a `(Int) -> Unit` lambda. In Groovy, it accepts a `Closure`.

## Data Conversion

```
pin n  →  port n/8, bit n%8

PORTA = port 0 (pins  0–7,  GPA0–GPA7)
PORTB = port 1 (pins 8–15, GPB0–GPB7)

Read pin n:
    byte = read_port(n / 8)          # reads GPIOA or GPIOB
    value = (byte >> (n % 8)) & 1

Write pin n (using shadow OLAT):
    port = n / 8
    bit  = n % 8
    if value == 1:  shadow[port] |=  (1 << bit)
    else:           shadow[port] &= ~(1 << bit)
    write 0x14+port, shadow[port]    # OLATA (0x14) or OLATB (0x15)
```

The shadow register for OLAT is maintained because writing individual pins requires read-modify-write. Reading OLAT is possible but costs an extra I²C transaction; the shadow eliminates that cost.

### Demo flow

Two inject nodes periodically write rotating patterns to PORTA outputs (GPA0–GPA6). A third inject reads PORTB (GPB0–GPB6, wired to buttons or links to PORTA outputs for loopback). ## Examples

### Demo

Wire seven LEDs (active-high, anode through 220 Ω to GPA0–GPA6, cathodes to GND) and seven push buttons (GPB0–GPB6 pulled to VCC via GPPU, driven low when pressed). The demo runs a Knight Rider scanner pattern on the LEDs and mirrors the button states so pressing button N lights LED N, overriding the scanner. The loop reads GPIOB every 100 ms, builds an output mask from the button state (inverted, since active-low), ORs it with the scanner position unless a button is pressed, then writes to OLATA. Prints the raw scanner position, button mask, and output mask each iteration.

## Timing Constraints

- No startup delay required after power-on reset or after RESET pin is released.
- RESET pulse width must be at least 1 µs.
- IOC event to INT active: max 600 ns.
- GP input change to register valid: max 450 ns.
- Interrupt pin disable time after clearing: max 600 ns.
- Glitch filter on GP pins: max 150 ns suppression.
- At 400 kHz, a single-register read (address phase + read) takes approximately 30–40 µs; reading all 16 GPIO bits in one sequential read (GPIOA→GPIOB) takes approximately 50 µs.

## Implementation Notes

- **GPA7 and GPB7 are output-only.** The POR value of IODIRA/IODIRB is `0xFF` (all inputs), but GPA7/GPB7 are physically output-only on the MCP23017. The driver must write IODIRA = IODIRB = `0x7F` at init before any other operation. These pins may not be used as inputs and should not be returned by `pin()` with direction INPUT.
- **Never change IOCON.BANK.** Changing the BANK bit shifts all register addresses mid-transaction, which corrupts the address pointer. Keep BANK = 0 (reset default) always. The spec uses BANK=0 addresses throughout.
- **Use OLAT for output writes.** Writing to GPIOA/GPIOB is equivalent to writing OLATA/OLATB, but reading GPIOA/GPIOB returns the actual pin level, not the latch value. For read-modify-write on outputs, use the shadow register or read OLATA/OLATB directly.
- **Interrupt cleared by reading GPIO or INTCAP.** Reading GPIOA clears INTA; reading GPIOB clears INTB. Reading INTCAPA clears INTA; reading INTCAPB clears INTB. The interrupt must be cleared before the next interrupt event can be captured. Do not mix GPIO and INTCAP reads for clearing — the INTCAP value can be overwritten if a second interrupt occurs before INTCAP is read.
- **IOCON.MIRROR interaction.** When MIRROR=1, both INTA and INTB fire for any port change, but clearing requires reading BOTH GPIOA (or INTCAPA) and GPIOB (or INTCAPB); reading only one port leaves the other INT asserted.
- **Sequential read for efficiency.** With SEQOP=0 (default), issuing a sequential read starting at GPIOA (0x12) reads GPIOA then GPIOB in one transaction, giving the full 16-bit state atomically.
- **Pull-ups apply only to inputs.** Enabling GPPU on a pin configured as output has no effect on electrical behavior, but the driver should prevent this in Full to avoid user confusion.

## Sigrok Decoder

Decoder id `mcp23017`, input `['i2c']`. Matches I²C addresses `0x20`–`0x27`. Annotates every register write and read with the register name and value, decoded into symbolic field names (e.g. IODIR→`IN=GPA6,GPA5…`, GPPU→`PU=GPA0,GPA1`, IOCON→`BANK=0 MIRROR=1 ODR=0`). Distinguishes multi-byte sequential transactions and annotates each byte with its register name using the BANK=0 address table. Emits warning annotations for writes to read-only registers (INTFA, INTFB, INTCAPA, INTCAPB) and for attempts to set IODIRA/IODIRB bit 7 to 1 (input mode on output-only pin). Optionally decodes GPIOA/GPIOB reads as 16-bit combined port value when both bytes appear in the same transaction.
