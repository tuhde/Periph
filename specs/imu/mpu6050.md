# Chip Spec: MPU-6050 / MPU-6000

**Manufacturer:** InvenSense  
**Datasheet:** `datasheets/imu/mpu6050.pdf`  
**Category:** `imu`  
**Transports:** I²C, SPI

## Overview

The MPU-6050 is a 6-axis MotionTracking device combining a 3-axis MEMS gyroscope and a 3-axis MEMS accelerometer in a 4×4×0.9 mm QFN package. It also includes a Digital Motion Processor (DMP), a 1024-byte FIFO, and a digital temperature sensor. An auxiliary I²C master bus (AUX_DA / AUX_CL) allows connection of external sensors (e.g. a magnetometer) so the DMP can perform 9-axis fusion. Two variants exist: the **MPU-6050** (I²C only; VLOGIC pin accepts 1.71 V–VDD for 1.8 V or 3.3 V logic) and the **MPU-6000** (I²C + SPI; VLOGIC replaced by /CS). Primary use case: inertial measurement for stabilisation, dead-reckoning, and 6-DOF sensor fusion.

## Transport Configuration

### I²C

- **Address:** `0x68` (AD0 = GND) — `0x69` (AD0 = VCC)
- **Max clock:** 400 kHz (Fast mode)

### SPI (MPU-6000 only)

- **Mode:** CPOL=1 CPHA=1 (Mode 3)
- **Max clock:** 1 MHz for all registers (read/write); 20 MHz for sensor and interrupt registers (read only)
- **Bit order:** MSB first
- **CS active:** low
- **First byte:** `[R/W̄][A6..A0]` — R/W̄=1 for read, 0 for write; supports burst read (auto-increment)

> The MPU-6050 does not have a /CS pin; it is I²C only. The SPI interface is exclusive to the MPU-6000.

### Bit Fields

#### `GYRO_CONFIG` (`0x1B`)

| Bits | Name      | Description |
|------|-----------|-------------|
| 7    | XG_ST     | X gyro self-test enable |
| 6    | YG_ST     | Y gyro self-test enable |
| 5    | ZG_ST     | Z gyro self-test enable |
| 4:3  | FS_SEL    | Full-scale: 00=±250, 01=±500, 10=±1000, 11=±2000 dps |
| 2:0  | —         | Reserved |

#### `ACCEL_CONFIG` (`0x1C`)

| Bits | Name      | Description |
|------|-----------|-------------|
| 7    | XA_ST     | X accel self-test enable |
| 6    | YA_ST     | Y accel self-test enable |
| 5    | ZA_ST     | Z accel self-test enable |
| 4:3  | AFS_SEL   | Full-scale: 00=±2g, 01=±4g, 10=±8g, 11=±16g |
| 2:0  | —         | Reserved |

#### `CONFIG` (`0x1A`)

| Bits | Name      | Description |
|------|-----------|-------------|
| 5:3  | EXT_SYNC_SET | External frame synchronisation (FSYNC pin) |
| 2:0  | DLPF_CFG  | Digital low-pass filter setting; also selects gyro output rate (1 kHz when 1–6, 8 kHz when 0 or 7) |

#### `PWR_MGMT_1` (`0x6B`)

| Bits | Name         | Description |
|------|--------------|-------------|
| 7    | DEVICE_RESET | 1 = reset all registers to default |
| 6    | SLEEP        | 1 = low-power sleep mode (set on power-up by default) |
| 5    | CYCLE        | 1 = cycle between sleep and accel sample (LP mode) |
| 3    | TEMP_DIS     | 1 = disable internal temperature sensor |
| 2:0  | CLKSEL       | 0=internal 8 MHz, 1=PLL gyro X, 2=PLL gyro Y, 3=PLL gyro Z, 4=PLL ext 32.768 kHz, 5=PLL ext 19.2 MHz, 7=stop clock |

#### `INT_PIN_CFG` (`0x37`)

| Bits | Name            | Description |
|------|-----------------|-------------|
| 1    | I2C_BYPASS_EN   | 1 = host can access auxiliary I²C bus directly |

## Initialization Sequence

1. Write PWR_MGMT_1 (`0x6B`) = `0x80` (DEVICE_RESET)
2. Wait 100 ms (device power-on reset; PWR_MGMT_1 resets to `0x40` = SLEEP)
3. Write PWR_MGMT_1 = `0x01` (wake up; CLKSEL=1 = PLL with gyro X reference)
4. Verify WHO_AM_I (`0x75`) = `0x68`; raise error otherwise
5. Write GYRO_CONFIG (`0x1B`) = `0x00` (±250 dps)
6. Write ACCEL_CONFIG (`0x1C`) = `0x00` (±2 g)
7. Write CONFIG (`0x1A`) = `0x03` (DLPF 44 Hz gyro / 42 Hz accel, 1 kHz sample rate)
8. Write SMPLRT_DIV (`0x19`) = `0x04` (200 Hz output rate = 1 kHz / 5)
9. When using SPI (MPU-6000): write USER_CTRL (`0x6A`) = `0x10` (I2C_IF_DIS)
10. Wait 35 ms (gyroscope startup from sleep, Table 8)

## Implementation Stages

Each chip is implemented in two stages. The Full class extends Minimal — it inherits everything and adds the rest.

### Minimal

Goal: expose 3-axis acceleration and 3-axis angular rate with no configuration required beyond the transport.

| Operation | Parameters | Returns                 | Notes |
|-----------|------------|-------------------------|-------|
| `init`    | transport  | —                       | Reset, WHO_AM_I check, enable all sensors at defaults; 35 ms startup wait |
| `accel`   | —          | `(float, float, float)` | Linear acceleration (x, y, z) in m/s² |
| `gyro`    | —          | `(float, float, float)` | Angular rate (x, y, z) in rad/s |

**Sensible defaults baked into Minimal:**
- Gyroscope full-scale: ±250 dps (FS_SEL=0)
- Accelerometer full-scale: ±2 g (AFS_SEL=0)
- DLPF: 44 Hz bandwidth (CONFIG DLPF_CFG=3, 1 kHz gyro rate)
- Sample rate: 200 Hz (SMPLRT_DIV=4)
- Clock: PLL with gyro X reference (CLKSEL=1)
- All six axes enabled
- SPI (MPU-6000) only: I2C_IF_DIS set to prevent accidental I²C re-enable

### Full

Goal: expose complete chip functionality. Extends Minimal.

| Operation               | Parameters                                          | Returns                 | Notes |
|-------------------------|-----------------------------------------------------|-------------------------|-------|
| *(inherits Minimal)*    |                                                     |                         |       |
| `configure_gyro`        | `full_scale=0`                                      | —                       | 0=±250, 1=±500, 2=±1000, 3=±2000 dps |
| `configure_accel`       | `full_scale=0`                                      | —                       | 0=±2g, 1=±4g, 2=±8g, 3=±16g |
| `configure_dlpf`        | `dlpf=3`                                            | —                       | 0=260/256 Hz, 1=184/188 Hz, 2=94/98 Hz, 3=44/42 Hz, 4=21/20 Hz, 5=10/10 Hz, 6=5/5 Hz (gyro/accel BW) |
| `configure_sample_rate` | `divider=4`                                         | —                       | SMPLRT_DIV; output rate = 1 kHz / (1 + divider) when DLPF active |
| `temperature`           | —                                                   | `float`                 | Die temperature in °C |
| `accel_raw`             | —                                                   | `(int, int, int)`       | Raw 16-bit signed accel values |
| `gyro_raw`              | —                                                   | `(int, int, int)`       | Raw 16-bit signed gyro values |
| `data_ready`            | —                                                   | `bool`                  | True when DATA_RDY_INT set in INT_STATUS |
| `set_sleep`             | `sleep=True`                                        | —                       | Set/clear SLEEP bit in PWR_MGMT_1 |
| `set_standby`           | `xa=False, ya=False, za=False, xg=False, yg=False, zg=False` | — | Put individual axes into standby |
| `fifo_count`            | —                                                   | `int`                   | Number of bytes available in FIFO |
| `read_fifo`             | —                                                   | `bytes`                 | Read all available FIFO data |
| `enable_fifo`           | `gyro=True, accel=True, temp=False`                 | —                       | Configure and enable FIFO sources |
| `reset_fifo`            | —                                                   | —                       | Set FIFO_RST in USER_CTRL |

**Additional configuration options:**
- Gyroscope full-scale: ±250 / ±500 / ±1000 / ±2000 dps
- Accelerometer full-scale: ±2 / ±4 / ±8 / ±16 g
- Digital low-pass filter: 7 settings (0=~260 Hz bypass, 1–6 progressively narrower to 5 Hz)
- Sample rate divider: 0–255 (effective only when DLPF active)
- Per-axis standby control
- FIFO: 1024 bytes; configurable sources

## Data Conversion

### Accelerometer

Sensitivity (LSB/g) depends on AFS_SEL:

| AFS_SEL | Full-scale | Sensitivity  |
|---------|------------|--------------|
| `00`    | ±2 g       | 16384 LSB/g  |
| `01`    | ±4 g       | 8192 LSB/g   |
| `10`    | ±8 g       | 4096 LSB/g   |
| `11`    | ±16 g      | 2048 LSB/g   |

```
accel_g    = raw_signed_16 / sensitivity
accel_m_s2 = accel_g * 9.80665
```

Data is big-endian (high byte at lower address). Combine as:
```
raw = (ACCEL_XOUT_H << 8) | ACCEL_XOUT_L
if raw >= 0x8000: raw -= 0x10000
```

### Gyroscope

Sensitivity (LSB/(°/s)) depends on FS_SEL:

| FS_SEL | Full-scale  | Sensitivity      |
|--------|-------------|------------------|
| `00`   | ±250 dps    | 131.0 LSB/(°/s)  |
| `01`   | ±500 dps    | 65.5 LSB/(°/s)   |
| `10`   | ±1000 dps   | 32.8 LSB/(°/s)   |
| `11`   | ±2000 dps   | 16.4 LSB/(°/s)   |

```
gyro_dps   = raw_signed_16 / sensitivity
gyro_rad_s = gyro_dps * π / 180.0
```

### Temperature

```
temperature_C = raw_signed_16 / 340.0 + 36.53
```

Sensitivity 340 LSB/°C; reference point −521 LSB at 35 °C (from datasheet Section 4.6.2).

## Timing Constraints

- **Power-on to register access:** 30 ms maximum
- **After DEVICE_RESET:** wait 100 ms before writing configuration
- **Gyroscope startup from sleep:** 35 ms (Table 8 in datasheet)
- **Accelerometer startup from sleep:** 20 ms
- **DLPF_CFG=0 or 7:** gyro output rate = 8 kHz; all other settings = 1 kHz
- **SPI (MPU-6000) config registers:** 1 MHz max; sensor/interrupt registers: 20 MHz max

## Implementation Notes

- **WHO_AM_I:** Register 0x75; bits 6:1 always `0x68`. Raise a descriptive error for any other value.
- **Power-on SLEEP:** PWR_MGMT_1 resets to `0x40` (SLEEP=1). The driver must clear SLEEP before any sensor reads. Step 3 of init handles this by writing `0x01`.
- **Clock source:** CLKSEL=1 (PLL with gyro X) is recommended over the internal 8 MHz oscillator for better gyro accuracy. Internal oscillator is acceptable for accel-only use.
- **I2C_IF_DIS (SPI / MPU-6000 only):** When initialising via SPI, set bit 4 of USER_CTRL immediately after reset to disable the I²C interface. Without this, spurious I²C transactions can switch the chip into I²C mode, disabling SPI.
- **MPU-6050 vs MPU-6000:** Both devices share an identical register map and sensor core. The only functional differences are: (1) MPU-6000 adds SPI with /CS pin; (2) MPU-6050 replaces /CS with a VLOGIC pin that sets the I²C logic level. A single driver serves both variants; the SPI path is used only when the transport is SPI (implying MPU-6000 hardware).
- **FIFO size:** 1024 bytes (twice the 512 bytes of the MPU-9250). When accel + gyro are both enabled (12 bytes/sample at 200 Hz), the FIFO holds ~42 samples before overflow.
- **FIFO ordering:** When multiple sources are enabled in FIFO_EN, data is packed in this order: ACCEL → TEMP → GYRO → EXT_SENS (if configured). Each field is 2 bytes, big-endian.
- **Burst sensor read:** Read all 14 sensor bytes (accel + temp + gyro: 0x3B–0x48) in a single burst transaction for consistency.
- **Auxiliary I²C bus:** AUX_DA / AUX_CL pins provide a secondary I²C master for external sensors (e.g. a magnetometer). The Full driver does not implement the auxiliary I²C master; document its existence in the API but leave it to the application.

## Sigrok Decoder

The `mpu6050` decoder matches I²C addresses `0x68` and `0x69`. For SPI, it matches any CS channel. It annotates register writes with decoded field values (e.g., "GYRO_CONFIG: FS=±250dps DLPF_active"), burst reads of 0x3B–0x48 as computed accel (m/s²), temperature (°C), and gyro (rad/s) on all three axes, and PWR_MGMT_1 writes annotating the power state change. WHO_AM_I reads annotate the ID value.
