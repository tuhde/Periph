# Chip Spec: MPU-9250

**Manufacturer:** InvenSense  
**Datasheet:** `datasheets/imu/mpu9250.pdf`  
**Category:** `imu`  
**Transports:** I²C, SPI

## Overview

The MPU-9250 is a 9-axis MotionTracking device in a 3×3×1 mm QFN package. It integrates a 3-axis MEMS gyroscope, a 3-axis MEMS accelerometer, and an AK8963 3-axis magnetometer (a separate die from Asahi Kasei Microdevices) along with a Digital Motion Processor (DMP), a 512-byte FIFO, and a digital temperature sensor. All three sensors share a single I²C or SPI interface to the host, with the magnetometer optionally accessible via I²C bypass or through the MPU's internal I²C master. Primary use case: orientation, stabilisation, dead-reckoning, and 9-DOF sensor fusion.

## Transport Configuration

### I²C
- **Address:** `0x68` (AD0 = GND) — `0x69` (AD0 = VCC)
- **Max clock:** 400 kHz (Fast mode)

### SPI
- **Mode:** CPOL=1 CPHA=1 (Mode 3)
- **Max clock:** 1 MHz for all registers (read/write); 20 MHz for sensor and interrupt registers (read only)
- **Bit order:** MSB first
- **CS active:** low
- **First byte:** `[R/W̄][A6..A0]` — R/W̄=1 for read, 0 for write; supports burst read (auto-increment)

### Bit Fields

#### `GYRO_CONFIG` (`0x1B`)

| Bits | Name         | Description |
|------|--------------|-------------|
| 7    | XGYRO_CTEN   | X gyro self-test |
| 6    | YGYRO_CTEN   | Y gyro self-test |
| 5    | ZGYRO_CTEN   | Z gyro self-test |
| 4:3  | GYRO_FS_SEL  | Full-scale: 00=±250, 01=±500, 10=±1000, 11=±2000 dps |
| 1:0  | FCHOICE_B    | Bypass DLPF: 00=use DLPF_CFG, 01/10/11=bypass (8 kHz BW) |

#### `ACCEL_CONFIG` (`0x1C`)

| Bits | Name        | Description |
|------|-------------|-------------|
| 7    | ax_st_en    | X accel self-test |
| 6    | ay_st_en    | Y accel self-test |
| 5    | az_st_en    | Z accel self-test |
| 4:3  | ACCEL_FS_SEL | Full-scale: 00=±2g, 01=±4g, 10=±8g, 11=±16g |

#### `PWR_MGMT_1` (`0x6B`)

| Bits | Name         | Description |
|------|--------------|-------------|
| 7    | DEVICE_RESET | 1 = reset all registers to default |
| 6    | SLEEP        | 1 = low-power sleep mode |
| 5    | CYCLE        | 1 = cycle between sleep and accel sample (low power) |
| 4    | GYRO_STANDBY | 1 = gyro drive on but PLL off |
| 3    | PD_PTAT      | 1 = power down internal temperature sensor |
| 2:0  | CLKSEL       | 0=internal 20 MHz, 1=auto (PLL if ready, else 20 MHz), 6=internal 20 MHz, 7=stop clock |

#### `INT_PIN_CFG` (`0x37`)

| Bits | Name       | Description |
|------|------------|-------------|
| 1    | BYPASS_EN  | 1 = enable I²C bypass so host can access AK8963 directly |

#### `AK8963 CNTL1` (`0x0A`)

| Bits | Name     | Description |
|------|----------|-------------|
| 4    | BIT      | 0 = 14-bit output, 1 = 16-bit output |
| 3:0  | MODE     | 0000=power-down, 0001=single, 0010=cont mode 1 (8 Hz), 0110=cont mode 2 (100 Hz), 1111=fuse ROM access |

## Initialization Sequence

### MPU-9250

1. Write PWR_MGMT_1 (`0x6B`) = `0x80` (DEVICE_RESET)
2. Wait 100 ms
3. Write PWR_MGMT_1 = `0x01` (wake up; CLKSEL=1 = auto-select PLL)
4. Verify WHO_AM_I (`0x75`) = `0x71`; raise error otherwise
5. Write GYRO_CONFIG (`0x1B`) = `0x00` (±250 dps, DLPF active)
6. Write ACCEL_CONFIG (`0x1C`) = `0x00` (±2 g)
7. Write ACCEL_CONFIG2 (`0x1D`) = `0x00` (accel DLPF 460 Hz)
8. Write CONFIG (`0x1A`) = `0x03` (DLPF 41 Hz gyro / 42 Hz accel, 1 kHz sample rate)
9. Write SMPLRT_DIV (`0x19`) = `0x04` (200 Hz output rate = 1 kHz / 5)
10. When using SPI: write USER_CTRL (`0x6A`) = `0x10` (I2C_IF_DIS — disable I²C to prevent spurious mode switch)
11. Wait 35 ms (gyroscope startup from Table 1)

### AK8963 Magnetometer (I²C bypass mode)

12. Write INT_PIN_CFG (`0x37`) = `0x22` (BYPASS_EN=1, INT_ANYRD_2CLEAR=1)
13. Write AK8963 CNTL1 (`0x0A`) = `0x00` (power down)
14. Wait 10 ms
15. Write AK8963 CNTL1 = `0x0F` (fuse ROM access mode)
16. Wait 10 ms
17. Read ASAX (`0x10`), ASAY (`0x11`), ASAZ (`0x12`) — store as calibration
18. Write AK8963 CNTL1 = `0x00` (power down)
19. Wait 10 ms
20. Write AK8963 CNTL1 = `0x16` (16-bit, continuous measurement mode 2 at 100 Hz)

## Implementation Stages

Each chip is implemented in two stages. The Full class extends Minimal — it inherits everything and adds the rest.

### Minimal

Goal: expose 3-axis acceleration and 3-axis angular rate with no configuration required beyond the transport. No magnetometer in Minimal — it requires a non-trivial secondary initialization path.

| Operation   | Parameters  | Returns                  | Notes |
|-------------|-------------|--------------------------|-------|
| `init`      | transport   | —                        | Reset, WHO_AM_I check, enable all sensors at defaults; 35 ms startup wait |
| `accel`     | —           | `(float, float, float)`  | Linear acceleration (x, y, z) in m/s² |
| `gyro`      | —           | `(float, float, float)`  | Angular rate (x, y, z) in rad/s |

**Sensible defaults baked into Minimal:**
- Gyroscope full-scale: ±250 dps (GYRO_FS_SEL=0)
- Accelerometer full-scale: ±2 g (ACCEL_FS_SEL=0)
- Gyroscope DLPF: 41 Hz bandwidth (CONFIG DLPF_CFG=3)
- Accelerometer DLPF: 42 Hz bandwidth (ACCEL_CONFIG2 A_DLPFCFG=3)
- Sample rate: 200 Hz (SMPLRT_DIV=4)
- Clock: auto PLL (CLKSEL=1)
- All six axes enabled
- SPI only: I2C_IF_DIS set to prevent accidental I²C re-enable

### Full

Goal: expose complete chip functionality. Extends Minimal.

| Operation            | Parameters                                        | Returns                         | Notes |
|----------------------|---------------------------------------------------|---------------------------------|-------|
| *(inherits Minimal)* | | | |
| `configure_gyro`     | `full_scale=0`                                    | —                               | 0=±250, 1=±500, 2=±1000, 3=±2000 dps |
| `configure_accel`    | `full_scale=0`                                    | —                               | 0=±2g, 1=±4g, 2=±8g, 3=±16g |
| `configure_dlpf`     | `gyro_dlpf=3`, `accel_dlpf=3`                    | —                               | 0=260/256 Hz (gyro/accel), 1=184/188 Hz, 2=92/98 Hz, 3=41/42 Hz, 4=20/20 Hz, 5=10/10 Hz, 6=5/5 Hz |
| `configure_sample_rate` | `divider=4`                                   | —                               | SMPLRT_DIV; output rate = 1 kHz / (1 + divider) when DLPF active |
| `temperature`        | —                                                 | `float`                         | Die temperature in °C |
| `enable_mag`         | `bits=16`, `mode=6`                               | —                               | Initialize AK8963; bits: 14 or 16; mode: 1=single, 2=8 Hz, 6=100 Hz |
| `mag`                | —                                                 | `(float, float, float)`         | Magnetic field (x, y, z) in µT; raises if mag not enabled |
| `accel_raw`          | —                                                 | `(int, int, int)`               | Raw 16-bit signed accel |
| `gyro_raw`           | —                                                 | `(int, int, int)`               | Raw 16-bit signed gyro |
| `mag_raw`            | —                                                 | `(int, int, int)`               | Raw magnetometer values |
| `data_ready`         | —                                                 | `bool`                          | True when RAW_DATA_RDY_INT set in INT_STATUS |
| `set_sleep`          | `sleep=True`                                      | —                               | Set/clear SLEEP bit in PWR_MGMT_1 |
| `fifo_count`         | —                                                 | `int`                           | Number of bytes available in FIFO |
| `read_fifo`          | —                                                 | `bytes`                         | Read all available FIFO data |
| `enable_fifo`        | `gyro=True`, `accel=True`, `temp=False`           | —                               | Configure and enable FIFO sources |
| `reset_fifo`         | —                                                 | —                               | Set FIFO_RST in USER_CTRL |

**Additional configuration options:**
- Gyroscope full-scale: ±250 / ±500 / ±1000 / ±2000 dps
- Accelerometer full-scale: ±2 / ±4 / ±8 / ±16 g
- Digital low-pass filter: 7 settings (0=bypass ~260 Hz, 1–6 progressively narrower down to 5 Hz)
- Sample rate divider: 0–255 (effective only when DLPF active)
- Magnetometer: 14-bit or 16-bit; 8 Hz or 100 Hz continuous; single-shot
- FIFO: 512 bytes; configurable sources

## Data Conversion

### Accelerometer

Sensitivity (LSB/g) depends on ACCEL_FS_SEL:

| AFS_SEL | Full-scale | Sensitivity  |
|---------|------------|--------------|
| `00`    | ±2 g       | 16384 LSB/g  |
| `01`    | ±4 g       | 8192 LSB/g   |
| `10`    | ±8 g       | 4096 LSB/g   |
| `11`    | ±16 g      | 2048 LSB/g   |

```
accel_g     = raw_signed_16 / sensitivity
accel_m_s2  = accel_g * 9.80665
```

Data is big-endian (high byte at lower address). Combine as:
```
raw = (ACCEL_XOUT_H << 8) | ACCEL_XOUT_L
if raw >= 0x8000: raw -= 0x10000
```

### Gyroscope

Sensitivity (LSB/(°/s)) depends on GYRO_FS_SEL:

| FS_SEL | Full-scale  | Sensitivity      |
|--------|-------------|------------------|
| `00`   | ±250 dps    | 131.0 LSB/(°/s)  |
| `01`   | ±500 dps    | 65.5 LSB/(°/s)   |
| `10`   | ±1000 dps   | 32.8 LSB/(°/s)   |
| `11`   | ±2000 dps   | 16.4 LSB/(°/s)   |

```
gyro_dps   = raw_signed_16 / sensitivity
gyro_rad_s = gyro_dps * π / 180.0
```

### Temperature

```
temperature_C = raw_signed_16 / 333.87 + 21.0
```

Sensitivity 333.87 LSB/°C; 0 LSB at 21 °C (from Table 4 A.C. Characteristics).

### Magnetometer (AK8963)

Sensitivity depends on output bit resolution (BIT in CNTL1):

| BIT | Resolution | Sensitivity    |
|-----|------------|----------------|
| 0   | 14-bit     | 0.6 µT/LSB     |
| 1   | 16-bit     | 0.15 µT/LSB    |

Factory sensitivity adjustment (ASA values read from fuse ROM):

```
scale = (ASA - 128) / 256.0 + 1.0          # per axis (ASAX, ASAY, ASAZ)
mag_uT = raw_signed_16 * sensitivity * scale
```

Note: the magnetometer Z-axis is oriented differently from the accelerometer/gyroscope axes (see Figure 5 in the datasheet). The driver exposes raw-axis values without remapping; users performing sensor fusion must account for this.

### Demo

"Levelling indicator": Configure Full at ±4 g accel, ±500 dps gyro, magnetometer 100 Hz. In a loop, read accel and gyro; compute roll = atan2(ay, az) and pitch = atan2(-ax, sqrt(ay²+az²)) in degrees. ## Timing Constraints

- **Power-on to register access:** 11 ms typical, 100 ms maximum (use 100 ms in driver)
- **After DEVICE_RESET:** wait 100 ms before writing configuration
- **Gyroscope startup from sleep:** 35 ms
- **Accelerometer startup from sleep:** 20 ms; from cold start: 30 ms
- **Magnetometer data valid (cold start):** 7.3 ms
- **AK8963 power-down to mode change:** 10 ms minimum
- **SPI config registers:** 1 MHz max; sensor registers: 20 MHz max

## Implementation Notes

- **WHO_AM_I:** Register 0x75; value is always `0x71` for MPU-9250. Raise a descriptive error for any other value.
- **Magnetometer access — I²C transport:** Enable BYPASS_EN in INT_PIN_CFG (`0x37` bit 1). The host can then address the AK8963 directly at 0x0C. This is the recommended approach for I²C transport.
- **Magnetometer access — SPI transport:** SPI shares its pins with the I²C bypass bus; BYPASS_EN cannot be used. Instead, configure the MPU's internal I²C master (USER_CTRL I2C_MST_EN=1) and set up I2C_SLV0 to auto-read 7 bytes from AK8963 starting at register 0x03 (ST1). The results land in EXT_SENS_DATA (0x49–0x4F). Implementation complexity is significant; the Full driver must handle both transports.
- **I2C_IF_DIS (SPI only):** When initialising via SPI, set bit 4 of USER_CTRL immediately after reset to disable the I²C interface and prevent inadvertent I²C transactions from switching the chip into I²C mode.
- **ST2 must be read:** After each magnetometer data read, read AK8963 ST2 (`0x09`). Without this read, the AK8963 does not refresh its output registers. Include ST2 in the burst read (read 7 bytes 0x03–0x09).
- **Factory ASA calibration:** Must be read once during `enable_mag` in fuse ROM access mode (CNTL1=0x0F). Cache the three scale factors; do not re-read on every measurement.
- **SPI burst read:** For maximum throughput, read all 14 sensor bytes (accel + temp + gyro: 0x3B–0x48) in a single burst transaction.
- **Magnetometer axis orientation:** The magnetometer die has a different coordinate system from the accelerometer/gyroscope (Figure 4 vs Figure 5 in the datasheet). The driver does not remap axes; document this clearly in the API.
- **FIFO ordering:** When multiple sources are enabled in FIFO_EN, data is packed in this order: ACCEL → GYRO → TEMP → EXT_SENS (if configured). Each field is 2 bytes, big-endian.

## Sigrok Decoder

The `mpu9250` decoder matches I²C addresses `0x68` and `0x69` (primary device) and `0x0C` (AK8963 magnetometer, when bypass mode is active). For SPI, it matches any CS channel. It annotates register writes with decoded field values (e.g., "GYRO_CONFIG: FS=±250dps DLPF_active"), burst reads of 0x3B–0x48 as computed accel (m/s²), temperature (°C), and gyro (rad/s) on all three axes, and AK8963 burst reads 0x03–0x09 as magnetic field strength in µT with calibration applied. PWR_MGMT_1 writes annotate the power state change. WHO_AM_I reads annotate the ID value.
