# Chip Spec: L3G4200D

**Manufacturer:** STMicroelectronics  
**Datasheet:** `datasheets/gyroscope/L3G4200D.pdf`  
**Category:** `gyroscope`  
**Transports:** I²C, SPI

## Overview

The L3G4200D is a low-power three-axis MEMS angular rate sensor (gyroscope) that measures angular velocity on the X, Y, and Z axes simultaneously. Three selectable full-scale ranges (±250, ±500, ±2000 dps) and four output data rates (100–800 Hz) make it suitable for gaming controllers, GPS navigation, robotics, and industrial motion sensing. The chip includes an embedded 32-slot FIFO per axis, user-selectable low-pass and high-pass digital filters, an 8-bit temperature sensor, a programmable interrupt pin (INT1), and a data-ready/FIFO interrupt pin (DRDY/INT2). Both I²C and SPI are available simultaneously on shared pins; tie CS high to select I²C.

## Transport Configuration

### I²C

- **Address:** `0x68` (SA0 = GND, default) — `0x69` (SA0 = VDD)  
- **Max clock:** 400 kHz (fast mode)  
- **Multi-byte reads:** Set MSB of sub-address to 1 to auto-increment the register pointer.

### SPI

- **Mode:** CPOL=1 CPHA=1 (Mode 3)  
- **Max clock:** 10 MHz  
- **Bit order:** MSB first  
- **CS active:** low  
- **Wires:** 4-wire by default; 3-wire enabled by setting SIM=1 in CTRL_REG4.  
- **Frame format:** First byte = RW[7] | MS[6] | AD[5:0]; subsequent bytes are data. RW=1 for read, MS=1 for address auto-increment.

## Initialization Sequence

### Minimal

1. Write `CTRL_REG4 = 0x80`: BDU=1, ±250 dps, 4-wire SPI, LSB first  
2. Write `CTRL_REG1 = 0x0F`: normal mode, 100 Hz ODR, 12.5 Hz LPF2 cutoff, all axes enabled  
3. No startup delay is required; first valid data is available after one ODR period (10 ms at 100 Hz).

### Full (additional setup)

After Minimal init, any additional configuration calls (ODR, full scale, FIFO, HPF, interrupts) may be issued in any order.

## Implementation Stages

### Minimal

Goal: read all three angular-rate axes with zero configuration beyond the transport.

## Data Conversion

### Angular rate

```
# raw is signed 16-bit two's complement (little-endian: OUT_X_L, OUT_X_H)
sensitivity = {250: 8.75e-3, 500: 17.5e-3, 2000: 70.0e-3}  # dps per digit
rate_dps = raw * sensitivity[full_scale]
rate_rad_s = rate_dps * (math.pi / 180.0)
```

Sensitivity values (from datasheet Table 4):

| Full scale | Sensitivity |
|------------|-------------|
| ±250 dps   | 8.75 mdps/digit |
| ±500 dps   | 17.50 mdps/digit |
| ±2000 dps  | 70.0 mdps/digit |

### Temperature

```
# OUT_TEMP is a signed 8-bit value; scale is -1 °C per digit (relative only)
# There is no absolute offset calibration; use for drift compensation only
temp_delta = -raw  # raw increase → temperature decrease
```

### Interrupt threshold conversion

```
# 15-bit unsigned value; MSB of high-byte is always 0
threshold_raw = int(threshold_dps / sensitivity[full_scale])
INT1_THS_H = (threshold_raw >> 8) & 0x7F
INT1_THS_L = threshold_raw & 0xFF
```

## Examples

### Demo

Scenario: rotation detector — mount on a servo or hand-held device, print current rotation rate every 100 ms, and display an alert when angular rate on any axis exceeds 90 dps (≈ 1.57 rad/s). Useful to demonstrate motion detection and threshold sensitivity.

The demo uses the Full class:
1. Configure for 200 Hz ODR, ±500 dps full scale, FIFO stream mode (watermark=10)
2. Enable HPF (mode=0, cutoff for 0.1 Hz @200 Hz) to remove DC drift
3. In a loop: wait for FIFO watermark, drain the FIFO, compute mean of the burst, print rates and alert if any axis exceeds threshold

## Timing Constraints

- **Startup:** No explicit startup delay needed; the chip is ready immediately after CTRL_REG1 PD is set to 1. First valid conversion is available after 1 ODR period (10 ms at 100 Hz).
- **BDU:** When BDU=1, the driver must read both OUT_*_L and OUT_*_H (or use burst read) before the register is refreshed. A burst read of all 6 bytes is always safe.
- **FIFO read:** In FIFO/stream/stream-to-FIFO modes, each burst read of OUT_X_L through OUT_Z_H pops the oldest entry. Use FSS[4:0] from FIFO_SRC_REG to know how many samples are available.
- **INT1_SRC:** Reading INT1_SRC clears the interrupt-active (IA) bit and deasserts the INT1 pin (unless latching is disabled, in which case it deasserts automatically).

## Implementation Notes

- **I²C multi-byte reads:** Set bit 7 of the sub-address to 1 to enable auto-increment. E.g., to read 6 bytes starting at OUT_X_L: send sub-address `0xA8` (= 0x28 | 0x80).
- **SPI multi-byte reads:** Set MS (bit 6 of first byte) to 1 for address auto-increment.
- **CS/I²C selection:** The CS pin selects the interface. Tie CS to VDD_IO for I²C; drive CS low for SPI. Both cannot be active simultaneously.
- **FS=11 is also ±2000 dps:** The two encodings for 2000 dps are identical in behavior; always write `0b10` for ±2000 dps.
- **Temperature is relative:** OUT_TEMP has no absolute calibration. It is only useful for delta measurements (e.g., detecting warm-up drift). Do not convert to absolute Celsius.
- **FIFO burst wrap-around:** In FIFO mode, when the FIFO is full the oldest data is held and must be burst-read from the beginning; set FIFO mode back to bypass and then re-enable to restart collection.

## Sigrok Decoder

The sigrok decoder for the L3G4200D annotates I²C bus transactions (decoder id: `i2c`) at device addresses `0x68` and `0x69`. It decodes register addresses from sub-address bytes and annotates reads/writes of every named register. For angular-rate output registers (OUT_X_L/H, OUT_Y_L/H, OUT_Z_L/H), it assembles the 16-bit two's complement value and annotates the converted rate in dps at the configured full scale. STATUS_REG bits (ZYXDA, ZYXOR) and WHO_AM_I verification are also annotated. CTRL_REG1–5 writes are annotated with a human-readable summary of the configuration change (ODR, bandwidth, power mode, full scale).
