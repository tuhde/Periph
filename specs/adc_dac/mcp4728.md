# Chip Spec: MCP4728

**Manufacturer:** Microchip  
**Datasheet:** `datasheets/adc_dac/mcp4728.pdf`  
**Category:** adc_dac  
**Transports:** I²C  
**See also:** `specs/adc_dac/mcp4725.md` (single-channel variant)

## Overview

The MCP4728 is a quad-channel, 12-bit buffered voltage-output DAC with on-chip nonvolatile EEPROM. It provides four independent analog outputs (channels A–D), each with its own configurable voltage reference (internal 2.048 V or external V_DD), gain (×1 or ×2), and power-down mode. EEPROM stores both I²C address bits and per-channel configuration, so outputs resume from saved values on every power-up without host intervention. Typical applications are multi-channel calibration, four-point sensor bias, and closed-loop servo control where independent voltage set-points are required.

## Transport Configuration

### I²C

- **Address:** `0x60` (factory default, A2:A1:A0 = 000) through `0x67` (A2:A1:A0 = 111)  
  Device code is fixed at `1100`; all three address bits (A2, A1, A0) are user-programmable into EEPROM (factory default `000`).
- **Max clock:** 3.4 MHz (high-speed); 400 kHz (fast); 100 kHz (standard)

## Register / Input Register Map

Each channel has an independent volatile DAC input register and a corresponding nonvolatile EEPROM copy. There is no register address byte; command type is encoded in the first data byte's upper bits.

### Input Register per Channel (volatile)

| Bit | 7 | 6 | 5:4 | 3:2 | 1 | — | — |
|-----|---|---|-----|-----|---|---|---|
| Name | RDY/BSY | POR | DAC1:DAC0 | `00` | A2 | A1 | A0 |
| (read response byte 1) | | | | | | | |

| Bit | 7 | 6 | 5:4 | 3 | 2:0 |
|-----|---|---|-----|---|-----|
| Name | V_REF | PD1 | PD0:Gx | D11 | D10–D8 |
| (bytes 2–3 of channel data) | | | | | |

### EEPROM per Channel

Same bit layout as the input register. Bits A2:A1:A0 in EEPROM are the stored I²C address.

### Configuration Bit Definitions

| Bit | Name | Description |
|-----|------|-------------|
| RDY/BSY | EEPROM status | 1 = not programming; 0 = EEPROM write in progress |
| POR | Power-on-reset | 1 = powered on with V_DD > V_POR |
| V_REF | Reference select | 0 = External (V_DD); 1 = Internal (2.048 V) |
| PD1:PD0 | Power-down | See table below |
| G_x | Gain | 0 = ×1; 1 = ×2 (only applicable when V_REF = Internal) |
| UDAC | DAC latch | 0 = update V_OUT immediately; 1 = hold (wait for LDAC) |

### Power-Down Bits (PD1:PD0)

| PD1 | PD0 | Mode |
|-----|-----|------|
| 0   | 0   | Normal operation |
| 0   | 1   | Power-down, 1 kΩ to GND |
| 1   | 0   | Power-down, 100 kΩ to GND |
| 1   | 1   | Power-down, 500 kΩ to GND |

### Channel Selection Bits (DAC1:DAC0)

| DAC1 | DAC0 | Channel |
|------|------|---------|
| 0    | 0    | A |
| 0    | 1    | B |
| 1    | 0    | C |
| 1    | 1    | D |

## Write Commands

### Fast Write (C2=0, C1=0) — DAC only, 8 data bytes total

Writes all four channels sequentially A→D; EEPROM unaffected. Per channel, 2 bytes:

| Byte | Bits 7:6 | Bits 5:4 | Bits 3:0 |
|------|----------|----------|----------|
| 1    | `00` (C2 C1, first channel only; `XX` for B–D) | PD1:PD0 | D11–D8 |
| 2    | D7–D0    | — | — |

V_OUT updates at the last ACK pulse of each channel (if LDAC=0) or all at once when LDAC is pulled low.

### Multi-Write (C2=0, C1=1, C0=0, W1=0, W0=0) — DAC only, one channel at a time

Writes any channel in any order by repeating 3-byte blocks; EEPROM unaffected:

| Byte | Bits 7:5 | Bits 4:3 | Bits 2:1 | Bit 0 |
|------|----------|----------|----------|-------|
| 1    | `010` (command) | `00` (W1 W0) | DAC1:DAC0 | UDAC |
| 2    | V_REF:PD1:PD0:Gx | — | D11–D8 | — |
| 3    | D7–D0    | — | — | — |

Byte 2 detail: `[V_REF PD1 PD0 Gx D11 D10 D9 D8]`  
Byte 3: `[D7 D6 D5 D4 D3 D2 D1 D0]`

### Sequential Write (C2=0, C1=1, C0=0, W1=1, W0=0) — DAC + EEPROM, starting channel to D

Writes from starting channel (set by DAC1:DAC0) through channel D; writes EEPROM at end. Per channel same byte 2–3 format as Multi-Write. Max 10 bytes per transaction (address + command + up to 4 channels × 2 data bytes).

### Single Write (C2=0, C1=1, C0=0, W1=1, W0=1) — DAC + EEPROM, one channel

Writes one channel and its EEPROM. Same byte format as Multi-Write. Byte 1: `[0 1 0 1 1 DAC1 DAC0 UDAC]`.

### Write V_REF (C2=1, C1=0, C0=0) — all channels, DAC only

2 bytes total after address:
- Byte 1: `[1 0 0 X V_REF_A V_REF_B V_REF_C V_REF_D]`

### Write Gain (C2=1, C1=1, C0=0) — all channels, DAC only

2 bytes total after address:
- Byte 1: `[1 1 0 X Gx_A Gx_B Gx_C Gx_D]`

### Write Power-Down (C2=1, C1=0, C0=1) — all channels, DAC only

3 bytes total after address:
- Byte 1: `[1 0 1 X PD1_A PD0_A PD1_B PD0_B]`
- Byte 2: `[PD1_C PD0_C PD1_D PD0_D X X X X]`

### General Call Commands (address 0x00)

| Second byte | Command | Effect |
|-------------|---------|--------|
| `0x06` | Reset | POR-like; loads EEPROM into all DAC input and output registers; V_OUT available immediately |
| `0x08` | Software Update | Updates all four V_OUT simultaneously (equivalent to LDAC pulse) |
| `0x09` | Wake-Up | Clears PD bits (PD=00) on all channels |

### Read Command (R/W = 1)

Returns 24 bytes sequentially: for each channel A–D, 3 bytes from the DAC input register followed by 3 bytes from EEPROM.

| Byte group | Byte 1 | Byte 2 | Byte 3 |
|-----------|--------|--------|--------|
| Input register (×4 channels) | `[RDY/BSY POR DAC1 DAC0 0 A2 A1 A0]` | `[V_REF PD1 PD0 Gx D11–D8]` | `[D7–D0]` |
| EEPROM (×4 channels) | Same format | Same format | Same format |

## Initialization Sequence

1. Apply power; device auto-loads EEPROM into all DAC input and output registers (no host action required).
2. *(Optional, if V_DD ramp rate < 1 V/ms)* Send General Call Reset: `[0x00, 0x06]`.
3. All four V_OUT are immediately valid.

## Implementation Stages

### Minimal

Goal: set any of the four channel outputs to a target fraction of V_DD, plus a convenience method to update all four channels at once. No configuration required beyond the transport.

| Operation | Parameters | Returns | Notes |
|-----------|------------|---------|-------|
| `init` | transport | — | Stores transport; no I²C traffic |
| `set_voltage` | `channel: int` (0–3), `fraction: float` (0.0–1.0) | — | Uses Multi-Write; UDAC=0 (immediate); V_REF=0 (V_DD), gain=1, PD=00; EEPROM unaffected |
| `set_raw` | `channel: int` (0–3), `code: int` (0–4095) | — | Same as `set_voltage` but takes raw 12-bit code; clamps to [0, 4095] |
| `set_all` | `fractions: list[float]` (4 elements) | — | Updates all four channels simultaneously via Fast Write; fractions[0]=A … fractions[3]=D; all V_REF=0, PD=00 |

**Sensible defaults:** V_REF=0 (external/V_DD), gain=1, PD=00 (normal) on every write. No EEPROM write on Minimal operations.

### Full

Goal: expose complete chip functionality. Extends Minimal.

| Operation | Parameters | Returns | Notes |
|-----------|------------|---------|-------|
| *(inherits Minimal)* | | | |
| `set_voltage_eeprom` | `channel: int` (0–3), `fraction: float`, `vref: int` (0/1), `gain: int` (1/2) | — | Single Write; updates DAC register + EEPROM |
| `set_raw_eeprom` | `channel: int` (0–3), `code: int`, `vref: int`, `gain: int` | — | Same but raw code |
| `set_all_eeprom` | `fractions: list[float]`, `vrefs: list[int]`, `gains: list[int]` | — | Sequential Write all 4 channels + EEPROM; waits for EEPROM ready |
| `set_vref` | `vref_a, vref_b, vref_c, vref_d: int` (0/1 each) | — | Write V_REF command; EEPROM unaffected |
| `set_gain` | `gain_a, gain_b, gain_c, gain_d: int` (0/1 each) | — | Write Gain command; EEPROM unaffected |
| `set_power_down` | `pd_a, pd_b, pd_c, pd_d: int` (0–3 each) | — | Write Power-Down command; EEPROM unaffected |
| `read` | — | `list[dict]` (4 elements) | Each element: `{"code": int, "vref": int, "gain": int, "power_down": int, "eeprom_code": int, "eeprom_vref": int, "eeprom_gain": int, "eeprom_power_down": int, "eeprom_ready": bool}` |
| `is_eeprom_ready` | — | `bool` | Read RDY/BSY bit from first byte of read response |
| `software_update` | — | — | General Call Software Update (0x00, 0x08); latches all V_OUT simultaneously |
| `wake_up` | — | — | General Call Wake-Up (0x00, 0x09); clears PD bits |
| `reset` | — | — | General Call Reset (0x00, 0x06); reloads EEPROM |

**Additional configuration options:**
- Per-channel voltage reference selection (internal 2.048 V or external V_DD)
- Per-channel gain selection (×1 or ×2, internal reference only)
- Per-channel power-down modes
- EEPROM persistence of all per-channel settings
- Simultaneous update of all V_OUT via General Call Software Update

## Data Conversion

```
# V_REF = V_DD (external reference):
V_OUT = V_DD × code / 4096

# V_REF = Internal (2.048 V), Gain = 1:
V_OUT = 2.048 × code / 4096       # range 0–2.048 V, LSB = 0.5 mV

# V_REF = Internal, Gain = 2:
V_OUT = 4.096 × code / 4096       # range 0–4.096 V, LSB = 1.0 mV

# Convert fraction → code (external reference):
code = round(fraction × 4095)     # 0.0 → 0, 1.0 → 4095

# Convert code → fraction (external reference):
fraction = code / 4095.0
```

For internal reference with gain:
```
code = round(target_voltage / (2.048 × gain) × 4095)
```

## Examples

### Demo

Set all four channels to different voltages representing a calibration use case: channel A = 0 V (zero reference), channel B = 1/3 of full scale, channel C = 2/3 of full scale, channel D = full scale. Print the target voltage (assuming 3.3 V supply) and actual applied code for each channel. Then sweep all channels together from 0 to full scale in 16 equal steps with a 50 ms pause between steps, using `set_all` for simultaneous update. This demonstrates both individual-channel targeting and synchronous multi-channel update.

## Timing Constraints

- **Output settling time:** 6 µs typical (within 0.5 LSB, code 400h → C00h).
- **Exit power-down time:** 4.5 µs typical (from last ACK pulse to V_OUT settling).
- **Reference switch time:** 26 µs (external → internal), 44 µs (internal → external).
- **EEPROM write time:** 25 ms typical, 50 ms maximum. RDY/BSY = 0 during write; any new write command is ignored until RDY/BSY returns high. Poll `is_eeprom_ready()` or wait 50 ms before issuing a second EEPROM write.
- **LDAC minimum low pulse:** 210 ns.

## Implementation Notes

- The MCP4728 has **no register address byte**. Command type is encoded in bits 7–5 of the first data byte (after the I²C address byte).
- **Factory EEPROM default:** All channels: V_REF=1 (internal), PD=00, Gain=0 (×1), code=0x000 → V_OUT=0 V. I²C address bits A2:A1:A0 = 000 → address `0x60`.
- **I²C address bits are user-writable** via the Write I²C Address Bits command (requires LDAC pin manipulation for security). Unlike the MCP4725, all three address bits are reprogrammable. The driver does not need to implement this command in Minimal or Full.
- **Fast Write** does not carry V_REF or Gx bits — it only sets PD and DAC code. V_REF and Gx retain their current register values. Use Multi-Write or Set V_REF/Gain commands to change reference or gain.
- **UDAC bit vs LDAC pin:** UDAC=0 in a write command causes the selected channel's V_OUT to update immediately after that channel's last ACK. LDAC pulled low updates all channels simultaneously, regardless of UDAC. For synchronous multi-channel update without an LDAC GPIO, use `software_update()` (General Call 0x08).
- **Gain=2 is ignored when V_REF=V_DD.** Always enforce gain=1 when using external reference in Full-class methods.
- The internal voltage reference circuit remains on as long as at least one channel selects it. Switching all channels to external reference turns it off; switching any one channel back switches it on after the reference switch delay (26 µs).
- **Sequential Write** is limited to 10 bytes per transaction (1 address + 1 command + up to 4 channels × 2 data bytes); any bytes beyond the 10th are ignored.
- For best analog output accuracy, decouple V_DD with 0.1 µF ceramic + 10 µF tantalum capacitors within 4 mm of the V_DD pin.
- `wake_up()`, `reset()`, and `is_eeprom_ready()` share identical General Call semantics with the MCP4725. Implementations may derive from a shared `_MCP47xxBase` containing `_general_call(cmd)`, `wake_up()`, `reset()`, and `is_eeprom_ready()`.
