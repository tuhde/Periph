# Chip Spec: PCF8591

**Manufacturer:** NXP  
**Datasheet:** `datasheets/adc_dac/PCF8591.pdf`  
**Category:** adc_dac  
**Transports:** I²C

## Overview

The PCF8591 is a single-supply 8-bit A/D and D/A converter combining four analog inputs, one analog output, and an I²C interface in one package. The four analog inputs can be configured as four single-ended channels, three differential channels, two differential channels, or a single-ended/differential mixed mode. The on-chip DAC shares the same successive-approximation reference path as the ADC. Three hardware address pins allow up to eight devices on the same bus. Typical applications are supply monitoring, reference setting, and analog control loops.

## Transport Configuration

### I²C

- **Address:** `0x48` (A0=A1=A2=0) through `0x4F` (A0=A1=A2=1)  
  Fixed upper bits `1001`; address bits 2:0 are set by the hardware A0, A1, A2 pins.
- **Max clock:** 100 kHz (standard mode only)

## Protocol

The PCF8591 has no register address byte. The control byte sent immediately after the I²C address byte configures the chip; an optional following byte sets the DAC output value. Reading returns ADC sample bytes.

### Write transaction (configure and/or set DAC output)

```
START → [addr+W] → [CONTROL_BYTE] → [DAC_VALUE (optional)] → ... → STOP
```

### Read transaction (ADC)

```
START → [addr+R] → [BYTE_0 (stale)] → [BYTE_1 (fresh)] → ... → NACK → STOP
```

A conversion cycle is triggered at the trailing edge of the ACK following the address byte. The first byte clocked out is therefore the result of the **previous** conversion cycle. Discard it; BYTE_1 onward are fresh. After Power-On Reset (POR) the first-read stale byte is `0x80`.

### Control Byte

| Bit  | 7     | 6     | 5–4   | 3     | 2     | 1–0    |
|------|-------|-------|-------|-------|-------|--------|
| Name | `0`   | `AOE` | `AIP` | `0`   | `AI`  | `CHN`  |

| Field | Bits | Description |
|-------|------|-------------|
| `AOE` | 6    | Analog output enable — 1 = AOUT active, 0 = high-impedance |
| `AIP` | 5:4  | Analog input programming (see table below) |
| `AI`  | 2    | Auto-increment — 1 = channel number incremented after each conversion |
| `CHN` | 1:0  | Starting A/D channel number (0–3) |

Bits 7 and 3 are reserved and must be written as `0`.

#### AIP — Analog Input Programming

| AIP | Mode | Channels |
|-----|------|---------|
| `00` | 4 single-ended inputs | AIN0, AIN1, AIN2, AIN3 (channels 0–3) |
| `01` | 3 differential inputs | AIN0−AIN3 (ch0), AIN1−AIN3 (ch1), AIN2−AIN3 (ch2) |
| `10` | Single-ended + differential mixed | AIN0 (ch0, SE), AIN1 (ch1, SE), AIN2−AIN3 (ch2, diff) |
| `11` | 2 differential inputs | AIN0−AIN1 (ch0), AIN2−AIN3 (ch1) |

## Initialization Sequence

1. At Power-On Reset (POR) the control register is cleared to `0x00`; DAC is disabled, AOUT is high-impedance.
2. Send a write transaction with the desired control byte to configure input mode and optionally enable the DAC.
3. No startup delay is required; the first read transaction immediately triggers conversion.

## Implementation Stages

### Minimal

Goal: read all four analog inputs as raw 8-bit unsigned values with a single call per channel. No configuration required beyond the transport.

| Operation | Parameters | Returns | Notes |
|-----------|------------|---------|-------|
| `init` | transport | — | Stores transport; no I²C traffic |
| `read_channel` | `channel: int` (0–3) | `int` (0–255) | Single-ended mode; discards stale first byte; clamps channel to [0, 3] |
| `read_all` | — | `list[int]` (4 values) | Reads all four channels using auto-increment; returns [ch0, ch1, ch2, ch3] |

**Sensible defaults:** AIP=`00` (4 single-ended inputs), `AOE`=0 (DAC disabled), `AI`=0 for `read_channel`, `AI`=1 for `read_all`.

### Full

Goal: expose complete chip functionality. Extends Minimal.

| Operation | Parameters | Returns | Notes |
|-----------|------------|---------|-------|
| *(inherits Minimal)* | | | |
| `configure` | `input_mode: int` (0–3), `auto_increment: bool`, `dac_enabled: bool` | — | Sets AIP, AI, and AOE; writes control byte immediately |
| `read_channel` | `channel: int`, `vref: float`, `vagnd: float` | `float` | Overload returning voltage in volts given reference and ground voltages |
| `read_all_voltage` | `vref: float`, `vagnd: float` | `list[float]` | All four channels as voltages |
| `read_differential` | `channel: int` (0–2 or 0–1 depending on mode) | `int` | Returns signed two's complement value (−128 to 127) |
| `set_dac` | `value: int` (0–255) | — | Enable DAC and write value; AOUT = VAGND + (VREF−VAGND)×value/256 |
| `set_dac_voltage` | `voltage_fraction: float` (0.0–1.0) | — | Sets DAC to fraction of (VREF−VAGND); clamps to [0.0, 1.0] |
| `disable_dac` | — | — | Clear AOE bit; AOUT returns to high-impedance |

**Additional configuration options:**
- Analog input mode (single-ended, differential variants, mixed)
- Auto-increment for burst channel reads
- DAC enable/disable with raw or fractional voltage control
- Voltage-calibrated ADC reads using user-supplied VREF and VAGND

## Data Conversion

### ADC — single-ended

```
V_IN = V_AGND + raw × (V_REF − V_AGND) / 256

raw = int((V_IN − V_AGND) / (V_REF − V_AGND) × 256)   # 0–255
```

### ADC — differential (two's complement)

```
signed_raw = raw if raw < 128 else raw − 256            # interpret as signed int8

V_DIFF = signed_raw × (V_REF − V_AGND) / 256           # −V_FS/2 to +V_FS/2
```

where `V_FS = V_REF − V_AGND`.

### DAC output

```
V_AOUT = V_AGND + value × (V_REF − V_AGND) / 256

value = round(fraction × 255)                           # 0.0 → 0, 1.0 → 255
```

## Examples

### Demo

Wire a potentiometer across VAGND–VREF with the wiper connected to AIN0. Connect a small LED (with series resistor) to AOUT. In a loop, read AIN0, map the 0–255 value directly to a DAC output value, and write it to AOUT — the LED brightness tracks the potentiometer. Print the raw ADC value and the computed input and output voltages (assuming 3.3 V supply) on each iteration. This demonstrates the ADC→DAC feedback path in a single chip.

## Timing Constraints

- **I²C max SCL:** 100 kHz (standard mode; fast and high-speed modes are not supported).
- **ADC conversion time:** 90 µs maximum per channel.
- **DAC settling time:** 90 µs maximum (to within ½ LSB).
- **Auto-increment reads:** At 100 kHz I²C, back-to-back channel reads are self-paced by the bus; no additional delay is required between bytes.

## Implementation Notes

- **Stale first byte:** The first byte returned in any read transaction is the result of the *previous* conversion. Always read `n+1` bytes and discard byte 0 to obtain `n` fresh samples. For `read_all` using auto-increment, read 5 bytes and use bytes 1–4.
- **Auto-increment wrap:** If an out-of-range channel is selected, the chip allocates the highest valid channel. With AI=1, the next channel after the last valid one is always channel 0.
- **DAC and oscillator interaction:** When using auto-increment with the internal oscillator, the AOE bit (bit 6) in the control byte must be set to 1. This keeps the internal oscillator running continuously, preventing conversion errors from oscillator start-up delay. Clear AOE when auto-increment is not needed to reduce power consumption.
- **EXT pin:** For internal oscillator operation (the normal case), EXT must be connected to V_SS. Connect EXT to V_DD only to supply an external clock via OSC.
- **POR state:** After power-on, control register = `0x00`, DAC disabled, AOUT high-impedance. First read byte is `0x80`.
- **Internal oscillator frequency:** 0.75–1.25 MHz (typical 1 MHz). This clock is used internally for successive approximation; it is not related to I²C speed.
- **Unused analog inputs** should be tied to V_SS or V_DD (not left floating).
- **Reference voltage:** VREF and AGND set the full-scale range for both ADC and DAC. AGND may have a DC offset from V_SS. Decoupling capacitors (>1 µF) on V_DD and V_REF are recommended.
