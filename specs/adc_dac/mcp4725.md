# Chip Spec: MCP4725

**Manufacturer:** Microchip  
**Datasheet:** `datasheets/adc_dac/mcp4725.pdf`  
**Category:** adc_dac  
**Transports:** I²C  
**See also:** `specs/adc_dac/mcp4728.md` (quad-channel variant)

## Overview

The MCP4725 is a single-channel, 12-bit buffered voltage-output DAC with on-chip nonvolatile EEPROM. It outputs a rail-to-rail analog voltage from 0 V to V_DD based on a 12-bit code written over I²C. On every power-up the EEPROM contents are automatically loaded into the DAC register, so the output resumes from its last saved value without any host action. Typical applications are set-point or offset calibration, sensor biasing, and closed-loop servo control.

## Transport Configuration

### I²C

- **Address:** `0x60` (A0 pin = GND) — `0x61` (A0 pin = VDD)  
  Device code is fixed at `1100`; A2 and A1 are factory hard-wired to `00`; A0 tracks the logic level of the A0 pin. Full address byte = `1100 A2 A1 A0`.
- **Max clock:** 3.4 MHz (high-speed); 400 kHz (fast); 100 kHz (standard)

## Command / Register Map

The MCP4725 has no register address byte. Command type is encoded in the data payload itself.

### Write Command Types

| C2 | C1 | C0 | Command Name | Bytes (excl. address) | Description |
|----|----|----|--------------|----------------------|-------------|
| 0  | 0  | X  | Fast Write | 2 | Write DAC register only; EEPROM unaffected |
| 0  | 1  | 0  | Write DAC Register | 3 | Write DAC register; EEPROM unaffected |
| 0  | 1  | 1  | Write DAC + EEPROM | 3 | Write DAC register and EEPROM |

### Fast Write Payload (2 bytes, most common)

| Byte | Bits 7:6 | Bits 5:4 | Bits 3:0 |
|------|----------|----------|----------|
| 1    | `00` (C2/C1 = Fast) | PD1, PD0 | D11–D8 |
| 2    | D7–D0    | —        | — |

### Write DAC Register / Write DAC + EEPROM Payload (3 bytes)

| Byte | Bits 7:5 | Bits 4:3 | Bit 2 | Bits 1:0 | Bit 0 |
|------|----------|----------|-------|----------|-------|
| 1    | C2 C1 C0 | `XX` (unused) | PD1 | PD0 | X |
| 2    | D11–D4   | —        | —    | —        | — |
| 3    | D3–D0    | `XXXX` (unused) | — | — | — |

### Read Response (5 bytes)

| Byte | Bit 7 | Bit 6 | Bits 5:4 | Bits 3:1 | Bit 0 |
|------|-------|-------|----------|----------|-------|
| 1    | RDY/BSY | POR | `XX` | PD1 PD0 (DAC) | X |
| 2    | D11–D4 (DAC register) | — | — | — | — |
| 3    | D3–D0 (DAC register), `XXXX` | — | — | — | — |
| 4    | X PD1 PD0 (EEPROM) X D11–D8 (EEPROM) | — | — | — | — |
| 5    | D7–D0 (EEPROM) | — | — | — | — |

### Bit Fields

#### Status byte (read byte 1)

| Bits | Name | Description |
|------|------|-------------|
| 7    | RDY/BSY | 1 = EEPROM write complete; 0 = write in progress |
| 6    | POR | 1 = device has not been reset since last POR |
| 3:2  | PD1:PD0 | Current power-down setting of DAC register |

#### Power-Down Bits (PD1:PD0)

| PD1 | PD0 | Mode |
|-----|-----|------|
| 0   | 0   | Normal operation |
| 0   | 1   | Power-down, 1 kΩ to GND |
| 1   | 0   | Power-down, 100 kΩ to GND |
| 1   | 1   | Power-down, 500 kΩ to GND |

#### General Call Commands (sent to address 0x00)

| Second byte | Command |
|-------------|---------|
| `0x06`      | Reset — triggers internal POR; reloads EEPROM into DAC register |
| `0x09`      | Wake-Up — clears power-down bits in DAC register |

## Initialization Sequence

1. Apply power; device auto-loads EEPROM into DAC register (no host action required).
2. *(Optional but recommended when V_DD ramp rate < 1 V/ms)* Send General Call Reset: write `[0x00, 0x06]` to address `0x00`.
3. The output is immediately valid after POR completes.

## Implementation Stages

### Minimal

Goal: output a target voltage as a fraction of V_DD with a single call. No configuration required beyond the transport.

| Operation | Parameters | Returns | Notes |
|-----------|------------|---------|-------|
| `init` | transport | — | Stores transport; no I²C traffic needed |
| `set_voltage` | `fraction: float` (0.0–1.0) | — | Clamps to [0.0, 1.0]; uses Fast Write; updates DAC register only |
| `set_raw` | `code: int` (0–4095) | — | Clamps to [0, 4095]; uses Fast Write; updates DAC register only |

**Sensible defaults:** Normal mode (PD1=0, PD0=0) on every write. No EEPROM write on Minimal operations.

### Full

Goal: expose complete chip functionality. Extends Minimal.

| Operation | Parameters | Returns | Notes |
|-----------|------------|---------|-------|
| *(inherits Minimal)* | | | |
| `set_voltage_eeprom` | `fraction: float` (0.0–1.0) | — | Write DAC + EEPROM; persists across power cycles |
| `set_raw_eeprom` | `code: int` (0–4095) | — | Write DAC + EEPROM |
| `read` | — | `dict` / struct | Returns `{"code": int, "voltage_fraction": float, "power_down": int, "eeprom_code": int, "eeprom_power_down": int, "eeprom_ready": bool}` |
| `set_power_down` | `mode: int` (0–3) | — | Fast Write with PD bits set; code 0 preserves last value; exits PD immediately when mode=0 |
| `wake_up` | — | — | Send General Call Wake-Up (0x00, 0x09); clears PD bits in DAC register |
| `reset` | — | — | Send General Call Reset (0x00, 0x06); reloads EEPROM into DAC register |
| `is_eeprom_ready` | — | `bool` | Read RDY/BSY bit; True = EEPROM write complete |

**Additional configuration options:**
- EEPROM persistence for startup voltage
- All four power-down modes
- General Call Reset and Wake-Up
- Read-back of both DAC register and EEPROM contents

## Data Conversion

```
V_OUT = V_DD × code / 4096

code = round(fraction × 4095)   # 0.0 → 0, 1.0 → 4095
fraction = code / 4095.0
```

`V_DD` is both the supply voltage and the DAC reference. A 3.3 V supply gives 0.806 mV/LSB; a 5.0 V supply gives 1.221 mV/LSB.

## Examples

### Demo

Simulate a smooth triangle wave: ramp the DAC output from 0 to full scale and back down in steps, printing the current fraction and approximate voltage (assuming 3.3 V supply) on each step. Use a step size of 1/20 of full scale and a 100 ms delay between steps. This makes a visible sawtooth on an oscilloscope and illustrates the resolution available from a 12-bit DAC.

## Timing Constraints

- **Output settling time:** 6 µs typical (within 0.5 LSB, code transition 400h → C00h).
- **Exit power-down settling:** < 10 µs.
- **EEPROM write time:** 25 ms typical, 50 ms maximum. The RDY/BSY bit is low during the write; new write commands are ignored. Poll `is_eeprom_ready()` or wait 50 ms before issuing a second EEPROM write.
- **Fast Write V_OUT update:** occurs at the falling edge of the ACK pulse of the 3rd byte (2nd data byte).

## Implementation Notes

- The MCP4725 has **no register address byte**. After the I²C address byte, the first data byte's upper bits encode the command type (C2, C1, C0). Fast Write uses only 2 data bytes; Write DAC Register / Write DAC + EEPROM uses 3 data bytes.
- **Factory EEPROM default:** D11=1, all others 0 → code = 0x800 → output = 0.5 × V_DD (middle scale). PD1=PD0=0 (normal mode).
- **A2 and A1 address bits** are factory hard-wired; users can only choose A0 via the A0 pin. Two devices can coexist on the same bus using the A0 pin for differentiation.
- **V_DD ramp rate:** if V_DD rises slower than 1 V/ms at power-up, the EEPROM may not load correctly. Send a General Call Reset (0x00, 0x06) after V_DD stabilizes to guarantee correct initial output.
- The chip operates as an I²C **slave only**; it never initiates transactions.
- During an EEPROM write, the I²C interface remains active but any new write command (including repeat bytes) is ignored until RDY/BSY returns high.
- For best analog output accuracy, decouple V_DD with 0.1 µF ceramic + 10 µF tantalum capacitors placed within 4 mm of the VDD pin.
- `wake_up()`, `reset()`, and `is_eeprom_ready()` share identical General Call semantics with the MCP4728. Implementations may derive from a shared `_MCP47xxBase` containing `_general_call(cmd)`, `wake_up()`, `reset()`, and `is_eeprom_ready()`.
