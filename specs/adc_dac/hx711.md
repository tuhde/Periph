# Chip Spec: HX711

**Manufacturer:** Avia Semiconductor  
**Datasheet:** `datasheets/adc_dac/hx711.pdf`  
**Category:** adc_dac  
**Transports:** HX711 GPIO bit-bang — see `specs/transport_hx711.md`

## Overview

The HX711 is a precision 24-bit analog-to-digital converter designed for weigh scales and industrial force-measurement applications. It interfaces directly with a resistive bridge sensor (load cell) through an on-chip low-noise programmable gain amplifier (PGA). Two selectable differential input channels are provided: Channel A with software-selectable gain of 128 or 64, and Channel B with fixed gain of 32. An on-chip voltage regulator, bandgap reference, and oscillator eliminate the need for external components. Communication uses a simple 2-wire bit-bang protocol (no SPI or I²C bus required) with selectable output rates of 10 SPS or 80 SPS.

## Transport

The HX711 uses a chip-specific 2-wire GPIO bit-bang transport. See `specs/transport_hx711.md` for the full protocol, timing constraints, pin types per platform, and interface contract.

The driver constructs an `HX711Transport` instance and calls `read_raw(num_pulses)` to clock conversions. The pulse count (25, 26, or 27) selects the channel and gain for the **next** conversion.

## Initialization Sequence

1. Drive PD_SCK LOW to ensure chip is in normal mode
2. Wait for DOUT to go LOW (first conversion complete)
3. Issue one dummy read with 25 pulses to discard the first conversion result
4. Chip is now ready for use with Channel A, Gain 128

## Implementation Stages

Each chip is implemented in two stages. The Full class extends Minimal — it inherits everything and adds the rest.

### Minimal

Goal: read a raw 24-bit signed ADC value from Channel A with gain 128. No configuration required beyond the two GPIO pins.

| Operation | Parameters | Returns | Notes |
|-----------|------------|---------|-------|
| `init` | `dout`, `pd_sck` | — | Store pins; drive PD_SCK LOW; wait for first conversion, discard it |
| `is_ready` | — | bool | True if DOUT is LOW (data available), non-blocking |
| `read_raw` | — | int | Block until ready; clock out 24 bits; send 25 pulses total; return signed 24-bit value |

**Sensible defaults:** Channel A, Gain 128 (highest sensitivity, widest dynamic range for load cells); blocks indefinitely until DOUT goes LOW.

### Full

Goal: expose complete chip functionality. Extends Minimal.

| Operation | Parameters | Returns | Notes |
|-----------|------------|---------|-------|
| *(inherits Minimal)* | | | |
| `set_gain` | `gain: int` (128, 64, or 32) | — | Select gain; 128 and 64 use Channel A, 32 uses Channel B; takes effect after next `read_raw` call |
| `read_average` | `times: int` (default 10) | int | Average of `times` raw readings; blocks until all readings complete |
| `tare` | `times: int` (default 10) | — | Capture average of `times` readings as zero offset; stores in `_offset` |
| `get_offset` | — | int | Return current tare offset |
| `set_scale` | `factor: float` | — | Set calibration scale factor; stores in `_scale` |
| `get_scale` | — | float | Return current scale factor |
| `read_weight` | `times: int` (default 1) | float | Returns `(read_average(times) - _offset) / _scale` |
| `power_down` | — | — | Drive PD_SCK HIGH for > 60 µs to enter power-down mode |
| `power_up` | — | — | Drive PD_SCK LOW; chip resets to Channel A, Gain 128; next reading should be discarded |

**Additional configuration options:**
- Gain selection (128, 64, 32) with implicit channel selection
- Multi-sample averaging
- Tare offset capture and subtraction
- Scale factor calibration for real-world units (e.g., grams, kilograms)
- Power-down and wake-up

## Data Conversion

The HX711 outputs 24-bit two's complement values, transmitted MSB first. Sign-extend to a platform integer:

```
raw (uint24, MSB-first from wire) → signed int:
    if raw >= 0x800000: signed = raw - 0x1000000
    else:              signed = raw

signed range: -8388608 (0x800000) to +8388607 (0x7FFFFF)

weight = (raw_signed - tare_offset) / scale_factor
```

The scale factor is determined by calibration with a known weight:
1. Call `tare()` with nothing on the scale
2. Place a known weight W (in the target unit, e.g. grams)
3. `scale_factor = (read_average() - tare_offset) / W`

There is no fixed LSB size — effective resolution depends on gain, supply voltage, and load cell characteristics. At 5 V, Gain = 128: full-scale input = ±20 mV → ±8,388,607 counts (≈ 2.4 nV/count ideal).

Config panel fields:
- **DOUT pin** — GPIO pin number for serial data input from HX711
- **PD_SCK pin** — GPIO pin number for serial clock / power-down control
- **Gain** — 128 (ChA), 64 (ChA), or 32 (ChB) dropdown
- **Scale factor** — float; leave 1.0 if calibration is not configured
- **Tare offset** — int; set manually or populated via tare trigger message

## Timing Constraints

| Symbol | Event | Min | Typ | Max | Unit |
|--------|-------|-----|-----|-----|------|
| T1 | DOUT falling edge to PD_SCK rising edge | 0.1 | — | — | µs |
| T2 | PD_SCK rising edge to DOUT data valid | — | — | 0.1 | µs |
| T3 | PD_SCK high time | 0.2 | 1 | 50 | µs |
| T4 | PD_SCK low time | 0.2 | 1 | — | µs |

- **Settling time after power-up, reset, or channel/gain change:** 400 ms at 10 SPS, 50 ms at 80 SPS
- **Power-down entry:** PD_SCK must be held HIGH continuously for > 60 µs
- **PD_SCK pulses per conversion:** must be exactly 25, 26, or 27 — fewer than 25 leaves the conversion incomplete; more than 27 causes serial communication errors

## Implementation Notes

- **No register map:** All chip configuration is implicit in the clock pulse count (25/26/27). There are no I²C or SPI registers.
- **Sign extension:** The 24-bit value arrives MSB-first as a raw unsigned integer. Explicitly sign-extend: `if raw >= 0x800000: raw -= 0x1000000` (Python) or `if (raw & 0x800000) raw |= 0xFF000000u` (C/C++). Do not use the XOR approach from the Avia reference driver — it requires the caller to treat the result as signed, which is implicit and error-prone.
- **First reading after init:** The chip begins converting immediately on power-up; the first result may be invalid. The `init` method must clock out and discard one reading before returning.
- **Blocking reads on Linux:** `read_raw()` polls DOUT in a tight loop. On Linux, insert a `time.sleep(0.001)` between polls to avoid busy-waiting a CPU core.
- **RATE pin:** The sample rate (10 or 80 SPS) is determined by the RATE hardware pin, not by software. Most breakout boards tie RATE to GND (10 SPS). The driver does not control RATE.
- **Channel B fixed gain:** Channel B always operates at gain 32; there is no way to change it in software.
- **Power-down clock pulse limit:** During normal data clocking, T3 (PD_SCK high time) must not exceed 50 µs or the chip interprets the pulse as a power-down command. Keep clock pulses well under 50 µs.
- **Rust blocking:** `embedded-hal` 1.0 defines no standard `Wait` trait for GPIO edge detection. The Rust driver polls `InputPin::is_low()` in a spin loop, which is compatible with both `no_std` bare-metal and Linux host targets.
- **Channel/gain change settling:** After changing gain via `set_gain`, the new setting takes effect only after the *next* read completes a conversion with the new pulse count. That first reading under the new setting must be discarded; call `read_raw()` once and ignore the return value before relying on data.
