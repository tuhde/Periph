# Chip Spec: BMP581

**Manufacturer:** Bosch Sensortec  
**Datasheet:** `datasheets/pressure/bmp581.pdf`  
**Category:** `pressure`  
**Transports:** I²C, SPI

## Overview

The BMP581 is a high-accuracy MEMS barometric pressure and temperature sensor from Bosch Sensortec. It covers 30–125 kPa absolute pressure across −40 to +85 °C with a 24-bit output, giving 1/64 Pa (≈ 0.016 Pa) pressure resolution and 1/65536 °C temperature resolution. No user calibration is required; factory compensation is applied internally. An integrated IIR low-pass filter removes short-term disturbances. The chip supports four power modes (DEEP STANDBY, STANDBY, FORCED, NORMAL, CONTINUOUS), a configurable ODR from 0.125 to 240 Hz, per-channel oversampling (×1 to ×128), a 32-frame FIFO, and a programmable out-of-range detector. Two independent supply rails (VDD 1.71–3.6 V analog, VDDIO 1.08–3.6 V digital) allow direct interfacing to 1.2 V MCU logic. Typical applications: altimeters, drones, weather stations, GPS dead-reckoning, indoor navigation.

## Transport Configuration

### I²C

- **Address:** `0x46` (SDO=0) / `0x47` (SDO=1)
- **Max clock:** 1 MHz (fast-mode plus)
- **Protocol selection:** CSB pin must be tied to VDDIO to select I²C mode; if CSB goes low, the chip switches to SPI and I²C is disabled until the next power-on reset
- **Auto-increment:** register address pointer increments automatically on multi-byte reads; FIFO_DATA (0x29) is the only exception (non-incrementing)

Modes supported: standard (100 kHz), fast (400 kHz), fast-mode plus (400 kHz – 1 MHz). In I²C mode, enabling the internal CSB pull-up via DRIVE_CONFIG.i2c_csb_pup_en (bit 0) is optional; external pull-ups to VDDIO are the normal configuration.

Note: when writing CMD = 0xB6 (soft reset) via I²C, no ACK will be returned to the host — this is expected behavior and not an error.

### SPI

- **Mode:** Mode 0 (CPOL=CPHA=0) or Mode 3 (CPOL=CPHA=1), auto-selected by SCK level after CSB falling edge
- **Max clock:** 12 MHz (VDDIO ≥ 1.62 V)
- **Bit order:** MSB first
- **CS active:** low (CSB)
- **Wiring:** 4-wire (default) or 3-wire (DRIVE_CONFIG.spi3_en = 1)
- **Auto-increment:** same rule as I²C; burst reads increment address except at FIFO_DATA

Write: W(1) + Addr(7) + Data(8). Burst write: repeated W(1)+Addr(7)+Data(8) per byte — address sent separately for each byte.  
Read: R(1) + Addr(7) then continuous data bytes with auto-increment.

After initial power-up or CSB low-to-high transition, perform a dummy single-byte read (e.g. CHIP_ID) before real SPI communication — the first read after CSB falls returns invalid data while the SPI FSM initialises.

### ODR Table

ODR field is bits [6:2] of ODR_CONFIG. Selected common rates:

| odr[6:2] | Frequency | odr[6:2] | Frequency |
|----------|-----------|----------|-----------|
| `0x00` | 240 Hz | `0x11` | 40 Hz |
| `0x04` | 160 Hz | `0x17` | 10 Hz |
| `0x08` | 120 Hz | `0x1C` | 1 Hz (reset) |
| `0x0D` | 70 Hz | `0x1D` | 0.5 Hz |
| `0x0E` | 60 Hz | `0x1E` | 0.25 Hz |
| `0x0F` | 50 Hz | `0x1F` | 0.125 Hz |

To compute the ODR_CONFIG byte for NORMAL mode: `(odr_field << 2) | 0x01`. E.g. 1 Hz NORMAL: `(0x1C << 2) | 0x01 = 0x71`.

## Initialization Sequence

1. Apply power; wait ≤2 ms for VDD and VDDIO to stabilize
2. For SPI: perform one dummy read (e.g. CHIP_ID) and discard result; the first byte after CSB falls is invalid
3. Read CHIP_ID (0x01); verify = `0x50` — fail if mismatch
4. Read STATUS (0x28); poll until status_nvm_rdy (bit 1) = 1 and status_nvm_err (bit 2) = 0
5. Read INT_STATUS (0x27); verify por (bit 4) = 1 (expected value `0x10`) — clears on read
6. Write CMD (0x7E) ← `0xB6` (soft reset); wait ≤2 ms; note: no I²C ACK returned
7. Poll STATUS again: status_nvm_rdy=1, status_nvm_err=0; read INT_STATUS to verify por=1
8. Write OSR_CONFIG (0x36) ← `0x40` (press_en=1, osr_p=×1, osr_t=×1)
9. DSP_CONFIG (0x30) is already `0x03` (comp_pt_en=P+T) at reset — no write needed
10. Write ODR_CONFIG (0x37) ← `0x71` (odr=1 Hz, NORMAL mode)

## Implementation Stages

### Minimal

Goal: deliver calibrated temperature and pressure at 1 Hz NORMAL mode with no configuration required beyond the transport.

| Operation | Parameters | Returns | Notes |
|-----------|------------|---------|-------|
| `init` | transport | — | Runs initialization sequence; chip enters continuous NORMAL mode at 1 Hz |
| `pressure` | — | float | Pa; burst-reads 0x20–0x22, converts using data conversion formula |
| `temperature` | — | float | °C; burst-reads 0x1D–0x1F, converts using data conversion formula |
| `both` | — | (float, float) | (Pa, °C); single burst read of 0x1D–0x22 for both channels atomically |

**Sensible defaults baked in:**
- NORMAL mode (ODR-based, duty-cycled; chip manages timing autonomously)
- ODR = 1 Hz (odr=0x1C → ODR_CONFIG byte 0x71)
- osr_p = ×1, osr_t = ×1 (fastest conversion; ~1 ms each)
- press_en = 1 (pressure measurements enabled)
- comp_pt_en = 0x03 (P+T compensation; default value, not written)
- IIR filter = bypass (set_iir_p=0, set_iir_t=0; default value, not written)
- FIFO disabled (default)
- INT pin not configured for any interrupt source

## Data Conversion

Temperature (°C) from 24-bit signed output (arithmetic representation: signed, 24, 16):

```
raw_t = sign_extend_24((MSB << 16) | (LSB << 8) | XLSB)
T_degC = raw_t / 65536.0
```

Burst-read 0x1D (XLSB), 0x1E (LSB), 0x1F (MSB) → 3 bytes.

Pressure (Pa) from 24-bit signed output (arithmetic representation: signed, 24, 6):

```
raw_p = sign_extend_24((MSB << 16) | (LSB << 8) | XLSB)
p_Pa = raw_p / 64.0
```

Burst-read 0x20 (XLSB), 0x21 (LSB), 0x22 (MSB) → 3 bytes.

Both channels atomically: burst-read 0x1D–0x22 → 6 bytes; decode T from bytes 0–2, P from bytes 3–5.

sign_extend_24: if bit 23 is set, subtract 2^24 from the unsigned value.

### Altitude

```
altitude_m = 44330.0 * (1.0 - (p_Pa / sea_level_Pa) ** (1.0 / 5.255))
```

Standard sea-level pressure: 101325 Pa.

### OOR threshold encoding

The 17-bit OOR threshold (oor_thr_p_16 + OOR_THR_P_MSB + OOR_THR_P_LSB) is compared against bits [23:7] of the 24-bit raw pressure value:

```
oor_thr_17bit = int(threshold_Pa * 64.0) >> 7   # = int(threshold_Pa / 2)
oor_thr_p_16  = (oor_thr_17bit >> 16) & 0x01
oor_thr_p_msb = (oor_thr_17bit >> 8) & 0xFF
oor_thr_p_lsb = oor_thr_17bit & 0xFF
```

OOR_RANGE.oor_range_p encodes the symmetric window (±range) around the threshold using the same 8-bit LSB of the 17-bit scale.

## Timing Constraints

- Power-up time: ≤2 ms before first I²C/SPI transaction
- Soft reset duration: ≤2 ms; poll STATUS.nvm_rdy before proceeding
- STANDBY-to-NORMAL startup: ≤3 ms
- Conversion times at OSR ×1: ~1.0 ms pressure, ~1.0 ms temperature (both in series: ~2.0 ms)
- Conversion times at OSR ×128: ~80.4 ms pressure, ~20.8 ms temperature
- Minimum idle time after SPI write to DRIVE_CONFIG, INT_CONFIG, NVM_DATA_LSB, NVM_DATA_MSB before read-back: 1 µs
- NVM write/read sequences require the device to be in STANDBY mode
- FIFO_SEL changes must be made in STANDBY mode

## Implementation Notes

- **No factory calibration to read:** The BMP581 performs compensation internally; the output registers already contain calibrated Pa and °C values. There is no user-visible calibration step. This is the key difference from the BMP180/BMP280.
- **Data format is fixed-point, not integer:** raw_t / 65536.0 gives °C; raw_p / 64.0 gives Pa. Both are 24-bit two's-complement. The data sheets calls this "arithmetic representation: (signed, 24, N)" where N is the number of fractional bits (16 for T, 6 for P).
- **Burst read is mandatory for data consistency:** The chip's data shadow registers update atomically only within a burst read. Reading XLSB, LSB, MSB of pressure in three separate single-byte reads is not guaranteed to be consistent. Always burst-read 0x20–0x22 (or 0x1D–0x22 for both).
- **FIFO_DATA auto-increment exception:** All registers auto-increment on burst read except FIFO_DATA (0x29). To read N FIFO frames, read N×3 bytes starting at 0x29 — each byte delivered is the next FIFO byte, not the next register address.
- **INT_STATUS is destructive:** Reading INT_STATUS (0x27) clears all its bits (clear-on-read). `data_ready()` in Full reads it and returns only the drdy bit; callers that need multiple bits at once should call `interrupt_status()` (returns the raw byte) and inspect the bits themselves.
- **INT_CONFIG reset=0x35:** The interrupt pin is enabled by default (int_en=1) but INT_SOURCE resets to 0x00 (no sources active), so the pin does not fire on anything until sources are enabled. In Minimal, leave INT_SOURCE at 0x00 (no write needed).
- **SPI dummy read:** After CSB first falls (power-up or CSB transition), the device needs ≥16 SCK edges before it reliably returns data. Perform one dummy single-byte read of CHIP_ID and discard the result.
- **Soft reset via I²C — no ACK:** When writing 0xB6 to CMD via I²C, the chip resets before transmitting the ACK. The host will see a NACK (or bus timeout). This is expected; do not treat it as an error. Wait 2 ms and then re-check STATUS.nvm_rdy.
- **FORCED mode pwr_mode read-back:** After a FORCED measurement completes, the chip returns to STANDBY and clears pwr_mode to 0x0. Reading ODR_CONFIG while FORCED is ongoing returns 0x2; after completion it returns 0x0.
- **deep_dis and DEEP STANDBY:** When ODR_CONFIG.deep_dis=0 (default) and the chip is in STANDBY, it enters DEEP STANDBY (0.55 µA). DEEP STANDBY has a ≤3 ms wake-up latency before the first NORMAL measurement. If latency matters, set deep_dis=1 to keep the chip in regular STANDBY (1.0 µA).
- **IIR and data routing:** By default (DSP_CONFIG reset=0x03, shdw_sel_iir_p/t=0) the data registers hold pre-IIR values. To route IIR-filtered output to the data registers, set shdw_sel_iir_p/t=1 in DSP_CONFIG in addition to setting the filter coefficient in DSP_IIR.
- **NVM row addresses:** User NVM rows are 0x20, 0x21, 0x22 (3 rows × 2 bytes = 48 bits of user storage). These are different from the NVM row address encoding — row 0x20 maps to an internal address, not register address 0x20. Maximum 10,000 write cycles total (not per row).
- **OSR/ODR validity:** If the ODR is set too fast for the selected OSR combination, the chip silently reduces the effective OSR. Read OSR_EFF.odr_is_valid to detect this; read osr_p_eff/osr_t_eff to see the actual OSR in use.

## Sigrok Decoder

Decoder id `bmp581`, input `['i2c', 'spi']`. Matches I²C addresses `0x46` and `0x47`; matches SPI by protocol type. Annotates: CHIP_ID read (0x01) verified against 0x50; STATUS byte decoded to named flags; INT_STATUS byte decoded with bit names; OSR_CONFIG decoded to pressure-enable flag, numeric osr_p and osr_t values (×1 through ×128); ODR_CONFIG decoded to power mode name and ODR in Hz; DSP_IIR decoded to P and T filter coefficient names; pressure data reads (0x20–0x22) showing raw 24-bit value and computed Pa; temperature data reads (0x1D–0x1F) showing raw 24-bit value and computed °C; combined burst reads (0x1D–0x22) decoded as both channels; CMD writes decoded by value (soft reset, NVM sequences); FIFO_DATA reads showing per-frame decoded values; INT_SOURCE register changes noted.
