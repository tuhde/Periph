# Chip Spec: BMP384

**Manufacturer:** Bosch Sensortec  
**Datasheet:** `datasheets/pressure/bmp384.pdf`  
**Category:** pressure  
**Transports:** I²C, SPI

## Overview

The BMP384 is a high-precision, ultra-low-power digital barometric pressure and temperature sensor. It measures absolute pressure from 300 to 1250 hPa with ±0.5 hPa typical accuracy and temperature from −40 to +85 °C with ±0.5 °C accuracy. On-chip oversampling (up to ×32) and an IIR filter reduce noise without host intervention. Typical supply current is 4 µA at 1 Hz normal mode. The BMP384 targets weather stations, altitude tracking, indoor navigation, drone flight control, and IoT nodes that need barometric data at very low power.

## Transport Configuration

### I²C

- **Address:** `0x76` (SDO = GND) — `0x77` (SDO = VDD)
- **Max clock:** 3.4 MHz (high-speed); 400 kHz (fast); 100 kHz (standard)

### SPI

- **Mode:** CPOL=0 CPHA=0 (Mode 0) or CPOL=1 CPHA=1 (Mode 3) — auto-detected from SCK polarity at CSB falling edge
- **Max clock:** 10 MHz
- **Bit order:** MSB first
- **CS active:** low
- **Default wiring:** 4-wire; 3-wire mode enabled via `spi3` bit in IF_CONF (`0x1A`)

Read transactions set bit 7 of the address byte; write transactions clear it. On SPI, the first byte returned after the address byte is a dummy byte and must be discarded.

## Initialization Sequence

1. Power up; wait 2 ms for boot.
2. *(Optional)* Soft reset: write `0xB6` to CMD (`0x7E`); wait 2 ms.
3. Verify CHIP_ID (`0x00`) == `0x50`; abort if mismatch.
4. Burst-read 21 bytes from `0x31`–`0x45`; parse trimming coefficients and convert to floating-point PAR values (see Data Conversion).
5. Write OSR (`0x1C`): configure osr_p and osr_t.
6. Write CONFIG (`0x1F`): configure iir_filter.
7. Write ODR (`0x1D`): configure odr_sel.
8. Write PWR_CTRL (`0x1B`): set press_en=1, temp_en=1, mode=`11` (normal) or mode=`01` (forced).

## Implementation Stages

### Minimal

Goal: read barometric pressure and temperature with a single call, no configuration required beyond the transport.

| Operation | Parameters | Returns | Notes |
|-----------|------------|---------|-------|
| `init` | transport | — | Reads calibration; applies defaults; starts normal mode |
| `read_pressure` | — | float hPa | Compensated; burst-reads 0x04–0x09 (needs temp for t_lin) |
| `read_temperature` | — | float °C | Compensated; reads 0x07–0x09 |

**Sensible defaults:** osr_p=4 (×16), osr_t=1 (×2), iir_filter=2 (coef 3), odr_sel=`0x03` (25 Hz), normal mode, both sensors enabled.

### Full

Goal: expose complete chip functionality. Extends Minimal.

| Operation | Parameters | Returns | Notes |
|-----------|------------|---------|-------|
| *(inherits Minimal)* | | | |
| `configure` | `osr_p: int` (0–5), `osr_t: int` (0–5), `iir_filter: int` (0–7), `odr_sel: int` (0x00–0x11) | — | Writes OSR, CONFIG, ODR; validates ODR ≥ T_conv |
| `read` | — | dict | `{"pressure": float, "temperature": float}`; single burst read of 0x04–0x09 |
| `read_forced` | — | dict | Writes forced mode, waits T_conv, reads, returns same dict as `read` |
| `set_mode` | `mode: int` (0, 1, 3) | — | 0=sleep, 1=forced, 3=normal; writes PWR_CTRL bits 5:4 |
| `is_data_ready` | — | bool | True if STATUS.drdy_press is set |
| `softreset` | — | — | Writes 0xB6 to CMD; waits 2 ms; re-reads calibration |
| `fifo_configure` | `press_en: bool`, `temp_en: bool`, `wtm: int` (0–511), `stop_on_full: bool` | — | Enables FIFO; writes FIFO_CONFIG_1, FIFO_WTM_0/1 |
| `fifo_read` | — | list of dicts | Reads FIFO_LENGTH bytes from FIFO_DATA; parses frames; each dict: `{"type": str, "value": float}` |
| `fifo_flush` | — | — | Writes 0xB0 to CMD |

**Additional configuration options:**
- Oversampling up to ×32 for pressure and temperature independently
- IIR filter coefficients 0 (bypass) through 127
- 18 ODR settings from 200 Hz down to 25/16384 Hz (≈0.0015 Hz)
- Forced mode for power-efficient single-shot measurements
- 512-byte FIFO (~64 pressure+temperature frame pairs at 4 bytes each), configurable watermark and stop-on-full
- Data-ready, FIFO-full, and FIFO-watermark interrupt output on INT pin

## Data Conversion

### Calibration coefficient scaling

```
PAR_T1  = NVM_PAR_T1 / 2**-8        # = NVM_PAR_T1 × 256.0
PAR_T2  = NVM_PAR_T2 / 2**30
PAR_T3  = NVM_PAR_T3 / 2**48

PAR_P1  = (NVM_PAR_P1  - 2**14) / 2**20
PAR_P2  = (NVM_PAR_P2  - 2**14) / 2**29
PAR_P3  = NVM_PAR_P3  / 2**32
PAR_P4  = NVM_PAR_P4  / 2**37
PAR_P5  = NVM_PAR_P5  / 2**-3       # = NVM_PAR_P5 × 8.0
PAR_P6  = NVM_PAR_P6  / 2**6
PAR_P7  = NVM_PAR_P7  / 2**8
PAR_P8  = NVM_PAR_P8  / 2**15
PAR_P9  = NVM_PAR_P9  / 2**48
PAR_P10 = NVM_PAR_P10 / 2**48
PAR_P11 = NVM_PAR_P11 / 2**65
```

### Temperature compensation (result in °C)

```
partial1 = float(uncomp_temp) - PAR_T1
partial2 = partial1 * PAR_T2
t_lin    = partial2 + (partial1 * partial1) * PAR_T3
# t_lin is the compensated temperature in °C
# t_lin must also be passed to the pressure compensation below
```

### Pressure compensation (result in Pa; divide by 100 for hPa)

```
partial1     = PAR_P6 * t_lin
partial2     = PAR_P7 * t_lin * t_lin
partial3     = PAR_P8 * t_lin * t_lin * t_lin
partial_out1 = PAR_P5 + partial1 + partial2 + partial3

partial1     = PAR_P2 * t_lin
partial2     = PAR_P3 * t_lin * t_lin
partial3     = PAR_P4 * t_lin * t_lin * t_lin
partial_out2 = float(uncomp_press) * (PAR_P1 + partial1 + partial2 + partial3)

partial1     = float(uncomp_press) * float(uncomp_press)
partial2     = PAR_P9 + PAR_P10 * t_lin
partial3     = partial1 * partial2
partial4     = partial3 + float(uncomp_press)**3 * PAR_P11

comp_press   = partial_out1 + partial_out2 + partial4
```

### Altitude (approximate barometric formula)

```
altitude = 44330.0 * (1.0 - (P / P0) ** (1.0 / 5.255))
# P  = measured pressure in Pa
# P0 = 101325 Pa (ISA sea-level standard)
# result in metres
```

## FIFO Frame Format

Each sensor data frame is 4 bytes: a 1-byte header followed by 3 bytes of raw data (XLSB first). Error and empty frames are 1 byte (header only).

| Header | Frame type | Payload |
|--------|-----------|---------|
| `0x84` | Pressure | 3 bytes raw pressure (XLSB, LSB, MSB) |
| `0x90` | Temperature | 3 bytes raw temperature (XLSB, LSB, MSB) |
| `0xA0` | Sensor time | 3 bytes sensortime (XLSB, LSB, MSB) |
| `0x44` | Error | No payload |
| `0x80` | Empty | No payload |

Read FIFO_LENGTH_0/1 (`0x12`–`0x13`) to obtain the available byte count, then burst-read that many bytes from FIFO_DATA (`0x14`). Parse each frame in a loop: inspect the header byte, consume the correct number of payload bytes, then advance.

## Timing Constraints

- **Boot time:** 2 ms after power-up before first register access.
- **Soft-reset recovery:** 2 ms after writing `0xB6` to CMD.
- **Measurement time (T_conv):**

```
T_conv = 234 µs
       + press_en × (392 µs + 2^osr_p × 2000 µs)
       + temp_en  × (313 µs + 2^osr_t × 2000 µs)
```

Example with Minimal defaults (osr_p=4 →×16, osr_t=1 →×2, both enabled):
T_conv = 234 + 392 + 16×2000 + 313 + 2×2000 = 37,939 µs ≈ 38 ms.

The ODR period must be ≥ T_conv; otherwise ERR_REG.conf_err is set and measurements are unreliable. At 25 Hz (40 ms period) the Minimal defaults satisfy this constraint with ≈2 ms margin.

- **Forced mode:** poll STATUS.drdy_press or wait T_conv before reading output registers.

## Implementation Notes

- **Temperature before pressure** — the pressure compensation formula uses `t_lin` (the compensated temperature value). Always run temperature compensation first, even when only pressure is requested. Burst-read registers `0x04`–`0x09` in one transaction to get both raw values, then compute t_lin, then compute pressure.
- **SPI dummy byte** — on SPI, the byte immediately following the address byte in a read transaction is a dummy byte and must be discarded; actual data begins at the second response byte.
- **Forced mode re-trigger** — after one forced-mode measurement the device returns to sleep. To take another reading, write mode=`01` to PWR_CTRL again. `read_forced()` handles this automatically.
- **ODR/oversampling validation** — before writing to ODR or OSR, compute T_conv and verify the chosen ODR period ≥ T_conv. `configure()` should raise an error or clamp the ODR rather than silently producing bad data.
- **IIR filter scope** — the IIR filter is applied to the output registers and FIFO (when FIFO_CONFIG_2.data_select=1). It reduces the effect of short disturbances (door slams, wind gusts) but introduces step-response lag proportional to the coefficient.
- **CHIP_ID check on init** — read `0x00` and compare to `0x50`; raise an exception on mismatch to catch address/wiring errors early.
- **Calibration re-read after softreset** — `softreset()` clears the device state; call the internal calibration-read helper again before the next measurement.
- **Shared base class** — `_compensate_temperature()` and `_compensate_pressure()` are good candidates for a `_BMP3xxBase` helper, shared with any future BMP390 driver.
