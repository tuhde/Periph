# Chip Spec: RDA5807M

**Manufacturer:** RDA Microelectronics
**Datasheet:** `datasheets/comms/rda5807m.pdf`
**Category:** `comms`
**Transports:** I²C

## Overview

The RDA5807M is a single-chip broadcast FM stereo radio tuner: fully integrated synthesizer, low-IF receiver, digital audio DSP, RDS/RBDS decoder, and stereo DAC on one die, controlled entirely over I²C. It needs almost no external components (LNA, VCO, ADC/DAC, and loop filter are all on-chip) and drives a 32 Ω headphone load directly from LOUT/ROUT, which is why it shows up in cheap "Arduino FM radio" and portable-media-player designs. Frequency range is 50–115 MHz (covers all regional FM broadcast bands); tuning is done by writing a channel number relative to a per-region base frequency, not a raw frequency value.

Unlike every other chip currently in this project, the RDA5807M has no conventional "register pointer" protocol: the I²C interface has a **fixed** start register for writes (`0x02`) and a **different fixed** start register for reads (`0x0A`), with an internal auto-incrementing counter. There is no address byte sent to select which register a transaction targets — position in the byte stream is everything. This spec documents that scheme in detail because it drives most of the API design below. Register-level information is not otherwise well documented by RDA (the datasheet is a bare register table with no usage guidance), so this spec is cross-checked against several long-established open-source drivers (mathertel/Radio, pu2clr/RDA5807, the Linux kernel `radio-rda5807` driver) for behavior the datasheet leaves implicit — those points are called out explicitly in [Implementation Notes](#implementation-notes).

## Transport Configuration

### I²C

- **Address:** `0x10` (7-bit) — fixed, no address pins/straps. This is the *only* address documented in the datasheet ("command byte includes a 7-bit chip address (0010000b)"). Third-party sources report an undocumented "random access" address `0x11` that allows direct single-register addressing; this driver does not use it — see [Implementation Notes](#implementation-notes).
- **Max clock:** 400 kHz

There is no register-pointer byte in any transfer. A **write** transaction always begins at register `0x02` and auto-increments (`0x02` high byte, `0x02` low byte, `0x03` high byte, …) for as many bytes as are sent, stopping wherever the master issues STOP. A **read** transaction always begins at register `0x0A` and auto-increments the same way. If the running address reaches the end of the register file (`0x3A`), it wraps back to `0x00` — not relevant for normal operation since this driver never reads or writes past `0x0F`. Registers `0x00` (chip ID) and `0x01` (reserved) are read-only in this addressing scheme and are not reachable through the documented read path (which starts at `0x0A`), so this driver does not implement a chip-ID probe.

Because a write always starts at `0x02`, changing a single field (e.g. volume, which lives in register `0x05`) requires resending every register from `0x02` up through the one being changed. The driver keeps an in-memory shadow of registers `0x02`–`0x07` (6 × 16-bit words, big-endian, 12 bytes) and always writes all 12 bytes on any change, so unrelated fields are never clobbered.

## Register Map

### Write block (fixed start `0x02`, write-only)

| Address | Reset    | Description |
|---------|----------|-------------|
| `0x02`  | `0x0000` | CTRL — power, mute, seek control |
| `0x03`  | `0x0000` | CHAN — channel select, band, spacing, tune |
| `0x04`  | `0x0200` | R4 — de-emphasis, softmute, AFC |
| `0x05`  | `0x880F` | R5 — interrupt mode, seek threshold, volume |
| `0x06`  | `0x0000` | R6 — reserved-register access mode |
| `0x07`  | `0x4202` | R7 — soft-blend threshold, band extension, seek mode |

### Read block (fixed start `0x0A`, read-only)

| Address | Reset    | Description |
|---------|----------|-------------|
| `0x0A`  | `0x0400` | STATUSA — RDS ready, seek/tune complete, seek fail, stereo, read channel |
| `0x0B`  | `0x0000` | STATUSB — RSSI, station/ready flags, RDS block error levels |
| `0x0C`  | `0x5803` | RDSA — RDS block A (or E) raw data |
| `0x0D`  | `0x5804` | RDSB — RDS block B (or E) raw data |
| `0x0E`  | `0x5808` | RDSC — RDS block C (or E) raw data |
| `0x0F`  | `0x5804` | RDSD — RDS block D (or E) raw data |

The `0x0C`–`0x0F` reset values above are the datasheet's literal power-on defaults — they are not meaningful RDS data. Always gate on RDSR (`0x0A` bit 15) before trusting these registers.

### Bit Fields

#### `CTRL` (`0x02`) — reset `0x0000`

| Bits | Name | Description |
|------|------|-------------|
| 15   | DHIZ | Audio output high-Z disable. 0 = high impedance; 1 = normal operation |
| 14   | DMUTE | Mute disable. 0 = mute; 1 = normal operation |
| 13   | MONO | 0 = stereo; 1 = force mono |
| 12   | BASS | 0 = bass boost disabled; 1 = enabled |
| 11   | RCLK_NON_CALIBRATE | 0 = RCLK always supplied; 1 = RCLK not always supplied (restricts operating temperature range — see datasheet) |
| 10   | RCLK_DIRECT_INPUT | 1 = RCLK uses direct input mode |
| 9    | SEEKUP | Seek direction for the next seek. 0 = down; 1 = up |
| 8    | SEEK | Seek enable. Write 1 to start; begins in the direction given by SEEKUP and ends when a channel is found or the whole band has been searched. SEEK reads back 0 and STC reads back 1 when the operation completes |
| 7    | SKMODE | Seek mode. 0 = wrap at band limit and continue seeking; 1 = stop at band limit |
| 6:4  | CLK_MODE | Reference clock: `000`=32.768 kHz, `001`=12 MHz, `101`=24 MHz, `010`=13 MHz, `110`=26 MHz, `011`=19.2 MHz, `111`=38.4 MHz |
| 3    | RDS_EN | 1 = enable RDS/RBDS |
| 2    | NEW_METHOD | 1 = enable new demodulate method (~1 dB better sensitivity) |
| 1    | SOFT_RESET | Write 1 to reset, 0 to run normally |
| 0    | ENABLE | Power-up enable. 0 = disabled; 1 = enabled |

#### `CHAN` (`0x03`) — reset `0x0000`

| Bits | Name | Description |
|------|------|-------------|
| 15:6 | CHAN | Channel select (10-bit). See frequency formula below. Updated to the actual tuned channel after a seek |
| 5    | DIRECT_MODE | Test mode only; leave 0 |
| 4    | TUNE | Write 1 to start a tune to CHAN. STC is set high when complete; the bit self-clears in hardware when the tune completes |
| 3:2  | BAND | `00` = 87–108 MHz (US/Europe); `01` = 76–91 MHz (Japan); `10` = 76–108 MHz (world wide); `11` = 65–76 MHz (East Europe) or 50–65 MHz — selected by R7 bit 9 |
| 1:0  | SPACE | Channel spacing. `00` = 100 kHz; `01` = 200 kHz; `10` = 50 kHz; `11` = 25 kHz |

#### `R4` (`0x04`) — reset `0x0200`

| Bits | Name | Description |
|------|------|-------------|
| 11   | DE   | De-emphasis. 0 = 75 µs (US); 1 = 50 µs (Europe/most of world) |
| 9    | SOFTMUTE_EN | 1 = soft mute enabled (default) — smoothly attenuates audio as signal weakens instead of hard-muting |
| 8    | AFCD | AFC disable. 0 = AFC on (default); 1 = AFC off |

Unlisted bits are reserved; write 0.

#### `R5` (`0x05`) — reset `0x880F`

| Bits | Name | Description |
|------|------|-------------|
| 15   | INT_MODE | 0 = generate a 5 ms interrupt pulse; 1 = interrupt persists until `0x0C` is read (not used by this driver — no interrupt pin is wired in the supported transports) |
| 11:8 | SEEKTH | Seek SNR threshold, 4-bit; default `1000` (~32 dB SNR). Higher = seek is more selective (fewer, stronger stations found) |
| 3:0  | VOLUME | DAC gain, `0000` = mute/min, `1111` = max (default). Logarithmic scale |

#### `R6` (`0x06`) — reset `0x0000`

| Bits  | Name | Description |
|-------|------|-------------|
| 14:13 | OPEN_MODE | `11` = open extended register write access; other values = extended read access only. Not used by this driver — kept at `00` |

#### `R7` (`0x07`) — reset `0x4202`

| Bits  | Name | Description |
|-------|------|-------------|
| 14:10 | TH_SOFRBLEND | Noise soft-blend threshold, unit 2 dB; default `10000` |
| 9     | BAND_65M_50M | Valid only when BAND=`11`. 1 = 65–76 MHz (default); 0 = 50–76 MHz |
| 1     | SOFTBLEND_EN | 1 = soft blend enabled (default) |
| 0     | FREQ_MODE | 1 = use direct frequency offset (`0x08`) instead of CHAN/BAND. Not used by this driver |

#### `STATUSA` (`0x0A`) — reset `0x0400`

| Bits | Name | Description |
|------|------|-------------|
| 15   | RDSR | 1 = a new RDS/RBDS group is ready to read |
| 14   | STC  | Seek/Tune Complete. Set when a seek or tune operation finishes |
| 13   | SF   | Seek Fail. 1 = the last seek did not find a channel above the SEEKTH threshold |
| 12   | RDSS | RDS synchronized (verbose mode only) |
| 11   | BLK_E | 1 = an RDS Block E has been found (verbose mode only) |
| 10   | ST   | Stereo indicator. 0 = mono; 1 = stereo |
| 9:0  | READCHAN | The actually-tuned channel, updated after every tune/seek. Same encoding as CHAN in `0x03` |

#### `STATUSB` (`0x0B`) — reset `0x0000`

| Bits | Name | Description |
|------|------|-------------|
| 15:9 | RSSI | Received signal strength, 7-bit, logarithmic. `0000000` = weakest, `1111111` = strongest. No absolute dBµV mapping is published |
| 8    | FM_TRUE | 1 = the current channel is a real station |
| 7    | FM_READY | 1 = tuner ready |
| 4    | ABCD_E | 1 = registers `0x0C`–`0x0F` hold four copies of Block E (RBDS); 0 = normal Blocks A–D |
| 3:2  | BLERA | Block A (or E) error level (verbose mode only): `00`=0 errors … `11`=uncorrectable |
| 1:0  | BLERB | Block B (or E) error level (verbose mode only), same encoding |

#### `RDSA`/`RDSB`/`RDSC`/`RDSD` (`0x0C`–`0x0F`)

Raw 16-bit RDS blocks A–D (or four copies of block E in RBDS mode when ABCD_E=1). This driver does not decode RDS group content (PI, PS, RadioText, …) — it exposes the four raw blocks and lets the caller interpret them per the RDS/RBDS standard, consistent with how the [Sigrok Decoder](#sigrok-decoder) annotates them.

## Initialization Sequence

1. Build a shadow copy of registers `0x02`–`0x07` in memory, starting from the reset values above.
2. Set `ENABLE`=1, `DHIZ`=1, `DMUTE`=1, `NEW_METHOD`=1 in the `CTRL` shadow register.
3. Set `BAND` and `SPACE` in the `CHAN` shadow register per the requested defaults; compute `CHAN` for the initial frequency (see [Data Conversion](#data-conversion)) and set `TUNE`=1.
4. Set `VOLUME` in the `R5` shadow register.
5. Write all 12 shadow bytes (`0x02`–`0x07`) to the transport in one transaction.
6. Poll `STATUSA` (`0x0A`, 2 bytes) until STC=1 (or a timeout elapses — the datasheet gives no maximum tune time; a few hundred milliseconds is generous in practice).
7. Clear `TUNE` in the local shadow copy of `CHAN` (the bit self-clears in hardware, but the driver's shadow must match, or the next unrelated write will retune to the same channel — see [Implementation Notes](#implementation-notes)).

## Implementation Stages

Each chip is implemented in two stages. The Full class extends Minimal — it inherits everything and adds the rest.

### Minimal

Goal: tune to a station, hear it, adjust volume, and find the next station — no configuration required beyond the transport.

| Operation | Parameters | Returns | Notes |
|-----------|------------|---------|-------|
| `init` | transport, frequency_mhz=100.0, volume=8 | — | Runs the [Initialization Sequence](#initialization-sequence) with sensible defaults |
| `set_frequency` | frequency_mhz: float | — | Tunes to the given frequency (rounded to the nearest valid channel); blocks until STC |
| `frequency` | — | float | Currently tuned frequency in MHz, from READCHAN |
| `set_volume` | level: int (0–15) | — | Writes VOLUME |
| `mute` | enable: bool | — | Clears/sets DMUTE (note: `enable=True` mutes) |
| `seek` | up: bool = True | float or None | Blocking seek in the given direction; returns the new frequency, or `None` if SF (seek fail) is set |

**Sensible defaults:** BAND = world wide (`10`, 76–108 MHz), SPACE = 100 kHz, frequency = 100.0 MHz, volume = 8, DHIZ=1, DMUTE=1, NEW_METHOD=1, SOFTMUTE_EN=1 (chip default), DE=1 (50 µs, most of world), SKMODE=1 (stop at band limit — predictable seek behavior for a first-time user).

### Full

Goal: expose complete chip functionality. Extends Minimal.

| Operation | Parameters | Returns | Notes |
|-----------|------------|---------|-------|
| *(inherits Minimal)* | | | |
| `configure` | band, space, de_emphasis, seek_threshold, seek_mode, clk_mode, afc_disable | — | Writes BAND/SPACE/DE/SEEKTH/SKMODE/CLK_MODE/AFCD; re-tunes to the current frequency if band or space changed (CHAN's meaning depends on both) |
| `set_bass_boost` | enable: bool | — | Writes BASS |
| `set_mono` | enable: bool | — | Writes MONO |
| `set_softmute` | enable: bool | — | Writes SOFTMUTE_EN |
| `enable_rds` | enable: bool | — | Writes RDS_EN |
| `rds_ready` | — | bool | Reads RDSR |
| `read_rds_group` | — | tuple(int,int,int,int) or None | Reads the four raw RDS blocks (A–D) if RDSR is set, else `None` |
| `is_stereo` | — | bool | Reads ST |
| `is_station` | — | bool | Reads FM_TRUE |
| `is_ready` | — | bool | Reads FM_READY |
| `signal_strength` | — | int | Raw RSSI, 0–127 |
| `standby` | enable: bool | — | Clears/sets ENABLE (note: `enable=True` powers down). Waking (`enable=False`) blocks ~250 ms then re-tunes to the last known frequency — see [Timing Constraints](#timing-constraints) |
| `soft_reset` | — | — | Pulses SOFT_RESET, then blocks ~250 ms and re-tunes to the last known frequency — see [Timing Constraints](#timing-constraints) |

**Additional configuration options:** all fields of `CTRL`, `CHAN`, `R4`, `R5`, and the East-Europe/50–65 MHz sub-band select (R7 bit 9) are reachable via `configure`/the setters above. `R6` (OPEN_MODE) and the `FREQ_MODE` direct-frequency path in `R7` are intentionally not exposed — they are undocumented/test features.

**Constants exposed alongside the Full class:**

Band constants: `BAND_US_EUROPE`, `BAND_JAPAN`, `BAND_WORLD`, `BAND_EAST_EUROPE`
Spacing constants: `SPACE_100K`, `SPACE_200K`, `SPACE_50K`, `SPACE_25K`

Each language picks its idiomatic spelling (uppercase module constants in Python, `static constexpr` in C++, named exports in JS, `pub const` in Rust).

## Data Conversion

```
base_khz = { BAND_US_EUROPE: 87000, BAND_JAPAN: 76000, BAND_WORLD: 76000,
             BAND_EAST_EUROPE: 65000 (or 50000 if R7 bit 9 = 0) } [band]
spacing_khz = { SPACE_100K: 100, SPACE_200K: 200, SPACE_50K: 50, SPACE_25K: 25 } [space]

chan = round((frequency_khz - base_khz) / spacing_khz)          # write path (CHAN, 0..1023)
frequency_khz = base_khz + readchan * spacing_khz                # read path (READCHAN)
```

Volume (`VOLUME[3:0]`) and RSSI (`RSSI[6:0]`) are exposed as their raw integer codes — the datasheet documents both as logarithmic but publishes no dB/step or dBµV formula, so this driver does not attempt to convert them to physical units.

An inject node sends `{ command: "seek", direction: "up" }` into the `periph-rda5807m` node every 3 seconds (wired through a manual "next station" button as an alternative trigger). A function node formats the resulting frequency, RSSI, and stereo/mono status into one line; a debug node prints it to the sidebar — the user watches the tuner step across the band, station by station, exactly like pressing "seek" on a real radio.

## Examples

### Demo

**FM band scanner.** Starting at the bottom of the configured band, repeatedly call `seek(up=True)` with `SKMODE`=1 (stop at band limit) until a seek returns `None` (top of band reached). For each station found (`is_station()` true), print frequency, RSSI, and mono/stereo status. If `enable_rds(True)` was called at startup and `rds_ready()` goes true while parked on a station, read a handful of raw RDS groups with `read_rds_group()`, pick out group type `0A`/`0B` (from block B), and assemble the 8-character Program Service (station) name from block D two characters at a time — printing the station name next to its frequency once fully received. The result is a readable log of every station on the band together with its name, similar to what a car radio shows while auto-scanning.

## Timing Constraints

- I²C clock: 0–400 kHz (standard/fast mode).
- The chip resets itself when VIO powers up; a soft reset is also available (`SOFT_RESET` bit). No fixed settling delay is documented beyond that — poll `FM_READY` (`0x0B` bit 7) before issuing the first `TUNE` if paranoid about power-rail sequencing.
- No maximum tune/seek completion time is published. Poll STC with an application-defined timeout; tune typically completes in well under 100 ms, a full-band seek in low single-digit seconds.
- **`FM_READY` settle time (undocumented, measured on real hardware).** After *any* register write, `FM_READY` deasserts and takes up to ~20 ms to reassert — it lags `STC`, which is set first. Code that checks `is_ready()` immediately after a write (mute, volume, configure, ...) should allow a short settle time (this driver's tests use 30 ms) rather than treating a momentary `false` as an error.
- **Standby wake-up and soft reset never reacquire PLL lock on their own (undocumented, measured on real hardware).** After `standby(enable=False)` or `soft_reset()`, `FM_READY` stays `false` indefinitely — confirmed for 3+ seconds — unless the driver issues a fresh `TUNE` afterward, exactly as at initial power-up. Writing the same `CHAN` value again is not enough because `TUNE` is already `0` in the shadow register (no rising edge). Recovery was measured to need ~200 ms after the write that re-enables/resets the chip before a `TUNE` will take effect (180 ms was insufficient, 200 ms was reliable); this driver's `standby`/`soft_reset` wait 250 ms for margin, then call the internal tune-to-current-frequency path (which itself blocks on `STC`), then wait the `FM_READY` settle time above. Callers therefore see `standby(False)`/`soft_reset()` block for roughly a quarter of a second, but `is_ready()` is reliable immediately afterward.

## Implementation Notes

- **No register-pointer byte.** Unlike every other I²C chip in this project, RDA5807M writes always start at `0x02` and reads always start at `0x0A` — the position in the byte stream, not an address byte, selects the register. Drivers use plain `write`/`read` from the I²C transport contract, never `write_read` (there is nothing to write before the read).
- **Shadow registers are mandatory, not an optimization.** Because a write can't target a register directly, changing one field (e.g. volume in `0x05`) means resending `0x02`–`0x05` (or, simplest and what this driver does, always `0x02`–`0x07`). Without a shadow copy of the last-written values, a driver would silently reset every earlier field to `0x0000` on each write.
- **TUNE/SEEK self-clear in hardware but not in your shadow copy.** After STC is set, the chip internally clears `TUNE` (or `SEEK`), but if the shadow register in RAM still has that bit set to 1, the very next unrelated write (e.g. a volume change, which still must resend `0x02`–`0x05`) will retrigger a tune/seek to the same channel. Always clear the local shadow bit after observing STC.
- **CHAN vs READCHAN.** `CHAN` (`0x03` bits 15:6) is the last *commanded* channel; `READCHAN` (`0x0A` bits 9:0) is the chip-reported *actual* channel after a tune or seek completes. After a seek, read `READCHAN` — don't assume `CHAN` was updated (some clones don't echo it back).
- **BAND=`11` is dual-range.** 65–76 MHz vs 50–65 MHz is selected by a bit in a *different* register (`R7` bit 9), not by `CHAN`'s `BAND` field alone.
- **RDS block registers hold garbage at reset.** `0x0C`–`0x0F` power up to non-zero "test pattern" values (`0x5803`, `0x5804`, `0x5808`, `0x5804`) — always gate on `RDSR` before trusting them.
- **Undocumented random-access address `0x11`.** Widely reported by third-party drivers (Linux kernel `radio-rda5807`, several Arduino libraries) as an alternate 7-bit address that allows addressing any single register directly (bypassing the fixed `0x02`/`0x0A` start points), including reading the chip-ID register `0x00`. It is not in the official datasheet, so this driver relies only on the documented `0x10` sequential scheme; a future revision could use `0x11` to add a chip-presence probe.
- **Soft mute is on by default.** `SOFTMUTE_EN`=1 out of reset smoothly reduces volume as the signal weakens rather than producing full-volume noise — expected behavior, not a bug, but worth knowing when a weak station sounds quieter than a strong one at the same `VOLUME` setting.
- **Mute/standby polarity is inverted from what you'd guess.** `DMUTE`=1 means *not* muted (normal operation); `ENABLE`=1 means powered up. The Minimal/Full API above normalizes this so `mute(True)` mutes and `standby(True)` powers down, matching the sign of the method name rather than the raw bit.
- **Track the last known frequency for `standby`/`soft_reset` recovery.** Because those two calls must re-tune internally (see [Timing Constraints](#timing-constraints)), the driver keeps one extra piece of shadow state beyond the register array: the last frequency set via `set_frequency` or found via `seek`. This is separate from `CHAN`, since `CHAN` alone doesn't carry a rising `TUNE` edge.

## Sigrok Decoder

Decoder id `rda5807m`, input `['i2c']`, matches address `0x10`. Because there is no register-pointer byte, the decoder tracks transaction direction (write vs read) and byte offset within the transfer to know which fixed register (`0x02`+offset for writes, `0x0A`+offset for reads) each 16-bit word belongs to. Write transfers annotate CTRL fields (ENABLE, DHIZ, DMUTE, MONO, BASS, SEEK/SEEKUP, RDS_EN), CHAN fields (BAND, SPACE, CHAN, TUNE) with the derived tuned frequency in MHz, and R4/R5/R7 fields (DE, SOFTMUTE_EN, SEEKTH, VOLUME). Read transfers annotate STATUSA (STC, SF, RDSR, ST, READCHAN with derived frequency) and STATUSB (RSSI, FM_TRUE, FM_READY), plus the four raw RDS blocks shown as hex with an ASCII-pair hint for likely PS-name segments.
