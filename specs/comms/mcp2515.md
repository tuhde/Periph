# Chip Spec: MCP2515

**Manufacturer:** Microchip  
**Datasheet:** `datasheets/comms/mcp2515.pdf`  
**Category:** comms  
**Transports:** SPI

## Overview

The MCP2515 is a stand-alone CAN 2.0B controller with an SPI interface. It supports both standard (11-bit) and extended (29-bit) identifiers, data frames of 0–8 bytes, and CAN bus speeds up to 1 Mbit/s. Internally it contains three TX buffers, two RX buffers, six acceptance filters, and two acceptance masks. The device is independent of the host's CAN peripheral — any microcontroller with an SPI port can add CAN connectivity by pairing the MCP2515 with a CAN transceiver such as the MCP2551 or SN65HVD230. Supply voltage is 2.7–5.5 V; current draw is 10 mA active and 5 µA in sleep mode.

## Transport Configuration

### SPI

- **Mode:** CPOL=0 CPHA=0 (Mode 0) or CPOL=1 CPHA=1 (Mode 3)
- **Max clock:** 10 MHz
- **Bit order:** MSB first
- **CS active:** low

## SPI Instruction Set

| Instruction | Command byte | Additional bytes | Description |
|-------------|--------------|------------------|-------------|
| RESET | `0xC0` | — | Software reset; all registers to POR values; enters Configuration mode |
| READ | `0x03`, `addr` | `n` bytes clocked out | Read `n` consecutive registers starting at `addr` |
| READ RX BUFFER | `0x90\|n` (n=0,2,4,6) | bytes clocked out | Fast read: 0→RXB0SIDH, 2→RXB0D0, 4→RXB1SIDH, 6→RXB1D0; clears RXnIF automatically |
| WRITE | `0x02`, `addr` | `n` bytes | Write `n` consecutive registers starting at `addr` |
| LOAD TX BUFFER | `0x40\|n` (n=0,2,4,6,8,10) | bytes | Fast load: 0→TXB0SIDH, 2→TXB0D0, 4→TXB1SIDH, 6→TXB1D0, 8→TXB2SIDH, 10→TXB2D0 |
| RTS | `0x80\|mask` | — | Request-to-send; bits 0–2 trigger TXB0/1/2 respectively; multiple bits may be set simultaneously |
| READ STATUS | `0xA0` | 1 byte clocked out | Returns 8 status bits; byte may be clocked out repeatedly within the same CS assertion |
| RX STATUS | `0xB0` | 1 byte clocked out | Returns received message type and filter match |
| BIT MODIFY | `0x05`, `addr`, `mask`, `data` | — | Atomic read-modify-write; only valid for registers that support it (see §12 of datasheet) |

### Bit Fields

#### `CANCTRL` (`0x0F`) — POR `0x87`

| Bits | Name | Description |
|------|------|-------------|
| 7:5 | REQOP[2:0] | Mode request: `000`=Normal, `001`=Sleep, `010`=Loopback, `011`=Listen-Only, `100`=Configuration |
| 4 | ABAT | Abort all pending TX; cleared by hardware when abort completes |
| 3 | OSM | One-shot mode: 1 = do not retransmit on error or loss of arbitration |
| 2 | CLKEN | Enable CLKOUT pin |
| 1:0 | CLKPRE[1:0] | CLKOUT prescaler: `00`=F_OSC, `01`=F_OSC/2, `10`=F_OSC/4, `11`=F_OSC/8 |

#### `CANSTAT` (`0x0E`) — POR `0x80`

| Bits | Name | Description |
|------|------|-------------|
| 7:5 | OPMOD[2:0] | Current operating mode (same encoding as REQOP) |
| 3:1 | ICOD[2:0] | Interrupt code: `000`=none, `001`=error, `010`=wake-up, `011`=TXB0, `100`=TXB1, `101`=TXB2, `110`=RXB0, `111`=RXB1 |

#### `CNF1` (`0x2A`)

| Bits | Name | Description |
|------|------|-------------|
| 7:6 | SJW[1:0] | Synchronization jump width: SJW = SJW[1:0] + 1 TQ |
| 5:0 | BRP[5:0] | Baud rate prescaler: TQ = 2 × (BRP + 1) / F_OSC |

#### `CNF2` (`0x29`)

| Bits | Name | Description |
|------|------|-------------|
| 7 | BTLMODE | PS2 source: `0`=max(PS1, 2 TQ); `1`=from PHSEG2 in CNF3; always set to 1 |
| 6 | SAM | Triple sampling: 1 = sample 3 times at sample point (recommended for ≤ 125 kbit/s) |
| 5:3 | PHSEG1[2:0] | Phase segment 1: PS1 = PHSEG1 + 1 TQ (1–8 TQ) |
| 2:0 | PRSEG[2:0] | Propagation segment: PropSeg = PRSEG + 1 TQ (1–8 TQ) |

#### `CNF3` (`0x28`)

| Bits | Name | Description |
|------|------|-------------|
| 7 | SOF | Output start-of-frame signal on CLKOUT pin (when CLKEN=1) |
| 6 | WAKFIL | Wake-up filter: 1 = require 3 dominant samples before wake-up |
| 2:0 | PHSEG2[2:0] | Phase segment 2 (when BTLMODE=1): PS2 = PHSEG2 + 1 TQ (1–8 TQ) |

| Bit | Name | Description |
|-----|------|-------------|

## Initialization Sequence

1. Assert CS; issue RESET command (`0xC0`); deassert CS. Wait ≥ 2 µs.
2. Read CANSTAT; verify OPMOD[2:0] = `100` (Configuration mode).
3. Write CNF1, CNF2, CNF3 for the desired bit rate (see Bit Timing section).
4. Write acceptance masks: set RXM0 and RXM1 all-ones (`0xFF` per byte) to require exact filter match, or configure specific masks.
5. Set RXM[1:0]=`11` in RXB0CTRL and RXB1CTRL to accept all messages (bypasses filters), or configure filters 0–5.
6. Set BUKT=1 in RXB0CTRL to enable overflow rollover from RXB0 to RXB1.
7. Write CANINTE for desired interrupts (leave `0x00` for polled operation).
8. Write CANCTRL REQOP[2:0] = `000` to request Normal mode.
9. Poll CANSTAT until OPMOD[2:0] = `000`.

## Bit Timing

The CAN bit period is divided into four non-overlapping segments:

```
TQ = 2 × (BRP + 1) / F_OSC

Bit time = (1 + PropSeg + PS1 + PS2) × TQ

PropSeg = PRSEG[2:0] + 1    (1–8 TQ)
PS1     = PHSEG1[2:0] + 1   (1–8 TQ)
PS2     = PHSEG2[2:0] + 1   (1–8 TQ; requires BTLMODE=1 in CNF2)
SJW     = SJW[1:0] + 1      (1–4 TQ)
```

CAN 2.0B timing constraints:
- PropSeg + PS1 ≥ PS2
- PropSeg + PS1 ≥ 2 TQ (propagation delay budget)
- PS2 ≥ SJW
- Always set BTLMODE=1

## Implementation Stages

### CanFrame type

All platforms define a value type with these fields:

| Field | Type | Description |
|-------|------|-------------|
| `id` | uint32 / int | 11-bit (standard) or 29-bit (extended) CAN identifier |
| `data` | bytes / byte[] / Vec\<u8\> | Payload, 0–8 bytes |
| `extended` | bool | True if 29-bit extended identifier |
| `rtr` | bool | True if remote transmission request frame |

Python: `dataclass CanFrame` or `namedtuple`. C++: `struct CanFrame`. Rust: `struct CanFrame`. JVM: `record CanFrame`. Node.js: plain object.

### Minimal

Goal: send and receive CAN frames with a single, zero-configuration API. Supports standard and extended frames. Uses polled operation only (no INT pin).

| Operation | Parameters | Returns | Notes |
|-----------|------------|---------|-------|
| `init` | transport, bitrate_kbps: int, osc_mhz: int=8 | — | Runs initialization sequence; accepted bitrates: 125, 250, 500, 1000; accepted oscillators: 8, 16 MHz |
| `send` | id: int, data: bytes, extended: bool=False | — | Loads TX buffer and issues RTS; blocks until a TX buffer is free (max 10 ms) or raises on timeout; uses TXB0 |
| `recv` | timeout_ms: int=0 | CanFrame or None | Polls READ STATUS; returns None immediately when timeout_ms=0 and no frame available; blocks up to timeout_ms |

**Sensible defaults:**
- RXM[1:0]=`11` in both RXB0CTRL and RXB1CTRL (accept all messages, bypasses filters)
- BUKT=1 (rollover from RXB0 to RXB1 on overflow)
- CANINTE=0x00 (polled operation; INT pin not used)
- OSM=0 (retransmit on error or loss of arbitration)
- TXB0 is used exclusively for TX; priority TXP=`11` (highest)

### Full

Goal: expose complete chip functionality. Extends Minimal.

| Operation | Parameters | Returns | Notes |
|-----------|------------|---------|-------|
| *(inherits Minimal)* | | | |
| `send_buffered` | id: int, data: bytes, extended: bool=False, buf: int=0 | — | Like `send` but selects TX buffer 0, 1, or 2 explicitly |
| `set_filter` | filter_num: int (0–5), id: int, extended: bool=False | — | Configures acceptance filter; must call `set_mask` for the associated mask |
| `set_mask` | mask_num: int (0–1), mask: int, extended: bool=False | — | Configures acceptance mask; mask_num=0 applies to RXB0 (filters 0–1), mask_num=1 to RXB1 (filters 2–5); must be in Configuration mode |
| `set_rx_mode` | buf: int (0–1), mode: int (0–3) | — | Sets RXM[1:0] in RXBnCTRL: 0=standard filter, 1=extended filter, 3=accept all |
| `set_mode` | mode: str | — | `'normal'`, `'loopback'`, `'listen_only'`, `'sleep'`, `'config'`; polls CANSTAT until confirmed |
| `get_mode` | — | str | Returns current operating mode from CANSTAT OPMOD |
| `reset` | — | — | Issues SPI RESET command; device returns to Configuration mode |
| `read_errors` | — | dict/struct | Returns `{tec, rec, eflg}` from TEC, REC, EFLG registers |
| `clear_overflow` | buf: int (0–1) | — | Clears RX0OVR or RX1OVR flag in EFLG |
| `abort_tx` | — | — | Sets ABAT in CANCTRL; polls until cleared |
| `set_one_shot` | enable: bool | — | Sets/clears OSM in CANCTRL |

**Filter/mask configuration notes:**
- Filters and masks can only be written in Configuration mode; `set_filter`/`set_mask` must enter Config mode and return to the previous mode.
- Mask bit = 1: filter bit must match; mask bit = 0: don't care.
- Filters 0–1 apply to RXB0 with mask 0; filters 2–5 apply to RXB1 with mask 1.
- For standard frames, only SID[10:0] bits participate. For extended frames, EID[28:0] participates.
- RXF0 and RXF1 can additionally filter on the first two data bytes of standard frames when the associated mask bits are set.

## Data Conversion

### ID packing for TX (standard frame)

```
SIDH = id >> 3
SIDL = (id & 0x7) << 5  # SID[2:0] in bits 7:5; EXIDE=0; EID bits ignored
DLC  = len(data) | (0x40 if rtr else 0)
```

### ID packing for TX (extended frame)

```
SIDH = (id >> 21) & 0xFF        # EID[28:21]
SIDL = ((id >> 18) & 0x07) << 5 | 0x08 | ((id >> 16) & 0x03)
       # SID[2:0]=EID[20:18] | EXIDE=1 | EID[17:16]
EID8 = (id >> 8) & 0xFF         # EID[15:8]
EID0 = id & 0xFF                 # EID[7:0]
DLC  = len(data) | (0x40 if rtr else 0)
```

### ID unpacking from RX

```
ide = rxsidl & 0x08   # bit 3
if not ide:            # standard frame
    id = (rxsidh << 3) | (rxsidl >> 5)
    rtr = bool(rxsidl & 0x10)
else:                  # extended frame
    id = (rxsidh << 21) | ((rxsidl >> 5) << 18) | ((rxsidl & 0x03) << 16) | (rxeid8 << 8) | rxeid0
    rtr = bool(rxdlc & 0x40)
data_len = rxdlc & 0x0F
```

## Timing Constraints

- **RESET to Configuration mode:** ≥ 2 µs after the SPI RESET command before issuing further SPI transactions.
- **Mode transition:** after writing CANCTRL REQOP, poll CANSTAT until OPMOD matches; typically < 1 ms.
- **TX completion:** after asserting TXREQ or issuing RTS, TXREQ is cleared when the message has been sent or an error occurred; poll TXBnCTRL. At 125 kbit/s a worst-case 8-byte frame takes ≈ 1 ms.
- **Filter/mask write window:** CNF and filter/mask registers must be written while OPMOD=`100` (Configuration mode) only.
- **Bus-off recovery:** automatic after 128 occurrences of 11 consecutive recessive bits; no host action required.

## Implementation Notes

- The RESET command is the only SPI transaction that requires no preceding command byte acknowledgement — it is self-contained in a single byte.
- BIT MODIFY cannot be used on all registers; consult Table 11-1 of the datasheet for the list of bit-modifiable registers. Using BIT MODIFY on a non-supported register has no effect.
- When BUKT=1, a frame that overflows RXB0 is rolled into RXB1. The RX0OVR flag is not set in this case; only RX1OVR is set if RXB1 is also full.
- TXREQ remains set if the message loses arbitration and OSM=0 (the controller retries). It is cleared only when the message is successfully sent, aborted, or a bus error occurs with OSM=1.
- When reading an RX buffer with the READ RX BUFFER instruction, RXnIF is automatically cleared at the falling edge of CS at the end of the transaction. This is the preferred method — it avoids a separate BIT MODIFY to clear the flag.
- The three TX buffers compete based on priority (TXP[1:0]) and, if equal, by buffer number (TXB0 wins ties).
- In Listen-Only mode the MCP2515 does not acknowledge frames on the bus; it is purely passive. This is useful for bus monitoring without disrupting communication.
- The CLKOUT pin outputs F_OSC/CLKPRE by default (CLKEN=1, CLKPRE=11 at POR); disable it by clearing CLKEN to reduce EMI when the pin is unconnected.
- For reliable operation above 500 kbit/s, ensure the oscillator is ±0.5% or better and keep SPI wiring short (< 10 cm).

## Sigrok Decoder

Decoder id `mcp2515`, input `['spi']`. Each SPI transaction is identified by its first command byte. RESET is annotated as a single byte. READ and WRITE transactions decode the register address to its named register (e.g., `CNF2`, `CANINTF`) and show each data byte with the register name at consecutive addresses. LOAD TX BUFFER and READ RX BUFFER decode the buffer index and offset. RTS decodes which TX buffers (TXB0/1/2) are requested. READ STATUS decodes all eight flags by name. BIT MODIFY decodes register name, mask, and applied data. For TX buffer writes, the decoder reassembles the CAN identifier (standard or extended) from SIDH/SIDL/EID8/EID0 and shows it as a single hex value; DLC and data bytes are shown in a grouped annotation. For RX buffer reads the same reassembly is applied to the received ID.
