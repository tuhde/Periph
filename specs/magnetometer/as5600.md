# Chip Spec: AS5600

**Manufacturer:** AMS OSRAM  
**Datasheet:** `datasheets/magnetometer/as5600.pdf`  
**Category:** `magnetometer`  
**Transports:** I¬≤C

## Overview

The AS5600 is a 12-bit programmable contactless potentiometer and rotary position sensor. Four Hall elements on a 1 mm radius circle detect the perpendicular magnetic field component of a diametrically magnetised on-axis magnet. A CORDIC algorithm computes the 12-bit absolute angle (0‚Äì4095 counts over 0‚Äì360¬∞). An automatic gain control (AGC) loop compensates for temperature drift, airgap variation, and magnet degradation. The output can be read digitally over I¬≤C at any time, or provided as an analog voltage (ratiometric to VDD) or PWM signal on the OUT pin. A programmable start angle (ZPOS), stop angle (MPOS) or maximum angle span (MANG) narrows the output range to any arc ‚â• 18¬∞. All configuration is stored in on-chip OTP non-volatile memory. Three low-power modes and an optional watchdog reduce average current to as low as 1.5 mA. The fixed I¬≤C address is 0x36. Typical applications: contactless potentiometers, knobs, pedals, RC servos, motor feedback encoders, valve position sensing.

## Transport Configuration

### I¬≤C

- **Address:** `0x36` (fixed, cannot be changed)
- **Max clock:** 1 MHz (Fast-mode Plus; also Standard 100 kHz and Fast 400 kHz)
- **Auto-increment:** register address pointer increments automatically after each byte, **except** for special registers ANGLE (0x0E-0x0F), RAW_ANGLE (0x0C-0x0D), and MAGNITUDE (0x1B-0x1C). When the address pointer is set to the high byte of these special registers, it does not increment after reading both bytes ‚Äî allowing the same pair to be re-read without resending the register address.
- **Read protocol:** write slave address + register address (sets pointer), then repeated-start + slave address + R to read bytes.
- **Write protocol:** write slave address + register address + data bytes.
- **Invalid address behavior:** invalid addresses are NACK'd on the address byte, but the pointer still increments. Reads from invalid addresses return 0x00.

### Bit Fields

#### `ZMCO` (`0x00`)

| Bits | Name | Description |
|------|------|-------------|
| 7:2 | reserved | |
| 1:0 | ZMCO | Number of times ZPOS/MPOS have been permanently burned (0‚Äì3). Read before Burn_Angle to verify remaining writes. |

#### `STATUS` (`0x0B`)

| Bits | Name | Description |
|------|------|-------------|
| 7:6 | reserved | |
| 5 | MH | AGC minimum gain overflow ‚Äî magnet too strong (Bz > 90 mT) |
| 4 | ML | AGC maximum gain overflow ‚Äî magnet too weak (Bz < 30 mT) |
| 3 | MD | Magnet detected (Bz ‚â• 8 mT); output data is valid only when MD=1 |
| 2:0 | reserved | |

#### `CONF_H` (`0x07`) and `CONF_L` (`0x08`)

The CONF register is 14 bits split across two bytes. Burst-read or burst-write both addresses together.

| Register | Bits | Name | Description |
|----------|------|------|-------------|
| 0x07 | 7:6 | reserved | Preserve as read; factory-set |
| 0x07 | 5 | WD | Watchdog: `0`=off, `1`=auto-switch to LPM3 when angle stable ‚â§4 LSB for ‚â•1 minute |
| 0x07 | 4:2 | FTH[2:0] | Fast Filter Threshold (see table below) |
| 0x07 | 1:0 | SF[1:0] | Slow Filter: `00`=16√ó (2.2 ms settle), `01`=8√ó (1.1 ms), `10`=4√ó (0.55 ms), `11`=2√ó (0.286 ms) |
| 0x08 | 7:6 | PWMF[1:0] | PWM frequency: `00`=115 Hz, `01`=230 Hz, `10`=460 Hz, `11`=920 Hz |
| 0x08 | 5:4 | OUTS[1:0] | Output stage: `00`=analog 0‚ÄìVDD, `01`=analog 10‚Äì90% VDD, `10`=digital PWM |
| 0x08 | 3:2 | HYST[1:0] | Hysteresis: `00`=off, `01`=1 LSB, `10`=2 LSBs, `11`=3 LSBs |
| 0x08 | 1:0 | PM[1:0] | Power mode: `00`=NOM (6.5 mA), `01`=LPM1 (3.4 mA, 5 ms poll), `10`=LPM2 (1.8 mA, 20 ms), `11`=LPM3 (1.5 mA, 100 ms) |

**Fast Filter Threshold (FTH) ‚Äî threshold in LSBs above which the fast filter is engaged:**

| FTH | Slow‚ÜíFast threshold (LSBs) | Fast‚ÜíSlow threshold (LSBs) |
|-----|---------------------------|---------------------------|
| `000` | slow filter only | ‚Äî |
| `001` | 6 | 1 |
| `010` | 7 | 1 |
| `011` | 9 | 1 |
| `100` | 18 | 2 |
| `101` | 21 | 2 |
| `110` | 24 | 2 |
| `111` | 10 | 4 |

#### `BURN` (`0xFF`)

Write-only. Two distinct commands:
- `0x80` ‚Äî **Burn_Angle**: permanently burns ZPOS and MPOS to OTP. Can be executed up to 3 times (ZMCO tracks count). Requires MD=1. After burning, write 0x01, 0x11, 0x10 sequentially to 0xFF to reload and verify OTP.
- `0x40` ‚Äî **Burn_Setting**: permanently burns MANG and CONF to OTP. Can be executed only once, and only when ZMCO=0 (ZPOS/MPOS never burned). After burning, verify as above.

## Initialization Sequence

1. Wait ‚â§10 ms after power-up before first I¬≤C access.
2. Write register address `0x0B` to set the address pointer.
3. Read 1 byte from STATUS; verify MD=1 (bit 3). If MD=0, the magnet is absent or too far ‚Äî output data is invalid.
4. Read ANGLE (2 bytes from 0x0E) for the first angle measurement.

No register writes are required for basic angle readout; the factory reset configuration (CONF=0x0000) is suitable for normal mode, 16√ó slow filter, analog output.

## Implementation Stages

Each chip is implemented in two stages. The Full class extends Minimal ‚Äî it inherits everything and adds the rest.

### Minimal

Goal: Read the absolute angle in degrees with no configuration required beyond the transport.

| Operation | Parameters | Returns | Notes |
|-----------|------------|---------|-------|
| `init` | transport | ‚Äî | Reads STATUS once to verify magnet; raises if MD=0 |
| `angle` | ‚Äî | float | Angle in degrees, 0.0‚Äì360.0 (exclusive); reads ANGLE register |
| `angle_raw` | ‚Äî | int | Raw 12-bit angle count, 0‚Äì4095; reads ANGLE register |
| `is_magnet_detected` | ‚Äî | bool | STATUS.MD; returns False if magnet removed |
| `is_magnet_too_strong` | ‚Äî | bool | STATUS.MH |
| `is_magnet_too_weak` | ‚Äî | bool | STATUS.ML |

**Sensible defaults baked into Minimal:**
- No CONF writes ‚Äî uses power-on default CONF=0x0000 (NOM mode, hysteresis off, analog output, SF=16√ó)
- Reads ANGLE (0x0E-0x0F), not RAW_ANGLE, so any OTP-programmed ZPOS/MPOS range is respected

`init` reads STATUS; if MD=0 it raises an error (or equivalent platform exception). This prevents silently returning garbage when no magnet is placed.

### Full

Goal: Expose complete chip functionality. Extends Minimal.

| Operation | Parameters | Returns | Notes |
|-----------|------------|---------|-------|
| *(inherits Minimal)* | | | |
| `raw_angle` | ‚Äî | int | Unscaled RAW_ANGLE, 0‚Äì4095 for 0‚Äì360¬∞; unaffected by ZPOS/MPOS |
| `raw_angle_degrees` | ‚Äî | float | RAW_ANGLE converted to degrees |
| `agc` | ‚Äî | int | AGC value (0‚Äì255 for 5 V; 0‚Äì127 for 3.3 V); mid-range = optimal airgap |
| `magnitude` | ‚Äî | int | 12-bit CORDIC magnitude value |
| `status_byte` | ‚Äî | int | Raw STATUS register byte |
| `configure` | pm: int=0, hyst: int=0, outs: int=0, pwmf: int=0, sf: int=0, fth: int=0, wd: bool=False | ‚Äî | Write CONF_H and CONF_L; preserves reserved bits |
| `set_zero_position` | pos: int | ‚Äî | Write ZPOS (0‚Äì4095); volatile ‚Äî lost on power cycle unless burned |
| `set_max_position` | pos: int | ‚Äî | Write MPOS (0‚Äì4095); volatile |
| `set_max_angle` | span: int | ‚Äî | Write MANG (0‚Äì4095); volatile; span must correspond to ‚â•18¬∞ |
| `zero_position` | ‚Äî | int | Read ZPOS |
| `max_position` | ‚Äî | int | Read MPOS |
| `max_angle` | ‚Äî | int | Read MANG |
| `burn_count` | ‚Äî | int | Read ZMCO (0‚Äì3); number of permanent ZPOS/MPOS writes remaining = 3 ‚àí ZMCO |
| `burn_angle` | ‚Äî | ‚Äî | Write 0x80 to BURN; permanently stores ZPOS+MPOS; requires MD=1 and ZMCO<3 |
| `burn_setting` | ‚Äî | ‚Äî | Write 0x40 to BURN; permanently stores MANG+CONF; requires ZMCO=0 |

**Notes on Full methods:**

`configure()` reads the current CONF_H/CONF_L values before writing to preserve the reserved bits in CONF_H[7:6].

`set_zero_position()` and `set_max_position()` write to volatile RAM registers only. The new range takes effect ‚â•1 ms after the write. To make the range permanent, call `burn_angle()` afterward.

`burn_angle()` must precondition-check MD=1 (magnet present) before issuing the command; if MD=0, raise an error. After issuing the command, wait ‚â•1 ms, then verify by writing 0x01, 0x11, 0x10 to 0xFF and re-reading ZPOS/MPOS.

`burn_setting()` must check ZMCO=0 before issuing ‚Äî MANG/CONF can only be burned once, and only before any ZPOS/MPOS burn.

## Data Conversion

### Angle

```
angle_deg = raw_count * 360.0 / 4096
```

ANGLE register output is always 0‚Äì4095, representing the programmed angular range. With default ZPOS=0, MPOS=0 (full 360¬∞), this gives 0.0‚Äì<360.0¬∞. The ANGLE register applies a 10-LSB hysteresis at the 0/360¬∞ wrap point to avoid toggling.

RAW_ANGLE gives the unscaled physical angle, also 0‚Äì4095 for 0‚Äì360¬∞, without any ZPOS/MPOS scaling.

### Programmed angular range

```
# Setting a 180¬∞ range starting at the current position:
# 1. Read RAW_ANGLE to get current position
# 2. Write current raw value to ZPOS
# 3. Write (current + 180/360 * 4096) % 4096 to MPOS
# Alternatively, use MANG = round(180/360 * 4096) = 2048
```

### AGC interpretation

AGC mid-range = optimal airgap. In 5 V mode, target AGC ‚âà 128. In 3.3 V mode, target AGC ‚âà 64. Values near 0 = magnet too close/strong; values near max = magnet too far/weak.

## Timing Constraints

- Power-up to first I¬≤C access: ‚â§10 ms
- Sampling rate: ‚â§150 ¬µs (maximum update rate)
- Settling times (angle step response, slow filter only): SF=00 ‚Üí 2.2 ms, SF=01 ‚Üí 1.1 ms, SF=10 ‚Üí 0.55 ms, SF=11 ‚Üí 0.286 ms
- Register write to output taking effect: ‚â•1 ms after any CONF/ZPOS/MPOS/MANG write
- Burn command to OTP completion: ‚â•1 ms after write to BURN
- OTP verification: after burn, write 0x01, 0x11, 0x10 sequentially to 0xFF to reload OTP content before verifying registers
- Watchdog detection time: ~1 minute (¬±5%) of angle stability within 4 LSBs before automatic switch to LPM3
- Low-power polling intervals: LPM1 = 5 ms, LPM2 = 20 ms, LPM3 = 100 ms

## Implementation Notes

- **No internal identification register:** The AS5600 has no chip ID register. Presence is confirmed by checking STATUS.MD=1 with a magnet in place. When the bus address 0x36 is present but no magnet is placed, STATUS.MD=0 ‚Äî `init()` must reject this state.
- **Fixed I¬≤C address ‚Äî multi-sensor:** Only one AS5600 per I¬≤C bus is possible. Multiple sensors on the same bus require an I¬≤C multiplexer (e.g. TCA9548A).
- **CONF register must be read before write:** CONF_H bits 7:6 are reserved and may contain factory values. Always read CONF_H/CONF_L first, mask in the new values, then write back. Never write `0x00` for the reserved bits without reading first.
- **ZPOS=0 + MPOS=0 means full 360¬∞:** The reset values ZPOS=0, MPOS=0 are interpreted as a full 360¬∞ range, not a zero-width range. This is a special case ‚Äî do not write both to 0x000 if intending a narrow range; use MANG instead.
- **MANG vs MPOS:** These serve different purposes. MPOS specifies a stop angle in absolute raw counts. MANG specifies the span in raw counts, starting from ZPOS. MANG can only be burned if ZMCO=0. Prefer MANG for configuring a fixed-width window (e.g. 180¬∞ always) regardless of start position. Prefer MPOS for applications where both start and stop matter.
- **BURN_ANGLE limit ‚Äî 3 total writes:** ZMCO tracks the cumulative burn count. Once ZMCO=3, ZPOS and MPOS are permanently locked; BURN_ANGLE is silently ignored (or may be rejected). Read ZMCO before any burn.
- **BURN_SETTING can only be written once:** MANG and CONF OTP can only be permanently programmed once, and only when ZMCO=0 (ZPOS/MPOS never burned). Attempting BURN_SETTING after any ZPOS/MPOS burn has no effect.
- **Burn requires MD=1:** BURN_ANGLE will not execute if no magnet is detected (MD=0). The driver must verify STATUS.MD=1 before issuing the burn command.
- **ANGLE register hysteresis at 360¬∞:** The ANGLE register applies a ¬±10 LSB hysteresis at the 0/4096 boundary to avoid output toggling when the magnet is near 0¬∞ or 360¬∞. This means the output can wrap slightly early or late by up to ~0.88¬∞. RAW_ANGLE has no such hysteresis.
- **Auto-increment suppression for repeat reads:** Setting the address pointer to 0x0C (RAW_ANGLE_H), 0x0E (ANGLE_H), or 0x1B (MAGNITUDE_H) and then continuously reading byte pairs returns fresh data each time without resending the register address. Only effective when starting at the high byte.
- **AGC range differs by supply voltage:** In 5 V mode, AGC is 0‚Äì255; in 3.3 V mode, AGC is 0‚Äì127 (hardware difference). The driver cannot auto-detect this; document it as a constructor parameter or note.
- **Stray field rejection:** The AS5600 uses differential Hall sensing to reject homogeneous external stray fields. Non-homogeneous fields (e.g. nearby motors or ferromagnetic objects) can still cause errors; MH/ML bits indicate when the AGC is hitting its limits.
- **Minimum angular range:** ZPOS to MPOS or MANG must span at least 18¬∞ (= 204 raw counts). Narrower ranges produce undefined output.
