# Chip Spec: HMC5883L

**Manufacturer:** Honeywell  
**Datasheet:** `datasheets/magnetometer/HMC5883L.pdf`  
**Category:** `magnetometer`  
**Transports:** I²C

## Overview

The HMC5883L is a surface-mount 3-axis anisotropic magnetoresistive (AMR) sensor with integrated ASIC, 12-bit ADC, automatic degaussing strap drivers, and offset cancellation. It measures Earth's magnetic field on three axes with 1°–2° compass heading accuracy. A single fixed I²C address (0x1E) and a simple register model — three configuration registers, six output registers, one status register — make it easy to integrate. Two operating modes (continuous and single-shot) enable flexible power trade-offs: continuous draws ~100 µA at 7.5 Hz with no averaging; idle draws only ~2 µA. Typical applications: electronic compass, navigation, gaming, hand-held position sensing.

## Transport Configuration

### I²C

- **Address:** `0x1E` (fixed, 7-bit); write byte `0x3C`, read byte `0x3D`  
- **Max clock:** 400 kHz (fast mode); 3400 kHz (high-speed mode, enabled via HS bit in Mode Register)  
- **Register auto-increment:** The register pointer auto-increments after each byte read. A burst read of 6 bytes starting at register 0x03 reads all six output registers in one transaction.

### Bit Fields

#### `Configuration Register A` (`0x00`)

| Bits | Name      | Description |
|------|-----------|-------------|
| 7    | CRA7      | Reserved. Must be cleared (0). |
| 6:5  | MA[1:0]   | Samples averaged per output: `00`=1 (default); `01`=2; `10`=4; `11`=8 |
| 4:2  | DO[2:0]   | Data output rate (continuous mode only, see ODR table) |
| 1:0  | MS[1:0]   | Measurement configuration: `00`=normal (default); `01`=positive bias; `10`=negative bias; `11`=reserved |

**ODR (DO[2:0]) — continuous mode:**

| DO[2:0] | ODR (Hz) |
|---------|---------|
| `000`   | 0.75 |
| `001`   | 1.5 |
| `010`   | 3 |
| `011`   | 7.5 |
| `100`   | 15 (default) |
| `101`   | 30 |
| `110`   | 75 |
| `111`   | Reserved |

#### `Configuration Register B` (`0x01`)

| Bits | Name      | Description |
|------|-----------|-------------|
| 7:5  | GN[2:0]   | Gain configuration (see gain table) |
| 4:0  | —         | Must be cleared for correct operation |

**Gain settings:**

| GN[2:0] | Range  | Gain (LSb/Gauss) | Resolution (mG/LSb) |
|---------|--------|-----------------|---------------------|
| `000`   | ±0.88 Ga | 1370          | 0.73 |
| `001`   | ±1.3 Ga  | 1090 (default)| 0.92 |
| `010`   | ±1.9 Ga  | 820           | 1.22 |
| `011`   | ±2.5 Ga  | 660           | 1.52 |
| `100`   | ±4.0 Ga  | 440           | 2.27 |
| `101`   | ±4.7 Ga  | 390           | 2.56 |
| `110`   | ±5.6 Ga  | 330           | 3.03 |
| `111`   | ±8.1 Ga  | 230           | 4.35 |

#### `Mode Register` (`0x02`)

| Bits | Name      | Description |
|------|-----------|-------------|
| 7    | HS        | High-speed I²C: `0`=normal; `1`=3400 kHz mode |
| 6:2  | —         | Reserved |
| 1:0  | MD[1:0]   | Operating mode: `00`=continuous; `01`=single-measurement (default); `10`/`11`=idle |

#### `Status Register` (`0x09`)

| Bits | Name  | Description |
|------|-------|-------------|
| 7:2  | —     | Reserved |
| 1    | LOCK  | Output register lock: set when some but not all output bytes have been read, or when mode register has been read; cleared when all 6 output bytes are read |
| 0    | RDY   | Data ready: set when all 6 output bytes contain a valid measurement; cleared when device starts writing new data (for 250 µs) |

## Initialization Sequence

### Minimal (continuous mode)

1. Write `0x00 0x70`: Config A — 8 averages, 15 Hz ODR, normal measurement  
   (`0x70` = `0b01110000`: CRA7=0, MA=11, DO=100, MS=00)
2. Write `0x01 0x20`: Config B — gain=1 (±1.3 Ga, 1090 LSb/Ga, default)  
3. Write `0x02 0x00`: Mode Register — continuous measurement  
4. Wait **6 ms** for first measurement (one measurement period at 15 Hz ≈ 67 ms; first data is available in ~6 ms after command)

### Full (single-shot mode, per measurement)

1. Write `0x00 <CRA>`: configure averaging, ODR, measurement config  
2. Write `0x01 <CRB>`: configure gain  
3. For each reading: write `0x02 0x01` (single-measurement mode); wait 6 ms or poll RDY; read 6 bytes from 0x03

## Implementation Stages

### Minimal

Goal: continuous magnetic field reading on all three axes with no configuration required.

| Operation         | Parameters  | Returns                          | Notes |
|-------------------|-------------|----------------------------------|-------|
| `__init__`        | `transport` | —                                | Runs init sequence; defaults: 8 averages, 15 Hz ODR, ±1.3 Ga gain, continuous mode |

- All 7 ODR values (0.75–75 Hz)

## Data Conversion

### Magnetic field

```
# IMPORTANT: output register byte order is X, Z, Y (not X, Y, Z)
# Burst read 6 bytes from 0x03: bytes [0:1]=X, [2:3]=Z, [4:5]=Y
raw_x = int.from_bytes(buf[0:2], 'big', signed=True)
raw_z = int.from_bytes(buf[2:4], 'big', signed=True)
raw_y = int.from_bytes(buf[4:6], 'big', signed=True)

GAIN = {0: 1370, 1: 1090, 2: 820, 3: 660, 4: 440, 5: 390, 6: 330, 7: 230}  # LSb/Gauss

# Overflow sentinel: −4096; return None for that axis
def to_tesla(raw, gain_lsb_per_gauss):
    if raw == -4096:
        return None
    return (raw / gain_lsb_per_gauss) * 1e-4  # Gauss → Tesla
```

Gain tolerance: ±5% across all settings. Sensitivity temperature coefficient: −0.3 %/°C.

## Timing Constraints

- **Power-on analog stabilization:** 50 ms after applying VDD before analog circuits are ready for accurate measurements.
- **I²C ready:** 200 µs after power-on, I²C commands are accepted.
- **Measurement period:** ~6 ms from command/trigger to data ready (typical).
- **Continuous mode first data:** After switching to continuous mode, the first valid data set is available after 2 / f_DO periods (e.g., 133 ms at 15 Hz); subsequent measurements arrive every 1 / f_DO.
- **Single-shot mode:** Write MD=01, then poll RDY or wait ≥ 6 ms before reading.
- **Burst read all 6 bytes:** All six output registers must be read before new data can be placed in them. Partial reads lock the output registers (LOCK bit set). Always read all 6 bytes in one transaction.
- **Gain change latency:** A new gain setting takes effect from the second measurement after the Config B write. The first measurement after a gain change still uses the old gain.

## Implementation Notes

- **X, Z, Y byte order:** The output register burst order is X MSB/LSB, Z MSB/LSB, Y MSB/LSB (registers 0x03–0x08). Do not assume X, Y, Z order.
- **Fixed I²C address:** The HMC5883L has no address selection pin; only one device can be on a bus.
- **Overflow detection:** A raw value of −4096 (0xF800) indicates ADC overflow or underflow. The driver must detect this sentinel and return a distinct indicator (e.g., `None`/`NaN`/`float('nan')`), not convert it to a nonsense field value.
- **CRA7 must be 0:** Bit 7 of Config A is reserved and must always be written 0.
- **Config B lower bits must be 0:** Bits 4:0 of Config B must be cleared. Write only GN[2:0] in bits 7:5 and zero all other bits.
- **LOCK bit:** If LOCK is set (partial read occurred), all 6 output bytes must be read to unlock the registers. In the driver, always read 6 bytes starting at 0x03.
- **Single-shot auto-idles:** After a single-shot measurement completes, the Mode Register automatically returns to idle (MD=01). The driver must re-write MD=01 for each new single-shot measurement.

## Sigrok Decoder

The sigrok decoder for the HMC5883L annotates I²C bus transactions (decoder id: `i2c`) at device address `0x1E`. It decodes register writes (Config A: averaging, ODR, measurement mode; Config B: gain; Mode Register: operating mode) with human-readable summaries. For burst reads starting at register 0x03, it assembles the three 16-bit two's complement values in the correct X/Z/Y order, detects overflow (−4096), and annotates the converted field in microtesla at the configured gain. The Status Register RDY and LOCK bits are annotated when read.
