# Chip Spec: INA226

**Manufacturer:** Texas Instruments  
**Datasheet:** `datasheets/power/ina226.pdf`  
**Category:** `power`  
**Transports:** I²C, SMBus

## Overview

The INA226 is a 36V, 16-bit current, voltage, and power monitor. It measures the differential shunt voltage across IN+ and IN− to derive current, and the bus voltage at VBUS. An internal multiplier calculates power. Current and power readings require the Calibration Register to be programmed with the shunt resistor value and desired current LSB. The device operates from 2.7–5.5 V and supports up to 16 I²C addresses via A0/A1 pins.

## Transport Configuration

### I²C / SMBus

- **Addresses:** `0x40`–`0x4F` (16 addresses, set by A0 and A1 pins)

| A1  | A0  | Address |
|-----|-----|---------|
| GND | GND | `0x40`  |
| GND | VS  | `0x41`  |
| GND | SDA | `0x42`  |
| GND | SCL | `0x43`  |
| VS  | GND | `0x44`  |
| VS  | VS  | `0x45`  |
| VS  | SDA | `0x46`  |
| VS  | SCL | `0x47`  |
| SDA | GND | `0x48`  |
| SDA | VS  | `0x49`  |
| SDA | SDA | `0x4A`  |
| SDA | SCL | `0x4B`  |

- **Max clock:** 400 kHz (fast mode), 2.94 MHz (high-speed mode)
- **SMBus timeout:** 28 ms — the interface resets if SCL is held low longer than this

All registers are 16-bit, big-endian (MSB first). Read/write is via a register pointer byte sent before data.

| Bit | Name | Description |
|-----|------|-------------|
| 15  | SOL  | Alert on shunt voltage over-limit |
| 14  | SUL  | Alert on shunt voltage under-limit |
| 13  | BOL  | Alert on bus voltage over-limit |
| 12  | BUL  | Alert on bus voltage under-limit |

## Initialization Sequence

### Minimal

1. Write Calibration Register (`0x05`) — required for current and power readings:
   ```
   Current_LSB = max_current / 32768
   CAL = int(0.00512 / (Current_LSB × r_shunt))
   ```
2. Device is already in continuous shunt+bus mode at power-on; no further configuration needed.

### Full

1. Write Configuration Register (`0x00`) with desired averaging, conversion times, and mode.
2. Write Calibration Register (`0x05`) as above.
3. Optionally configure Mask/Enable (`0x06`) and Alert Limit (`0x07`) registers.

## Implementation Stages

### Minimal

Goal: read bus voltage, shunt voltage, current, and power with no configuration beyond the transport and shunt resistor.

Constructor parameters: `transport`, `r_shunt` (Ω), `max_current` (A).

Constructor computes and writes the Calibration Register automatically.

**Baked-in defaults (written to Configuration Register at init):**
- MODE = 111 (shunt + bus, continuous)
- VBUSCT = 100 (1.1 ms)
- VSHCT = 100 (1.1 ms)
- AVG = 000 (1 sample, no averaging)

| Operation        | Parameters | Returns | Notes |
|------------------|------------|---------|-------|
| `voltage()`      | —          | float   | Bus voltage in V |
| `shunt_voltage()`| —          | float   | Shunt voltage in V |
| `current()`      | —          | float   | Current in A (signed) |
| `power()`        | —          | float   | Power in W |

### Full

Extends Minimal. Adds full register access.

| Operation             | Parameters                              | Returns | Notes |
|-----------------------|-----------------------------------------|---------|-------|
| *(inherits Minimal)*  | | | |
| `configure()`         | `avg`, `vbus_ct`, `vsh_ct`, `mode`      | —       | Write Configuration Register |
| `conversion_ready()`  | —                                       | bool    | Read CVRF bit from Mask/Enable |
| `overflow()`          | —                                       | bool    | Read OVF bit from Mask/Enable |
| `set_alert()`         | `function`, `limit`, `polarity`, `latch`| —       | Write Mask/Enable + Alert Limit |
| `alert_flags()`       | —                                       | int     | Read Mask/Enable register |
| `reset()`             | —                                       | —       | Set RST bit; re-writes calibration afterward |
| `shutdown()`          | —                                       | —       | Set MODE = 000 |
| `wake()`              | —                                       | —       | Restore previous MODE |
| `manufacturer_id()`   | —                                       | int     | Read `0xFE`; expect `0x5449` |
| `die_id()`            | —                                       | int     | Read `0xFF`; expect `0x2260` |

**Alert function values for `set_alert(function, ...)`:**

| Value | Function |
|-------|----------|
| `SOL` | Shunt voltage over-limit |
| `SUL` | Shunt voltage under-limit |
| `BOL` | Bus voltage over-limit |
| `BUL` | Bus voltage under-limit |
| `POL` | Power over-limit |
| `CNVR`| Conversion ready |

The `limit` parameter for `set_alert` is in natural units (V for voltage functions, W for power).

## Data Conversion

```
Current_LSB = max_current / 32768            # A/bit
CAL         = int(0.00512 / (Current_LSB × r_shunt))

shunt_voltage (V) = raw_s16 × 2.5e-6        # raw is signed 16-bit
bus_voltage   (V) = raw_u16 × 1.25e-3       # raw is unsigned 16-bit
current       (A) = raw_s16 × Current_LSB   # raw is signed 16-bit
power         (W) = raw_u16 × 25 × Current_LSB
```

For `set_alert` limit conversion:
```
# Shunt voltage limit (SOL/SUL):
alert_raw = int(limit_V / 2.5e-6)

# Bus voltage limit (BOL/BUL):
alert_raw = int(limit_V / 1.25e-3)

# Power limit (POL):
alert_raw = int(limit_W / (25 × Current_LSB))
```

## Timing Constraints

- Recovery from power-down mode: 40 µs
- All registers are volatile — reprogrammed on every power-up
- Conversion Ready Flag (CVRF) clears on write to Configuration Register or on read of Mask/Enable Register

## Implementation Notes

- Current Register (`0x04`) and Power Register (`0x03`) read as zero until Calibration Register is programmed with a non-zero value
- Current Register is signed (2's complement); Power Register is unsigned
- Only one alert function can be active at a time; if multiple bits are set in Mask/Enable, the highest-priority bit (MSB) wins
- AFF bit clears automatically in transparent mode (LEN=0) once the fault clears; in latch mode (LEN=1) it clears only when Mask/Enable is read
- Reading Mask/Enable Register clears the CVRF bit — read it last if checking both flags
- The register pointer persists between transactions; consecutive reads of the same register do not need to re-send the pointer
