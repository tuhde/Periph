# Chip Spec: INA3221

**Manufacturer:** Texas Instruments  
**Datasheet:** `datasheets/power/ina3221.pdf`  
**Category:** `power`  
**Transports:** I²C, SMBus

## Overview

The INA3221 is a three-channel, 26 V, 13-bit current and bus-voltage monitor. Each channel measures its own differential shunt voltage (between IN+_n and IN−_n, ±163.8 mV full-scale, 40 µV LSB) and its own bus voltage (at IN−_n referenced to GND, 32.76 V full-scale, 8 mV LSB; the input is rated to 26 V). Programmable averaging and conversion times, four alert pins (Critical, Warning, Power-Valid, Timing-Control), and a shunt-voltage sum function let the device monitor up to three rails simultaneously without host polling. The device operates from 2.7–5.5 V and supports four I²C addresses via a single A0 pin.

The INA3221 is a three-channel **sibling** of the INA226 — same conversion-time encoding, same MODE encoding, same Manufacturer ID register. But the INA3221 does **not** have a Calibration register, a Current register, or a Power register; those values are computed in software from the shunt voltage and the user-supplied shunt resistance. The INA3221 also has a much richer alert engine (per-channel critical + warning, summation, power-valid, timing-control). Driver implementations of the two chips are siblings, not parent/child.

## Transport Configuration

### I²C / SMBus

- **Addresses:** `0x40`–`0x43` (4 addresses, set by single A0 pin)

| A0  | Address |
|-----|---------|
| GND | `0x40`  |
| VS  | `0x41`  |
| SDA | `0x42`  |
| SCL | `0x43`  |

- **Max clock:** 400 kHz (fast mode), 2.44 MHz (high-speed mode)
- **SMBus timeout:** 28 ms — the interface resets if SCL is held low longer than this

All registers are 16-bit, big-endian (MSB first). Read/write is via a register pointer byte sent before data. The pointer persists between transactions; consecutive reads of the same register do not need to re-send the pointer.

### Bit Fields

#### `Configuration` (`0x00`) — reset `0x7127`

| Bits  | Name              | Description |
|-------|-------------------|-------------|
| 15    | RST               | Write 1 to reset all registers; self-clears |
| 14    | CH1en             | Channel 1 enable (1 = enabled, default) |
| 13    | CH2en             | Channel 2 enable (1 = enabled, default) |
| 12    | CH3en             | Channel 3 enable (1 = enabled, default) |
| 11:9  | AVG               | Averaging count |
| 8:6   | VBUSCT            | Bus voltage conversion time |
| 5:3   | VSHCT             | Shunt voltage conversion time |
| 2:0   | MODE              | Operating mode |

Disabled channels are bypassed in the measurement sequence, regardless of MODE.

**AVG — number of averages:**

| AVG | Samples |
|-----|---------|
| 000 | 1 (default) |
| 001 | 4 |
| 010 | 16 |
| 011 | 64 |
| 100 | 128 |
| 101 | 256 |
| 110 | 512 |
| 111 | 1024 |

**VBUSCT / VSHCT — conversion time (same encoding):**

| Value | Time |
|-------|------|
| 000   | 140 µs |
| 001   | 204 µs |
| 010   | 332 µs |
| 011   | 588 µs |
| 100   | 1.1 ms (default) |
| 101   | 2.116 ms |
| 110   | 4.156 ms |
| 111   | 8.244 ms |

**MODE:**

| Value | Mode |
|-------|------|
| 000   | Power-down |
| 001   | Shunt voltage, single-shot (triggered) |
| 010   | Bus voltage, single-shot (triggered) |
| 011   | Shunt and bus, single-shot (triggered) |
| 100   | Power-down |
| 101   | Shunt voltage, continuous |
| 110   | Bus voltage, continuous |
| 111   | Shunt and bus, continuous (default) |

Writing any triggered-mode value (001/010/011) re-triggers a single-shot conversion, even if MODE already held that value.

#### Shunt Voltage Registers (`0x01`, `0x03`, `0x05`)

13-bit signed value, **left-aligned** in a 16-bit word. Bits 2:0 are reserved (read 0). Right-shift by 3 (with sign extension) to get the 13-bit signed count; LSB = 40 µV. Full-scale = ±163.8 mV (decimal `0x7FF8` / `0x8008` in raw register form).

| Bits  | Name      | Description |
|-------|-----------|-------------|
| 15    | SIGN      | Sign bit (2's complement) |
| 14:3  | SD11:SD0  | Shunt voltage data |
| 2:0   | —         | Reserved |

#### Bus Voltage Registers (`0x02`, `0x04`, `0x06`)

12-bit unsigned value, left-aligned. Bits 2:0 are reserved (read 0). Right-shift by 3 to get the count; LSB = 8 mV. Full-scale register value = 32.76 V, but the **input is rated to 26 V** — do not exceed.

| Bits  | Name      | Description |
|-------|-----------|-------------|
| 15    | SIGN      | Sign bit (always 0 for valid bus reading) |
| 14:3  | BD11:BD0  | Bus voltage data |
| 2:0   | —         | Reserved |

#### Critical / Warning Alert Limit Registers (`0x07`–`0x0C`)

Per-channel shunt-voltage thresholds. Same 13-bit-left-aligned signed format as the shunt voltage registers (LSB = 40 µV). Critical limits compare against single-conversion shunt readings; Warning limits compare against the **averaged** shunt reading (after AVG samples).

Power-on default `0x7FF8` = +163.8 mV = full-scale, effectively disabling the alert.

#### Shunt-Voltage Sum Register (`0x0D`)

Sum of the single-conversion shunt-voltage readings for the channels selected by `SCC1`–`SCC3` in the Mask/Enable register. **14-bit signed**, left-aligned by 1 (bit 0 reserved). LSB = 40 µV.

| Bits  | Name      |
|-------|-----------|
| 15    | SIGN      |
| 14:1  | SV13:SV0  |
| 0     | —         |

#### Shunt-Voltage Sum-Limit Register (`0x0E`)

Same format as Shunt-Voltage Sum. Threshold against the sum register; trips Critical alert (and the SF flag) when exceeded.

#### `Mask/Enable` (`0x0F`) — reset `0x0002`

| Bit | Name | Description |
|-----|------|-------------|
| 15  | —    | Reserved |
| 14  | SCC1 | Include channel 1 in shunt-voltage sum |
| 13  | SCC2 | Include channel 2 in shunt-voltage sum |
| 12  | SCC3 | Include channel 3 in shunt-voltage sum |
| 11  | WEN  | Warning-alert latch enable (0 = transparent, 1 = latched) |
| 10  | CEN  | Critical-alert latch enable (0 = transparent, 1 = latched) |
| 9   | CF1  | Channel-1 critical-alert flag (R) |
| 8   | CF2  | Channel-2 critical-alert flag (R) |
| 7   | CF3  | Channel-3 critical-alert flag (R) |
| 6   | SF   | Summation-alert flag (R) |
| 5   | WF1  | Channel-1 warning-alert flag (R) |
| 4   | WF2  | Channel-2 warning-alert flag (R) |
| 3   | WF3  | Channel-3 warning-alert flag (R) |
| 2   | PVF  | Power-valid alert flag (R, mirrors PV pin) |
| 1   | TCF  | Timing-control alert flag (R, mirrors TC pin; default 1) |
| 0   | CVRF | Conversion ready flag (R) |

CF / SF / WF flags clear when the Mask/Enable register is read. CVRF clears on a write to Configuration (except for power-down/disable modes) **or** on a read of Mask/Enable. PVF clears when the underlying power-valid condition is removed and the PV pin has cleared.

#### Power-Valid Upper / Lower Limit (`0x10`, `0x11`)

12-bit unsigned, left-aligned by 3 (same format as the Bus Voltage registers). LSB = 8 mV. Defaults: upper = 10.000 V, lower = 9.000 V.

The PV pin asserts high when **all enabled** bus-voltage readings exceed the upper limit, and de-asserts when any drops below the lower limit. Bus-voltage measurements must be enabled (MODE = 010, 011, 110, or 111) for PV to function.

## Initialization Sequence

### Minimal

1. Optionally read `Manufacturer ID` (`0xFE`) and confirm `0x5449`. (Not required for operation, but useful as a presence check.)
2. The chip is already in continuous shunt+bus mode at power-on with all three channels enabled — no register writes are needed.

### Full

1. Write Configuration Register (`0x00`) with desired channel enables, AVG, VBUSCT, VSHCT, MODE.
2. Optionally write critical/warning limit registers and configure the Mask/Enable register (latches, summation channel selection).
3. Optionally write Power-Valid Upper/Lower Limit and Shunt-Voltage Sum Limit registers.

## Implementation Stages

### Minimal

Goal: read bus voltage, shunt voltage, current, and power for each of the three channels with no configuration beyond the transport and the shunt resistors.

Constructor parameters: `transport`, `r_shunt` — either a single float (default `0.1` Ω, applied to all three channels) or a 3-element sequence specifying per-channel resistance.

The constructor does **not** write any registers — the chip's power-on default (`0x7127`) is "all three channels on, continuous shunt+bus, max range, no averaging" which is the right default for Minimal. Drivers store `r_shunt` for use in `current()` / `power()`.

| Operation             | Parameters | Returns | Notes |
|-----------------------|------------|---------|-------|
| `voltage(channel)`    | `1`–`3`    | float   | Bus voltage in V |
| `shunt_voltage(channel)` | `1`–`3` | float   | Shunt voltage in V |
| `current(channel)`    | `1`–`3`    | float   | `shunt_voltage(channel) / r_shunt[channel]`, in A |
| `power(channel)`      | `1`–`3`    | float   | `voltage(channel) × current(channel)`, in W |

Each language picks idiomatic 0- vs 1-based indexing (the spec uses 1-based to match the datasheet's CH1/CH2/CH3 nomenclature; Python and JS implementations should accept 1, 2, 3 and reject anything else).

### Full

Extends Minimal. Adds full register access and the alert engine.

## Data Conversion

```
shunt_voltage_V (ch) = sign_extend(read16(reg) >> 3, 13) × 40e-6
                     # equivalently: (read16 as i16) × 5e-6  if you skip the explicit shift
bus_voltage_V (ch)   = (read16(reg) >> 3) × 8e-3
                     # bit 15 is always 0 for valid bus reading; treat as unsigned

current_A (ch)       = shunt_voltage_V(ch) / r_shunt[ch]
power_W (ch)         = bus_voltage_V(ch) × current_A(ch)
```

For limit-register writes:
```
# Critical / warning limit (LSB = 40 µV at the 13-bit count; register stores
# count << 3, matching the shunt-voltage data layout):
limit_raw = (int(limit_V / 40e-6) << 3) & 0xFFF8

# Power-Valid Upper/Lower Limit (LSB = 8 mV at the 12-bit count):
limit_raw = (int(limit_V / 8e-3) << 3) & 0xFFF8

# Shunt-Voltage Sum Limit (LSB = 40 µV at the 14-bit count, left-aligned by 1):
limit_raw = (int(limit_V / 40e-6) << 1) & 0xFFFE
```

Note the off-by-one alignment between the per-channel registers (3-bit reserved at LSB end) and the Sum/Sum-Limit registers (1-bit reserved at LSB end) — the sum has one more bit of headroom because it can grow up to 3× a single-channel value.

## Examples

### Demo

A three-rail power monitor: the user wires the INA3221 between a 5 V supply (channel 1, e.g., USB rail), a 3.3 V supply (channel 2, e.g., MCU rail), and a 12 V supply (channel 3, e.g., motor rail). The demo polls all three channels once per second for 30 seconds, printing a one-line tabular update each tick (`V1 I1 P1 | V2 I2 P2 | V3 I3 P3`). At t=10 s the demo programs critical-alert limits at "1.5× current draw" for each channel and prints "alerts armed"; at t=20 s it programs the summation function across all three channels with a sum limit and prints "summation armed". After the run, the demo dumps the contents of the Mask/Enable register and decodes any flags that fired during the run, demonstrating both the per-channel and aggregate alert behavior.

## Timing Constraints

- Recovery from power-down mode: 40 µs
- All registers are volatile — reprogrammed on every power-up
- CVRF and alert flags clear on a read of Mask/Enable
- All-channel, all-measurement conversion time = (VBUSCT + VSHCT) × (number of enabled channels), so a single rail with 1.1 ms / 1.1 ms / no averaging takes ~2.2 ms; all three with 8.244 ms / 8.244 ms / 1024 averages takes ~50.7 s
- Timing-control alert function only runs at power-up or after a software reset; writing the Configuration register before the TC sequence completes disables TC until power-cycle or RST

## Implementation Notes

- The INA3221 has no Calibration / Current / Power registers — the driver computes current and power from the shunt voltage. This is the biggest API-shaping difference from the INA226: the constructor takes shunt resistance per channel, not a single value, and there is no need to compute a calibration constant.
- Shunt voltage and bus voltage registers store data **left-aligned** with the lower 3 bits reserved. Right-shift by 3 (with sign extension for shunt) before scaling. Treating the raw 16-bit signed read as `count × 5 µV` for shunt or `count × 1 mV` for bus is mathematically equivalent and avoids the shift.
- The Shunt-Voltage Sum register is left-aligned by **1** bit, not 3 — its 14-bit count has more headroom because the sum can be ~3× a single-channel value. Driver code must use the right shift amount for each register (see Data Conversion section).
- Reset behavior: setting RST resets *all* registers (including alert limits and the Mask/Enable register) to power-on defaults. Drivers must restore any user-programmed limits and configuration after `reset()`.
- The INA3221 has 4 alert pins (Critical, Warning, PV, TC) but the driver only sees them via the Mask/Enable flags. Hardware monitoring is via the open-drain pins; software monitoring is via reading Mask/Enable. Reading Mask/Enable clears the latched flags (when latch is enabled), so reads are destructive — the Full driver should expose `alert_flags()` as the single point of access and let callers decode bits with the named constants.
- Per-channel critical-alert limits compare against the **single-conversion** shunt reading; warning-alert limits compare against the **averaged** shunt reading. With AVG=000 (1 sample, default) the two are equivalent; with averaging enabled, critical fires faster but is noisier.
- The summation function only makes sense when all selected channels use the same shunt resistance — otherwise the sum is in volts but cannot be cleanly converted to a single current value. The driver does not enforce this; it is documented in the spec and in the `set_summation_channels()` docstring.
- The TC (timing-control) flag is set high (`0x0002`) at power-on by default and only clears on RST or power-cycle. Code that reads Mask/Enable should not interpret a set TCF as an active alert unless it has armed the timing-control feature.
- Manufacturer ID (`0x5449`) matches the INA226's value; Die ID (`0x3220`) is unique. Use Die ID, not Manufacturer ID, to distinguish chips at runtime.
- Disabled channels still occupy register space and read as `0x0000`. The Full driver's `voltage()` / `current()` methods on a disabled channel should return whatever the register says (zero) — it is the caller's responsibility to enable channels before reading them.
