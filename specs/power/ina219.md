# Chip Spec: INA219

**Manufacturer:** Texas Instruments  
**Datasheet:** `datasheets/power/ina219.pdf`  
**Category:** `power`  
**Transports:** I²C, SMBus

## Overview

The INA219 is a 26V, 12-bit current, voltage, and power monitor. It measures the differential shunt voltage between IN+ and IN− (with selectable PGA gain of ÷1, ÷2, ÷4, or ÷8 → ±40 / ±80 / ±160 / ±320 mV full-scale) to derive current, and the bus voltage at IN− referenced to GND (with selectable 16 V or 32 V full-scale). An internal multiplier calculates power. Current and power readings require the Calibration Register to be programmed with the shunt resistor value and desired current LSB. The device operates from 3–5.5 V and supports up to 16 I²C addresses via A0/A1 pins.

The INA219 is the older, lower-resolution sibling of the INA226. They share the same address scheme, calibration model, and the basic shunt + bus + current + power measurement architecture, but differ substantially in register layout, ADC resolution, scaling factors, and feature set: the INA219 has a programmable shunt PGA and selectable bus range, while the INA226 has higher resolution, hardware averaging, and an alert engine. Driver implementations of the two chips are siblings, not parent/child.

## Transport Configuration

### I²C / SMBus

- **Addresses:** `0x40`–`0x4F` (16 addresses, set by A0 and A1 pins — identical scheme to INA226)

| A1  | A0  | Address |
|-----|-----|---------|
| GND | GND | `0x40`  |
| GND | VS  | `0x41`  |
| GND | SDA | `0x42`  |
| GND | SCL | `0x43`  |
| VS  | GND | `0x44`  |
| VS  | VS  | `0x45`  |
| VS  | SDA | `0x46`  |
| VS  | SCL | `0x47`  |
| SDA | GND | `0x48`  |
| SDA | VS  | `0x49`  |
| SDA | SDA | `0x4A`  |
| SDA | SCL | `0x4B`  |
| SCL | GND | `0x4C`  |

- **Max clock:** 400 kHz (fast mode), 2.56 MHz (high-speed mode)
- **SMBus timeout:** 28 ms — the interface resets if SCL or SDA is held low longer than this

All registers are 16-bit, big-endian (MSB first). Read/write is via a register pointer byte sent before data. After completing a write, allow ≥4 µs before reading the same register at SCL > 1 MHz.

## Register Map

| Address | Name            | R/W | Reset    | Description |
|---------|-----------------|-----|----------|-------------|
| `0x00`  | Configuration   | R/W | `0x399F` | Reset, bus range, PGA, ADC res/avg, mode |
| `0x01`  | Shunt Voltage   | R   | `0x0000` | Differential shunt voltage (2's complement, sign-extended per PGA) |
| `0x02`  | Bus Voltage     | R   | `0x0000` | Bus voltage [15:3] + CNVR + OVF |
| `0x03`  | Power           | R   | `0x0000` | Calculated power (requires calibration) |
| `0x04`  | Current         | R   | `0x0000` | Calculated current (signed, requires calibration) |
| `0x05`  | Calibration     | R/W | `0x0000` | Shunt value and current LSB scaling (FS15:FS1; FS0 is void) |

The INA219 does **not** provide a Mask/Enable register, an Alert Limit register, a Manufacturer ID register, or a Die ID register. CNVR and OVF status flags live in the Bus Voltage register.

### Bit Fields

#### `Configuration` (`0x00`) — reset `0x399F`

| Bits  | Name | Description |
|-------|------|-------------|
| 15    | RST  | Write 1 to reset all registers; self-clears |
| 14    | —    | Reserved |
| 13    | BRNG | Bus voltage range: 0 = 16 V FSR, 1 = 32 V FSR (default) |
| 12:11 | PG   | Shunt PGA gain (see below) |
| 10:7  | BADC | Bus ADC resolution / averaging (see ADC table) |
| 6:3   | SADC | Shunt ADC resolution / averaging (same encoding) |
| 2:0   | MODE | Operating mode |

**PG — shunt PGA gain and full-scale range:**

| PG | Gain | Range    |
|----|------|----------|
| 00 | /1   | ±40 mV   |
| 01 | /2   | ±80 mV   |
| 10 | /4   | ±160 mV  |
| 11 | /8   | ±320 mV (default) |

**BADC / SADC — ADC resolution or averaging (same 4-bit encoding for both):**

| Value (b3 b2 b1 b0) | Mode/Samples | Conversion Time |
|--------------------|--------------|-----------------|
| `0X00`             | 9-bit        | 84 µs   |
| `0X01`             | 10-bit       | 148 µs  |
| `0X10`             | 11-bit       | 276 µs  |
| `0X11`             | 12-bit       | 532 µs (default) |
| `1000`             | 12-bit       | 532 µs |
| `1001`             | 2 samples avg   | 1.06 ms  |
| `1010`             | 4 samples avg   | 2.13 ms  |
| `1011`             | 8 samples avg   | 4.26 ms  |
| `1100`             | 16 samples avg  | 8.51 ms  |
| `1101`             | 32 samples avg  | 17.02 ms |
| `1110`             | 64 samples avg  | 34.05 ms |
| `1111`             | 128 samples avg | 68.10 ms |

(`X` = don't care for the 9/10/11/12-bit single-shot encodings.)

**MODE:**

| Value | Mode |
|-------|------|
| 000   | Power-down |
| 001   | Shunt voltage, triggered |
| 010   | Bus voltage, triggered |
| 011   | Shunt and bus, triggered |
| 100   | ADC off (disabled) |
| 101   | Shunt voltage, continuous |
| 110   | Bus voltage, continuous |
| 111   | Shunt and bus, continuous (default) |

Writing any triggered-mode value (even if already programmed) starts a single-shot conversion.

#### `Shunt Voltage` (`0x01`)

Two's-complement signed value with sign extension that depends on the PGA setting (the upper sign bits are duplicated; LSB is always 10 µV regardless of PGA).

| Bits  | Name | Description |
|-------|------|-------------|
| 15    | SIGN | Sign bit (2's complement; sign-extended to bits per PGA) |
| 14:0  | SDx  | Shunt voltage magnitude, LSB = 10 µV |

Out-of-range values for the active PGA setting saturate at ±full-scale.

#### `Bus Voltage` (`0x02`)

| Bits  | Name | Description |
|-------|------|-------------|
| 15:3  | BD12:BD0 | Bus voltage value (right-shift register by 3 to obtain a 13-bit unsigned count); LSB = 4 mV |
| 2     | —    | Reserved (read 0) |
| 1     | CNVR | Conversion Ready Flag (clears on read of Power register or on Configuration write that selects an active mode) |
| 0     | OVF  | Math Overflow Flag (set when current/power calculation overflows) |

## Initialization Sequence

### Minimal

1. Write Calibration Register (`0x05`) — required for current and power readings:
   ```
   Current_LSB = max_current / 32768
   CAL         = int(0.04096 / (Current_LSB × r_shunt)) & 0xFFFE   # FS0 is void
   ```
2. Device is already in continuous shunt+bus mode at power-on with 32 V bus range, PGA ÷8, and 12-bit ADC; no further configuration needed.

### Full

1. Optionally write Configuration Register (`0x00`) with desired BRNG, PGA, BADC, SADC, and MODE.
2. Write Calibration Register (`0x05`) as above.

## Implementation Stages

### Minimal

Goal: read bus voltage, shunt voltage, current, and power with no configuration beyond the transport and shunt resistor.

Constructor parameters: `transport`, `r_shunt` (Ω, default `0.1`), `max_current` (A, default `2.0`).

Constructor computes and writes the Calibration Register automatically. It does **not** rewrite the Configuration Register — the chip's power-on default (`0x399F`) is already a sensible "everything on, max range, max resolution" setting. Drivers store `current_lsb` for use by `current()` / `power()`.

**Effective defaults at construction (chip power-on values):**
- BRNG = 1 (32 V FSR)
- PG = 11 (÷8, ±320 mV shunt)
- BADC = 0011 (12-bit, 532 µs)
- SADC = 0011 (12-bit, 532 µs)
- MODE = 111 (shunt + bus, continuous)

| Operation        | Parameters | Returns | Notes |
|------------------|------------|---------|-------|
| `voltage()`      | —          | float   | Bus voltage in V |
| `shunt_voltage()`| —          | float   | Shunt voltage in V |
| `current()`      | —          | float   | Current in A (signed) |
| `power()`        | —          | float   | Power in W |

### Full

Extends Minimal. Adds full configuration access, status, and power management.

| Operation             | Parameters                              | Returns | Notes |
|-----------------------|-----------------------------------------|---------|-------|
| *(inherits Minimal)*  | | | |
| `configure()`         | `brng`, `pga`, `badc`, `sadc`, `mode`   | —       | Write Configuration Register; re-writes Calibration afterward |
| `conversion_ready()`  | —                                       | bool    | Read CNVR bit (bit 1) of Bus Voltage register |
| `overflow()`          | —                                       | bool    | Read OVF bit (bit 0) of Bus Voltage register |
| `reset()`             | —                                       | —       | Set RST bit; restore previous Configuration and Calibration |
| `shutdown()`          | —                                       | —       | Set MODE = 000 |
| `wake()`              | —                                       | —       | Restore previous MODE |
| `trigger()`           | —                                       | —       | Re-write current MODE bits (works only if MODE is a triggered mode) — starts a single-shot conversion |

## Data Conversion

```
Current_LSB = max_current / 32768                       # A/bit
CAL         = int(0.04096 / (Current_LSB × r_shunt)) & 0xFFFE
Power_LSB   = 20 × Current_LSB                          # W/bit (fixed by chip)

shunt_voltage (V) = raw_s16 × 10e-6                     # raw is signed 16-bit; LSB = 10 µV
bus_voltage   (V) = (raw_u16 >> 3) × 4e-3               # right-shift by 3, then ×4 mV; LSB = 4 mV
current       (A) = raw_s16 × Current_LSB               # raw is signed 16-bit
power         (W) = raw_u16 × Power_LSB
```

Notes:
- The Calibration Register's bit 0 (FS0) is **void** — it is forced to 0 by the chip. Implementations should mask the computed CAL with `0xFFFE` before writing, or simply not worry (the chip will discard the LSB).
- The Bus Voltage register's lowest 3 bits hold reserved/CNVR/OVF flags, **not** part of the bus voltage value. Mask or shift before scaling.
- INA219 calibration constant `0.04096` differs from INA226's `0.00512`; INA219 Power_LSB = 20 × Current_LSB (INA226 uses 25 ×).

### Demo

A power-supply sanity check: poll the INA219 once per second for 10 seconds, printing `V_bus`, `I`, and `P`. Between samples 4 and 5, prompt the user to switch on a load (e.g., plug in a USB device, switch on a fan); the demo highlights the change in current and power so the reader sees the chip respond to a real-world event. ## Timing Constraints

- Recovery from power-down mode: 40 µs
- Register read after write at SCL > 1 MHz: wait 4 µs
- All registers are volatile — reprogrammed on every power-up (no NVM)
- Maximum conversion time at 128-sample averaging: 68.10 ms (per channel; shunt and bus run sequentially in continuous mode)

## Implementation Notes

- Current Register (`0x04`) and Power Register (`0x03`) read as zero until Calibration Register is programmed with a non-zero value.
- Current Register is signed (2's complement); Power Register is unsigned.
- The Bus Voltage register layout puts the value in bits 15:3 — driver code must right-shift the 16-bit raw read by 3 (or mask off bits 2:0 and divide) before multiplying by 4 mV. A common bug is to forget the shift and report bus voltage as 8× the actual value.
- The Shunt Voltage register's sign-extension width depends on the active PGA setting, but the LSB (10 µV) is constant — sign-extending a full 16-bit signed read is always correct.
- The CAL register's FS0 bit is hard-wired to 0; computed calibrations get rounded down to the nearest even number.
- CNVR clears when the Power register is read (or on Configuration write to an active mode). To poll for "data ready" without consuming the flag, read Bus Voltage and check bit 1 — but be aware that any subsequent Power read will then clear it.
- Writing any triggered MODE value (`001`, `010`, `011`) re-triggers a single-shot conversion, even if MODE already held that value. Use `trigger()` for this in the Full API.
- INA219 has **no alert mechanism** — there is no Alert pin, no Alert Limit register, no Mask/Enable register. Code that ports cleanly from INA226 must drop the alert API.
- INA219 has **no ID registers** — driver tests should not check manufacturer/die ID; presence is verified by attempting a register read after configuration.
- Reset behavior: setting RST resets *all* registers (including Calibration) to power-on defaults. Drivers must re-write the Calibration Register after `reset()`. The Full driver should also restore the most recent user-supplied Configuration if `configure()` was called.
