# Chip Spec: APDS-9930

**Manufacturer:** Avago Technologies (Broadcom)  
**Datasheet:** `datasheets/light/apds-9930.pdf`  
**Category:** light  
**Transports:** I²C

## Overview

The APDS-9930 is a digital ambient light sensor (ALS) and proximity detector in a single 8-pin optical module. It contains two photodiodes (Ch0: visible + IR; Ch1: IR-only), two integrating ADCs, and an integrated 850 nm IR LED with synchronous driver. The dual-channel design allows the firmware to compensate for the IR component of ambient light, producing a lux estimate that approximates the human-eye response. The built-in IR LED and optical apertures provide factory-calibrated proximity detection to 100 mm without external calibration. Communication is via I²C with a dedicated open-drain interrupt pin. Typical applications include display backlight control, phone proximity sensing, and screen lock/unlock.

## Transport Configuration

### I²C

- **Address:** `0x39` (fixed; contact factory for alternatives)
- **Max clock:** 400 kHz (Fast-mode)

#### Command register protocol

Every register access on this device begins with a **command byte** written as the first byte after the I²C address. The command byte must always have bit 7 set (`CMD = 1`). Bits 6:5 select the transaction type:

| Bits 6:5 | Mode |
|----------|------|
| `00` | Repeated Byte — same register re-read/re-written each access |
| `01` | Auto-Increment — register address advances after each byte |
| `11` | Special Function — interrupt clear; ADD field selects the function |

For all normal register writes: `0x80 | reg_addr`.  
For reading 16-bit data registers (recommended): `0xA0 | reg_addr` (auto-increment).

Special function commands (TYPE = `11`, bits 6:5 = `11`):

| ADD field (bits 4:0) | Function |
|----------------------|----------|
| `0x00` (`00000`) | No action |
| `0x05` (`00101`) | Clear proximity interrupt |
| `0x06` (`00110`) | Clear ALS interrupt |
| `0x07` (`00111`) | Clear both proximity and ALS interrupts |

To issue a special function: send a single byte `0xE0 | function_code` after the I²C address+write.

### Bit Fields

#### `ENABLE` (`0x00`)

| Bits | Name | Description |
|------|------|-------------|
| 7 | Reserved | Write as 0 |
| 6 | SAI | Sleep After Interrupt — device powers down after an interrupt cycle when set |
| 5 | PIEN | Proximity Interrupt Enable |
| 4 | AIEN | ALS Interrupt Enable |
| 3 | WEN | Wait Enable — activates the wait timer between cycles |
| 2 | PEN | Proximity Enable |
| 1 | AEN | ALS Enable — activates both Ch0 and Ch1 ADCs |
| 0 | PON | Power ON — enables internal oscillator |

#### `CONTROL` (`0x0F`)

| Bits | Name | Values | Description |
|------|------|--------|-------------|
| 7:6 | PDRIVE | `00`=100 mA, `01`=50 mA, `10`=25 mA, `11`=12.5 mA | Proximity LED drive (when PDL=0) |
| 5:4 | PDIODE | `10`=Ch1 | Proximity diode select — **must be `0x2` (Ch1) for proximity to function** |
| 3:2 | PGAIN | `00`=1×, `01`=2×, `10`=4×, `11`=8× | Proximity ADC gain |
| 1:0 | AGAIN | `00`=1×, `01`=8×, `10`=16×, `11`=120× | ALS ADC gain |

#### `CONFIG` (`0x0D`)

| Bits | Name | Description |
|------|------|-------------|
| 7:3 | Reserved | Write as 0 |
| 2 | AGL | ALS Gain Level — scales 1× and 8× gains by 1/6 (do not use with AGAIN > 8×) |
| 1 | WLONG | Wait Long — multiplies WTIME wait duration by 12× |
| 0 | PDL | Proximity Drive Level — reduces LED current to 1/9 of PDRIVE setting |

#### `PERS` (`0x0C`)

| Bits | Name | Description |
|------|------|-------------|
| 7:4 | PPERS | Proximity persistence: 0=every cycle; 1–15=N consecutive out-of-range readings |
| 3:0 | APERS | ALS persistence: 0=every cycle; 1=1; 2=2; 3=3; 4=5; 5=10; 6=15; … 15=60 consecutive |

#### `STATUS` (`0x13`)

| Bits | Name | Description |
|------|------|-------------|
| 7 | Reserved | |
| 6 | PSAT | Proximity Saturation — integrator saturated; clear with proximity interrupt clear |
| 5 | PINT | Proximity Interrupt pending |
| 4 | AINT | ALS Interrupt pending |
| 3:2 | Reserved | |
| 1 | PVALID | Proximity conversion complete (set after first proximity cycle) |
| 0 | AVALID | ALS conversion complete (set after first ALS cycle) |

#### `POFFSET` (`0x1E`)

| Bits | Name | Description |
|------|------|-------------|
| 7 | SIGN | 0 = negative offset; 1 = positive offset |
| 6:0 | MAGNITUDE | Offset magnitude (0–127); actual shift depends on PGAIN, PDRIVE, PPULSE |

## Initialization Sequence

1. Write `ENABLE = 0x00` — power down and disable all functions
2. Write `ATIME` — ALS integration time
3. Write `PTIME = 0xFF` — proximity ADC time (keep at default)
4. Write `WTIME` — wait time (if wait is used)
5. Write `PPULSE` — proximity pulse count
6. Write `CONTROL` — set PDIODE=Ch1 (`0x20` base), PDRIVE, PGAIN, AGAIN
7. Write `CONFIG` — AGL, WLONG, PDL as needed
8. Write `ENABLE` — set PON and enable desired functions (AEN, PEN, WEN)
9. Wait ≥ 12 ms before reading data (allows first ALS and proximity cycles to complete)

> After initial VDD power-on, wait ≥ 4.5 ms before issuing any I²C command.

## Implementation Stages

Each chip is implemented in two stages. The Full class extends Minimal — it inherits everything and adds the rest.

### Minimal

Goal: expose ambient light sensing (lux) and raw proximity detection with a simple, user-friendly interface. No configuration required beyond the transport.

| Operation | Parameters | Returns | Notes |
|-----------|------------|---------|-------|
| `__init__` | `transport` | — | Runs initialization with sensible defaults; waits 12 ms |
| `lux` | — | `float` lx | Illuminance in lux using open-air coefficients |
| `proximity` | — | `int` (0–1023) | Raw proximity ADC count; higher = closer |

**Sensible defaults baked into Minimal:**

| Register | Value | Effect |
|----------|-------|--------|
| ATIME | `0xDB` | 101 ms integration (37 cycles) — rejects 50/60 Hz fluorescent flicker |
| PTIME | `0xFF` | 2.73 ms proximity ADC time (recommended default) |
| PPULSE | `0x08` | 8 LED pulses — factory-calibrated for 100 mm range |
| CONTROL | `0x20` | PDIODE=Ch1, PDRIVE=100 mA, PGAIN=1×, AGAIN=1× |
| ENABLE | `0x07` | PON + AEN + PEN; wait timer and interrupts disabled |

### Full

Goal: expose complete chip functionality. Extends Minimal.

| Operation | Parameters | Returns | Notes |
|-----------|------------|---------|-------|
| *(inherits Minimal)* | | | |
| `configure_als` | `atime=0xDB`, `again=0`, `agl=False` | — | ALS integration time register value and gain index (0–3 → 1×/8×/16×/120×) |
| `configure_proximity` | `ppulse=8`, `pgain=0`, `pdrive=0`, `pdl=False`, `ptime=0xFF` | — | Proximity LED pulses (1–255), gain index (0–3), drive index (0–3), reduced-drive flag, ADC time |
| `configure_wait` | `wtime=0xFF`, `wlong=False` | — | Wait time register value and 12× multiplier flag; enables WEN when called |

| `ch1` | — | `int` | Raw Ch1 ADC count (IR-only) |
| `status` | — | `dict` | `{avalid, pvalid, psat, aint, pint}` — boolean fields from STATUS register |
| `set_als_thresholds` | `low`, `high`, `persistence=1` | — | ALS interrupt thresholds (raw Ch0 counts, 16-bit); enables AIEN |
| `set_proximity_thresholds` | `low`, `high`, `persistence=1` | — | Proximity interrupt thresholds (raw counts, 16-bit); enables PIEN |

| `disable_wait` | — | — | Clear WEN in ENABLE register |

**Additional configuration options:**

- AGAIN: 1×, 8×, 16×, 120× (index 0–3 → CONTROL bits 1:0)
- PGAIN: 1×, 2×, 4×, 8× (index 0–3 → CONTROL bits 3:2)
- PDRIVE: 100 mA, 50 mA, 25 mA, 12.5 mA (index 0–3 → CONTROL bits 7:6)
- AGL: scales 1× → 1/6×, 8× → 8/6× (enables operation up to ~30 klx)
- WLONG: extends all WTIME durations by 12× (up to ~8.4 s)
- PDL: reduces LED drive by factor of 9 (e.g., 100 mA → 11.1 mA)
- APERS encoding: 0=every, 1=1, 2=2, 3=3, 4=5, 5=10, 6=15, 7=20, 8=25, 9=30, 10=35, 11=40, 12=45, 13=50, 14=55, 15=60 consecutive

## Data Conversion

### ALS — Lux Calculation (open air, no window)

```
ALSIT   = 2.73 × (256 − ATIME)          [ms]
AGAINx  = 1, 8, 16, or 120              [gain factor for AGAIN index 0–3]

CH0DATA = (Ch0DATAH << 8) | Ch0DATAL
CH1DATA = (Ch1DATAH << 8) | Ch1DATAL

IAC1 = CH0DATA − 1.862 × CH1DATA
IAC2 = 0.746 × CH0DATA − 1.291 × CH1DATA
IAC  = max(IAC1, IAC2, 0)

LPC  = (GA × DF) / (ALSIT × AGAINx)
     = (0.49 × 52) / (ALSIT × AGAINx)

lux  = IAC × LPC
```

Open-air coefficients: GA = 0.49, DF = 52, B = 1.862, C = 0.746, D = 1.291.

For window/lens assemblies, GA and the B/C/D coefficients must be recalibrated per the application note referenced in the datasheet.

When AGL is set, effective gain factors become: 1× → 1/6, 8× → 8/6, 16× → 16/6, 120× → 20×.

### Proximity — Raw Count

Proximity data is a 16-bit value but realistically limited to 10 bits (0–1023) at PTIME=0xFF (1 cycle). Higher count = object closer. No distance formula is provided by the datasheet; the relationship is application-specific and depends on reflectance, LED drive, pulse count, and glass parameters. The recommended baseline operating condition is PDRIVE=100 mA, PPULSE=8.

### ALS Integration Time

```
ALSIT = 2.73 ms × (256 − ATIME)
```

Common values:

| ATIME | Cycles | ALSIT | Max Count | 50/60 Hz rejection |
|-------|--------|-------|-----------|---------------------|
| `0xFF` | 1 | 2.73 ms | 1023 | No |
| `0xF6` | 10 | 27.3 ms | 10239 | No |
| `0xED` | 19 | 51.9 ms | — | 50 Hz half-cycle |
| `0xDB` | 37 | 101 ms | 37887 | Yes (50 and 60 Hz) |
| `0xC0` | 64 | 175 ms | 65535 | Yes |
| `0x00` | 256 | 699 ms | 65535 | Yes |

## Examples

### Demo

Scenario: **Adaptive backlight controller**. The demo reads lux and proximity in a loop, printing both values. When lux drops below 10 lx (dim environment), it logs a "dim backlight" action. When proximity exceeds 400 counts, it logs a "disable screen" action. This mirrors the intended use in cell phones — one sensor does both ambient brightness adaptation and face-detection screen lock.

The demo should:
1. Construct `Apds9930Full` with sensible defaults.
2. Configure ALS with ATIME=0xDB and 1× gain.
3. Configure proximity with 8 pulses.
4. Loop at ~1 Hz:
   - Read lux and proximity.
   - Print both values.
   - Print action message if lux < 10 lx or proximity > 400.

## Timing Constraints

- After VDD power-on: wait ≥ 4.5 ms before any I²C transaction.
- After writing ENABLE with PON set: wait ≥ 12 ms before reading valid data (allows first full ALS + proximity cycle to complete).
- AVALID (STATUS bit 0) and PVALID (STATUS bit 1) can be polled instead of using fixed delays.
- ATIME=0xDB gives 101 ms per ALS cycle; reading faster than that will return stale data.
- Proximity ADC conversion at PTIME=0xFF takes 2.73 ms (plus LED pulse accumulation time: PPULSE × 16 µs).
- WLONG extends all WTIME periods by 12×, enabling wait times up to ~8.4 s.

## Implementation Notes

- **PDIODE must be set to `0x2` (Ch1)** in CONTROL bits 5:4 for proximity to function. PDIODE=0x0 or 0x1 are reserved. The CONTROL write in `__init__` must always include `0x20` regardless of ALS-only use.
- Reading 16-bit registers (Ch0DATA, Ch1DATA, PDATA) must use the auto-increment protocol (`0xA0 | reg_addr`) to read both bytes atomically. The device latches the high byte in a shadow register when the low byte is read.
- The ALS interrupt thresholds are evaluated against **Ch0DATA** (not lux). Implementations that expose `set_als_thresholds` must document this clearly.
- If PSAT (STATUS bit 6) is set, the proximity reading is unreliable. It persists until cleared with a proximity interrupt clear command (`0xE5`).
- Setting the low ALS/proximity interrupt threshold above the high threshold causes the high threshold to be ignored; only the low threshold is evaluated. Guard against this in `set_als_thresholds` / `set_proximity_thresholds`.
- POFFSET sign convention: SIGN=0 → negative offset (shifts data down); SIGN=1 → positive offset (shifts data up). The magnitude is not in physical units — it is an empirical register value determined by per-unit calibration.
- Device ID register `0x12` reads `0x39`. An implementation may verify this on init to confirm the correct chip is present.
