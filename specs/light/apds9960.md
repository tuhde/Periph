# Chip Spec: APDS-9960

**Manufacturer:** Broadcom (Avago Technologies)  
**Datasheet:** `datasheets/light/apds9960.pdf`  
**Category:** `light`  
**Transports:** I²C

## Overview

The APDS-9960 is a digital proximity, ambient light, RGB color, and gesture sensor in a single 3.94 × 2.36 × 1.35 mm optical module. Four independent engines share one I²C interface: the ALS/Color engine delivers 16-bit RGBC channels (Clear, Red, Green, Blue) with configurable integration time and analog gain; the Proximity engine emits IR pulses and returns an 8-bit proximity count; the Gesture engine captures four-directional hand movement (UP, DOWN, LEFT, RIGHT) using four dedicated directional photodiodes and a 32-dataset FIFO; the Wait engine inserts programmable low-power idle periods between measurement cycles. The INT pin (open-drain, active low) can be asserted on ALS threshold crossings, proximity threshold crossings, or gesture FIFO watermarks. No user calibration is required. Typical applications: touchless gesture control, proximity-based display wake-on-approach, ambient light compensation for display brightness, and RGBC color measurement.

## Transport Configuration

### I²C

- **Address:** `0x39` (fixed; no alternative)
- **Max clock:** 400 kHz (Fast mode)
- **Protocol:** Standard write for register writes; standard repeated-start read for register reads; auto-increment on consecutive byte reads

Special "command-code" clear registers (0xE4–0xE7, 0xE5 PICLEAR, 0xE6 CICLEAR, 0xE7 AICLEAR) require an address-only transaction: write the chip address byte with R/W=0, then the register address byte, then generate a STOP — no data byte is written. This triggers the clear action.

The I²C address byte includes a fixed 3-bit command field: the upper 7 bits are the 7-bit slave address (0x39), so the write byte is `0x72` and the read byte is `0x73`. All registers are accessed using the standard combined-format transaction.

### Bit Fields

#### `ENABLE` (`0x80`)

| Bits | Name | Description |
|------|------|-------------|
| 7 | reserved | |
| 6 | GEN | Gesture engine enable |
| 5 | PIEN | Proximity interrupt enable |
| 4 | AIEN | ALS interrupt enable |
| 3 | WEN | Wait enable (inserts WTIME between ALS cycles) |
| 2 | PEN | Proximity detection enable |
| 1 | AEN | ALS and Color enable |
| 0 | PON | Power ON; must be set before enabling any engine |

#### `ATIME` (`0x81`)

Integration time control. CYCLES = 256 − ATIME; each cycle is 2.78 ms.

| ATIME | Cycles | Integration time | Max count |
|-------|--------|-----------------|-----------|
| `0xFF` | 1 | 2.78 ms | 1,025 |
| `0xF6` | 10 | 27.8 ms | 10,241 |
| `0xDB` | 37 | 102.9 ms | 37,889 |
| `0xB6` | 72 | 200.2 ms | 65,535 (capped) |
| `0x00` | 256 | 711.7 ms | 65,535 (capped) |

MAX_COUNT = min(65535, CYCLES × 1025).

#### `WTIME` (`0x83`)

Wait time between cycles (requires WEN=1). With WLONG=0 (CONFIG1 bit 1 = 0): WAIT = (256 − WTIME) × 2.78 ms. With WLONG=1: each cycle is 33.4 ms instead of 2.78 ms (×12 multiplier).

#### `PERS` (`0x8C`)

| Bits | Name | Description |
|------|------|-------------|
| 7:4 | PPERS | Proximity interrupt persistence: 0=every cycle, 1–15=N consecutive cycles out of range |
| 3:0 | APERS | ALS interrupt persistence: 0=every cycle, 1=1 cycle, 2=2 cycles, 3=3 cycles, 4=5, 5=10, 6=15, 7=20, 8=25, 9=30, 10=35, 11=40, 12=45, 13=50, 14=55, 15=60 |

#### `CONFIG1` (`0x8D`)

Bits 7:2 and bit 0 are reserved and must be written as they read (reset=`0x60`, so bits 6 and 5 must always be written as `1`). Bit 1 = WLONG: `0` = 2.78 ms per wait cycle (default), `1` = 33.4 ms per wait cycle (×12 multiplier).

#### `PPULSE` (`0x8E`)

| Bits | Name | Description |
|------|------|-------------|
| 7:6 | PPLEN | Proximity pulse length: `0`=4 µs, `1`=8 µs, `2`=16 µs, `3`=32 µs |
| 5:0 | PPULSE | Proximity pulse count − 1 (0=1 pulse … 63=64 pulses) |

Reset=`0x40`: 1 pulse, 8 µs length.

## Initialization Sequence

1. Wait 5.7 ms after power-up before first I²C access.
2. Read ID register (0x92); verify value is `0xAB`.
3. Write ENABLE = `0x00` (all engines off, PON=0) to ensure a clean start.
4. Write ATIME = `0xB6` (72 cycles, ~200 ms integration; MAX_COUNT = 65535).
5. Write CONTROL = `0x01` (LDRIVE=100 mA, PGAIN=1×, AGAIN=4×).
6. Write CONFIG2 = `0x01` (LED_BOOST=100%, reserved bit 0 = 1).
7. Write ENABLE = `0x03` (PON=1, AEN=1; ALS/Color engine on).
8. Wait one full integration cycle (~200 ms with ATIME=0xB6) or poll STATUS.AVALID.

## Implementation Stages

Each chip is implemented in two stages. The Full class extends Minimal — it inherits everything and adds the rest.

### Minimal

Goal: ALS/Color measurement (RGBC) with no configuration required beyond the transport.

| Operation | Parameters | Returns | Notes |
|-----------|------------|---------|-------|
| `init` | transport | — | Runs initialization sequence; ATIME=0xB6, AGAIN=4×, PON+AEN enabled |
| `color_clear` | — | int | Raw clear (unfiltered) channel, 0–65535 |
| `color_red` | — | int | Raw red channel, 0–65535 |
| `color_green` | — | int | Raw green channel, 0–65535 |
| `color_blue` | — | int | Raw blue channel, 0–65535 |
| `color` | — | tuple(int, int, int, int) | Returns (clear, red, green, blue) in one burst read |

**Sensible defaults baked into Minimal:**
- ATIME = `0xB6` (72 cycles, ~200 ms integration time, max count 65535)
- AGAIN = `0x01` (4× ALS gain, CONTROL bits 1:0)
- CONFIG2 = `0x01` (LED_BOOST=100%, reserved bit set)
- No wait, no proximity, no gesture, no interrupts

`color()` performs a single burst read of 0x94–0x9B (8 bytes); reading CDATAL at 0x94 simultaneously latches all eight RGBC bytes. Call `color()` only after STATUS.AVALID=1 or after waiting the integration period.

**Notes on Full methods:**

`read_gesture_fifo()` reads GFLVL first, then performs (GFLVL × 4) bytes of burst reads from 0xFC. Returns a list of (U, D, L, R) tuples. Caller computes direction from the delta stream (e.g. mean(U)−mean(D) for vertical axis, mean(L)−mean(R) for horizontal axis).

The proximity offset registers (POFFSET_UR, POFFSET_DL) use a sign-magnitude encoding: bit 7 is the sign (0=positive, 1=negative), bits 6:0 are the magnitude (0–127).

## Data Conversion

### ALS/Color raw-to-lux (approximation)

The APDS-9960 does not provide factory calibration for lux; the raw clear count is used directly or converted with a board-specific coefficient. A common approximation using the color channels:

```
lux ≈ -0.32466 × R + 1.57837 × G + -0.73191 × B
cct ≈ (3810 × (B / G)) + 1391        (correlated color temperature, approximation)
```

These coefficients are valid for the default glass lens of the SparkFun breakout and will vary with optical aperture. The raw clear channel value is always the most accurate representation.

Raw counts saturate when they reach MAX_COUNT = min(65535, (256 − ATIME) × 1025). At saturation, STATUS.CPSAT is set; consider reducing AGAIN or shortening ATIME.

### Integration time

```
cycles = 256 - ATIME
integration_ms = cycles × 2.78
max_count = min(65535, cycles × 1025)
```

### Proximity

Proximity count 0–255 increases as an object approaches. Absolute count depends on surface reflectivity, LED current, pulse count, and gain; threshold calibration is application-specific.

### Gesture direction

From a dataset array of (U, D, L, R) tuples:
```
ud_delta = mean(U values) - mean(D values)
lr_delta = mean(L values) - mean(R values)
direction = argmax(|ud_delta|, |lr_delta|)
           → "up" / "down" (ud_delta > 0 → up) or "left" / "right" (lr_delta > 0 → left)
```

Typically 4–32 datasets are collected per gesture. The FIFO can hold 32 datasets (128 bytes); enable GFIFOTH to interrupt when enough data is accumulated.

## Timing Constraints

- Power-up to first I²C access: 5.7 ms
- Integration time at ATIME=0xFF: 2.78 ms (1 cycle); at ATIME=0xB6: ~200 ms (72 cycles)
- PON to first AVALID: one complete integration cycle after AEN is set
- Proximity measurement cycle: depends on pulse count × pulse length + ADC conversion time (~1 ms typical)
- Gesture inter-dataset wait: GWTIME (0–39.2 ms, GCONF2 bits 2:0)
- WTIME wait with WLONG=0: 2.78–711.7 ms; with WLONG=1: 33.4–8541.0 ms

## Implementation Notes

- **CDATAL latches all RGBC bytes:** Reading CDATAL (0x94) atomically latches all eight bytes 0x94–0x9B. Always burst-read all eight bytes starting from 0x94 to ensure consistency. Never read individual color registers separately.
- **CONFIG1 reserved bits:** CONFIG1 resets to `0x60`; bits 7 and 6 are reserved and must be preserved as `1`. When writing CONFIG1, always write the reset value `0x60` ORed with the WLONG bit. Do not write `0x00`.
- **CONFIG2 reserved bit 0:** CONFIG2 bit 0 must always be `1`. When writing CONFIG2, include bit 0 in every write.
- **Special clear registers (0xE4–0xE7):** These are write-only "command" registers that require an address-only I²C transaction — write the chip address (0x72) and the register address, then STOP, without writing any data byte. Standard I²C write-with-data will not work for these registers.
- **Proximity offset sign-magnitude encoding:** POFFSET_UR and POFFSET_DL use sign-magnitude (not two's-complement): bit 7=sign (1=negative), bits 6:0=magnitude 0–127. Maximum offset is ±127 counts.
- **Gesture FIFO burst read:** Always read GFIFO as a 4-byte burst starting at 0xFC. Each burst yields one (U, D, L, R) dataset. Address auto-increments from 0xFC to 0xFF, then wraps. Read GFLVL first to determine how many bursts to perform.
- **GCONF4.GMODE:** The hardware sets GMODE=1 when the gesture engine enters collection mode (proximity exceeds GPENTH). To abort gesture collection, write GMODE=0 in GCONF4. After a gesture completes (proximity drops below GEXTH), hardware clears GMODE.
- **Wait engine with gestures:** The Wait engine (WEN=1) applies only between ALS and Proximity cycles. It has no effect on gesture timing; gesture inter-measurement delay is controlled by GWTIME in GCONF2.
- **ALS and Proximity cannot run simultaneously at their highest rates:** The ALS integration occupies the ADC; proximity pulses fire only during ADC idle time. Long ALS integration times reduce effective proximity measurement rate.
- **Interrupt persistence (PERS):** Setting APERS or PPERS = 0 generates an interrupt every ALS/proximity cycle regardless of threshold. For polling drivers, leave PERS at reset `0x00`; interrupt pin will not fire because AIEN and PIEN default to 0.
- **No onboard calibration or correction:** The APDS-9960 does not self-calibrate. Proximity offsets (POFFSET_UR, POFFSET_DL) compensate for optical crosstalk from the device's own IR LED, and should be characterized per board layout.
- **Address 0x82 gap:** There is no register at address 0x82; addresses increment from 0x81 (ATIME) directly to 0x83 (WTIME). Do not read or write 0x82.
- **Address 0x88, 0x8A gaps:** No registers at 0x88 or 0x8A; proximity thresholds are at 0x89 (PILT) and 0x8B (PIHT).
- **Gesture offsets address gap:** GOFFSET registers are at 0xA4 (U), 0xA5 (D), 0xA7 (L), 0xA9 (R). Note: 0xA6 is GPULSE (not an offset), and 0xA8 is absent.

## Sigrok Decoder

Decoder id `apds9960`, input `['i2c']`. Matches I²C address `0x39`. Annotates: ENABLE byte decoded to named enable bits (PON, AEN, PEN, WEN, GEN, AIEN, PIEN, GEN); ATIME decoded to integration time in ms and max count; CONTROL decoded to AGAIN in ×, PGAIN in ×, LDRIVE in mA; STATUS byte decoded to named flags (AVALID, PVALID, GINT, AINT, PINT, PGSAT, CPSAT); CDATAL burst read (8 bytes from 0x94) decoded as four 16-bit LE words showing clear/red/green/blue counts; PDATA read showing proximity count; GFLVL showing dataset count; GFIFO burst reads showing (U, D, L, R) per dataset; special address-only transactions to PICLEAR/CICLEAR/AICLEAR annotated by name; PPULSE and GPULSE decoded to pulse count and length in µs; CONFIG1.WLONG decoded as wait multiplier.
