# Chip Spec: ENS160

**Manufacturer:** ScioSense  
**Datasheet:** `datasheets/gas/ens160.pdf`  
**Category:** gas  
**Transports:** I²C, SPI

## Overview

The ENS160 is a digital metal-oxide (MOX) multi-gas sensor with four independent sensing elements and individual heater control. On-chip TrueVOC® algorithms produce calibrated air quality outputs — TVOC (Total Volatile Organic Compounds in ppb), eCO2 (equivalent CO₂ in ppm), and AQI-UBA (Air Quality Index per the German UBA standard, 1–5) — with no host-side signal processing required. The sensor performs automatic baseline correction, humidity and temperature compensation (when fed from an external sensor), and ozone compensation. Raw sensor resistance values are also available for custom processing.

## Transport Configuration

### I²C
- **Address:** `0x52` (MISO/ADDR pin low) — `0x53` (MISO/ADDR pin high)
- **Max clock:** 1 MHz (Fast Mode Plus); also supports 100 kHz and 400 kHz
- **Note:** CSn pin must be tied high to VDDIO to enable I²C. Auto-increment read supported.

### SPI
- **Mode:** CPOL=0 CPHA=1 (data clocked in on rising edge, out on falling edge) — MSB first
- **Max clock:** 10 MHz
- **Bit order:** MSB first
- **CS active:** low
- **Note:** Asserting CSn low locks the device into SPI mode until the next Power-On Reset. SPI write does not auto-increment; alternates address and data bytes.

## Register Map

| Address | Name | Size | R/W | Reset | Description |
|---------|------|------|-----|-------|-------------|
| `0x00` | PART_ID | 2 | R | `0x0160` | Device identity (LE: LSB=0x60, MSB=0x01) |
| `0x10` | OPMODE | 1 | R/W | `0x01` | Operating mode |
| `0x11` | CONFIG | 1 | R/W | `0x00` | Interrupt pin configuration |
| `0x12` | COMMAND | 1 | R/W | `0x00` | System commands (IDLE mode only) |
| `0x13` | TEMP_IN | 2 | R/W | `0x0000` | Host temperature for compensation (LE, Kelvin×64) |
| `0x15` | RH_IN | 2 | R/W | `0x0000` | Host relative humidity for compensation (LE, %RH×512) |
| `0x20` | DEVICE_STATUS | 1 | R | `0x00` | Operating status and validity flags |
| `0x21` | DATA_AQI | 1 | R | `0x01` | Air Quality Index UBA (bits 2:0, values 1–5) |
| `0x22` | DATA_TVOC | 2 | R | `0x0000` | TVOC concentration (LE, ppb) |
| `0x24` | DATA_ECO2 | 2 | R | `0x0000` | Equivalent CO₂ concentration (LE, ppm) |
| `0x30` | DATA_T | 2 | R | `0x4A8A` | Temperature used in calculations (LE, Kelvin×64; default=25°C) |
| `0x32` | DATA_RH | 2 | R | `0x6400` | RH used in calculations (LE, %RH×512; default=50%) |
| `0x38` | DATA_MISR | 1 | R | `0x00` | CRC of previous DATA_* read transaction |
| `0x40–0x47` | GPR_WRITE[0:7] | 8 | R/W | `0x00` | General purpose write registers (IDLE only) |
| `0x48–0x4F` | GPR_READ[0:7] | 8 | R | `0x00` | General purpose read registers |

> **Note:** DATA_ETOH (ethanol concentration in ppb) shares address `0x22` with DATA_TVOC — they are the same physical register. In standard operating mode DATA_TVOC returns the TVOC sum; DATA_ETOH is its ethanol-calibrated alias.

### Bit Fields

#### `OPMODE` (`0x10`)

| Bits | Name | Description |
|------|------|-------------|
| 7:0 | OPMODE | `0x00`=DEEP SLEEP, `0x01`=IDLE, `0x02`=STANDARD, `0xF0`=RESET |

#### `CONFIG` (`0x11`)

| Bits | Name | Default | Description |
|------|------|---------|-------------|
| 7 | — | 0 | Reserved |
| 6 | INTPOL | 0 | INTn polarity: 0=active low, 1=active high |
| 5 | INT_CFG | 0 | INTn drive: 0=open drain, 1=push/pull |
| 4 | — | 0 | Reserved |
| 3 | INTGPR | 0 | Assert INTn when new GPR_READ data available |
| 2 | — | 0 | Reserved |
| 1 | INTDAT | 0 | Assert INTn when new DATA_xxx data available |
| 0 | INTEN | 0 | Enable INTn pin |

> Typical value `0x23`: active-low, open-drain interrupt on new DATA (no pull-up needed).

#### `COMMAND` (`0x12`)

| Value | Name | Description |
|-------|------|-------------|
| `0x00` | NOP | No operation |
| `0x0E` | GET_APPVER | Firmware version → GPR_READ[4]=major, [5]=minor, [6]=release |
| `0xCC` | CLRGPR | Clear all GPR_READ registers |

#### `DEVICE_STATUS` (`0x20`)

| Bits | Name | Description |
|------|------|-------------|
| 7 | STATAS | 1=an OPMODE is currently running |
| 6 | STATER | 1=error (e.g. invalid OPMODE) |
| 5:4 | — | Reserved |
| 3:2 | VALIDITY_FLAG | 0=OK, 1=Warm-up, 2=Initial Start-up, 3=No valid output |
| 1 | NEWDAT | 1=new data in DATA_xxx registers; auto-cleared on first DATA_xxx read |
| 0 | NEWGPR | 1=new data in GPR_READ; auto-cleared on first GPR_READ read |

## State Diagram

```
POR → IDLE (0x01)
IDLE  --[write 0x00]--> DEEP SLEEP
IDLE  --[write 0x02]--> STANDARD (gas sensing active)
STANDARD --[write 0x01]--> IDLE
Any  --[write 0xF0]--> RESET → IDLE
```

## Initialization Sequence

1. Write `0x01` to OPMODE (ensure IDLE; device powers up in IDLE by default)
2. Read PART_ID (0x00, 2 bytes): verify `0x60` and `0x01` (little-endian `0x0160`)
3. (Optional) Write TEMP_IN and RH_IN for compensation
4. (Optional) Write CONFIG (`0x23`) to enable interrupt on new data
5. Write `0x02` to OPMODE to start STANDARD gas sensing
6. Poll DEVICE_STATUS until NEWDAT=1, or await INTn assertion
7. Check VALIDITY_FLAG before trusting readings (0=OK; discard while 1 or 2)

## Implementation Stages

Each chip is implemented in two stages. The Full class extends Minimal — it inherits everything and adds the rest.

### Minimal

Goal: expose calibrated air quality readings (AQI, TVOC, eCO2) with no configuration required beyond the transport.

| Operation | Parameters | Returns | Notes |
|-----------|------------|---------|-------|
| `init` | transport | — | IDLE check, PART_ID verify, start STANDARD mode |
| `read_air_quality` | — | `{aqi, tvoc_ppb, eco2_ppm}` | Polls NEWDAT; returns validity=0 data only |
| `status` | — | int (0–3) | VALIDITY_FLAG: 0=OK, 1=Warm-up, 2=Initial Start-up, 3=No valid output |

**Sensible defaults baked into Minimal:**
- OPMODE: STANDARD (0x02)
- No interrupt pin configured (polling only)
- No external T/RH compensation (device uses its internal defaults: 25°C, 50%RH)

### Full

Goal: expose complete chip functionality. Extends Minimal.

| Operation | Parameters | Returns | Notes |
|-----------|------------|---------|-------|
| *(inherits Minimal)* | | | |
| `set_compensation` | `temp_celsius: float, rh_percent: float` | — | Write TEMP_IN, RH_IN in IDLE or STANDARD |
| `read_tvoc` | — | float | TVOC in ppb |
| `read_eco2` | — | float | eCO₂ in ppm |
| `read_aqi` | — | int | AQI-UBA 1–5 |
| `read_ethanol` | — | float | Ethanol estimate in ppb (DATA_ETOH, alias of DATA_TVOC at 0x22) |
| `read_raw_resistance` | `sensor: int` (1 or 4) | float | Ohms; sensor 1→GPR_READ[0:1], sensor 4→GPR_READ[6:7] |
| `read_compensation_actuals` | — | `{temp_celsius, rh_percent}` | DATA_T and DATA_RH decoded |
| `get_firmware_version` | — | `(major, minor, release)` | Issues GET_APPVER command (IDLE mode required) |
| `configure_interrupt` | `enabled, active_high, push_pull, on_data, on_gpr` | — | Writes CONFIG register |
| `sleep` | — | — | Write OPMODE=0x00 (DEEP SLEEP) |
| `wake` | — | — | Write OPMODE=0x01 (IDLE), then 0x02 for sensing |

**Additional configuration options:**
- External T/RH compensation (improves accuracy outside 20–80%RH)
- INTn pin: polarity, drive mode, trigger source (DATA_xxx or GPR_READ updates)
- DEEP SLEEP for power saving between measurement bursts
- Raw sensor resistance for custom gas identification

## Data Conversion

### Temperature compensation input (TEMP_IN, 0x13)
```
raw_uint16 = round((celsius + 273.15) * 64)
write LSB first (0x13), then MSB (0x14) — update triggered on MSB write
Example: 25°C → (25 + 273.15) * 64 = 19082 = 0x4A8A
```

### Relative humidity compensation input (RH_IN, 0x15)
```
raw_uint16 = round(rh_percent * 512)
write LSB first (0x15), then MSB (0x16) — update triggered on MSB write
Example: 50%RH → 50 * 512 = 25600 = 0x6400
```

### Temperature readback (DATA_T, 0x30)
```
celsius = (uint16_le / 64.0) - 273.15
Example: 0x4A8A = 19082 → 19082/64 - 273.15 = 25.0°C
```

### Relative humidity readback (DATA_RH, 0x32)
```
rh_percent = uint16_le / 512.0
Example: 0x6400 = 25600 → 25600/512 = 50.0%RH
```

### TVOC, eCO2 (DATA_TVOC 0x22, DATA_ECO2 0x24)
```
value = uint16_le   (ppb for TVOC; ppm for eCO2 — no scaling needed)
```

### AQI (DATA_AQI, 0x21)
```
aqi = register_value & 0x07   (bits 2:0, range 1–5)
```

### Raw resistance (GPR_READ)
```
R_ohms = 2.0 ** (raw_uint16 / 2048.0)
Sensor 1 → GPR_READ[0:1] (bytes 0x48–0x49, little-endian uint16)
Sensor 4 → GPR_READ[6:7] (bytes 0x4E–0x4F, little-endian uint16)
Note: Sensor 1 values are in arbitrary logarithmic units (no resistance conversion needed for AQI).
```

| Event | Time | Notes |
|-------|------|-------|

## Implementation Notes

- **PART_ID verification:** PART_ID at 0x00 returns two bytes little-endian: `0x60` then `0x01`. The full 16-bit value is `0x0160`. Verify on init to confirm the device is present and not mis-wired.
- **VALIDITY_FLAG must be checked:** Outputs are unreliable while VALIDITY_FLAG ≠ 0. `read_air_quality` in Minimal must not return data until status=0. The driver should surface warm-up state via a separate `status()` call rather than silently returning stale readings.
- **NEWDAT auto-clear:** Reading any DATA_x register clears the NEWDAT flag. Read DATA_AQI, DATA_TVOC (4 bytes to 0x25), and DATA_ECO2 in a single burst read from 0x20 to capture consistent data and clear NEWDAT in one transaction.
- **SPI vs I²C exclusivity:** Once CSn is asserted low, the device switches to SPI permanently until Power-On Reset. Do not toggle CSn accidentally in I²C wiring.
- **COMMAND register is IDLE-only:** GET_APPVER and CLRGPR are only actioned when OPMODE=0x01. The driver must switch to IDLE, issue the command, then return to STANDARD.
- **Initial Start-Up persistence:** The 1-hour initial start-up flag is only stored to non-volatile memory after 24 continuous hours of operation. If powered off before 24 h, the device resumes Initial Start-Up mode on the next power-on.
- **DATA_MISR (CRC):** Optional. Polynomial `0x1D`, weight factor `76543210`. Tracks all bytes read from 0x20–0x37. Useful for validating data integrity over noisy buses.
- **Dual supply:** VDD (sensor core) must be 1.71–1.98 V. VDDIO (I/O) is independent and can be up to 3.6 V. Both require decoupling capacitors placed close to the chip. VDD must not drop below 1.71 V during operation.
